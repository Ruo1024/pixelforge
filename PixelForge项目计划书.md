## 第一部分 · 可行性分析

### 总评：Godot 是合适的选择吗？——是，且有双重先例背书

| 判断依据 | 结论 |
|---|---|
| 像素工具先例 | **Pixelorama**：纯 GDScript 写成的开源像素编辑器（动画时间线、洋葱皮、平铺、调色板、混合模式俱全），运行于 Godot 4.6.3，8500+ stars。它证明 Godot/GDScript 的性能与 UI 能力足以承载专业像素工具 |
| 节点工具先例 | **Material Maker 1.4**（2025-10）：用 Godot 内置 GraphEdit 控件实现了 200+ 节点的类 Substance Designer 程序化纹理工具——与你设想的节点工作流形态高度同构 |
| 关键内置能力 | GraphEdit（节点图）、TileMapLayer（拼地图）、HTTPRequest/WebSocketPeer（云 API 与 ComfyUI 通信）、Image API（像素算法）、多窗口、低功耗模式 |
| 跨平台与许可 | 一套代码导出 Win/macOS/Linux；MIT 许可可自由商用闭源 |
| 主要短板 | GDScript 数值运算慢（重算法需 PackedArray 批处理，必要时 Rust GDExtension/Compute Shader 加速）；GraphEdit 深度定制成本高（v1 克制使用）；内置 autotile 运行时 API 质量差（自实现 blob 映射表绕过） |

对比备选：Electron/Web（画布与图像性能皮实但打包臃肿、像素渲染锐利度控制繁琐）、Qt/C++（性能最强但开发速度慢数倍、AI 时代迭代效率劣势）、Tauri+Rust（生态年轻，画布组件自建成本高）。**结论：Godot 在"开发速度 × 像素渲染控制 × 节点图现成控件 × 跨平台"四个维度的综合分最高，且失败模式有先例兜底。**

### 功能 1：像素对齐 / 缩放 / 重采样 / 抖动 —— ★ 完全可行（核心价值所在）

**问题本质**：通用扩散模型生成的"像素画"是高分辨率伪像素——像素块不对齐网格、颜色数百上千、边缘带抗锯齿。这是行业公认痛点，也是你这个工具的第一立足点。

**算法路线已被开源先例验证**（unfake.js、proper-pixel-art、ai-pixelart-extractor 等）：

1. **网格检测**：Sobel 梯度 → 沿轴投影 → 自相关找周期 = 像素块尺寸；相位穷举 = 偏移。附置信度，低于阈值转手动模式（UI 拖网格线对齐，必须有的兜底）。
2. **重采样**：按网格逐单元取**众数色**（抗噪性远优于中心采样/最近邻），输出真·逻辑分辨率图。
3. **颜色量化**：median cut 自动缩减到目标色数，或映射到固定调色板（DB32、PICO-8 等），色距用 OKLab（感知均匀，优于 RGB 欧氏）。
4. **抖动**：Bayer 有序抖动为主（像素画社区美学偏好，网格感一致），Floyd-Steinberg 误差扩散备选（照片转像素场景）。

全部算法 GDScript 可实现（各 <500 行），512×512 图预算 2 秒内完成全管线，超标再上 Rust。**风险点唯一：对真实 AI 图的鲁棒性需持续调参——用程序生成的黄金测试集 + 手动模式兜底控制。**

### 功能 2：自动抠图 / 选取抠图 / 素材切分 —— ★ 完全可行（且不需要 AI）

像素画场景下传统算法碾压 AI 抠图：

- **白底/纯色底自动抠图**：四角采样推断背景色 → 边界泛洪填充（flood fill，OKLab 容差）。泛洪只清"外部连通"背景，物体内部同色高光自动保留——比全局色键更聪明的默认行为。
- **选取抠图**（从完整画面挖局部素材）：魔棒（容差泛洪选区）+ 矩形 + 套索三件套，选区布尔运算，提取为新素材并记录溯源链。
- **自动切分**（一图多素材）：8-连通域标记 + 近距合并启发（防止"飘浮火焰"被切离本体，合并距离可调可视）+ 碎屑过滤。AI 生成"一张图摆 6 个道具"的场景直接批量入库。
- **描边处理**：alpha 通道形态学膨胀/腐蚀实现 1px 内外描边的增删，对接风格预设。

rembg/SAM 类 AI 抠图仅在"复杂非纯色背景"才有意义，列为远期插件点，v1 不做。

### 功能 3：无限画布 + 节点工作流 + AI 批量生成 —— ★ 可行（工程量最大，是产品差异化核心）

三个子系统拆开看：

- **无限画布**：Godot 2D 场景树 + Camera2D 天生适配（平移缩放是引擎原生能力）；500 元素 60fps 预算有视口剔除护航；像素清晰靠全局最近邻采样（引擎一行设置）。成熟度高。
- **节点系统**：GraphEdit 现成控件 + Material Maker 先例。关键设计决策是**刻意比 ComfyUI 简单**：七种端口类型、线性链为主、自动批量展开（一个节点收到列表输入时自动逐项执行）、参数面板由节点声明自动生成（新节点零 UI 代码）。你设想的"风格提示词 → 尺寸比例 → 物体形态 → 模型 → 切分"链路正是 schema 的第一用例。
- **AI 生成**：Provider 抽象层统一云端/本地。2026-06 生态核实结论：
  - **Retro Diffusion**（像素专用模型）：活跃，公开 REST API，<$0.01/张，原生输出真像素+透明背景+tileset 风格——**首选默认后端**，其输出可跳过网格检测直接进量化。
  - **OpenAI gpt-image-1**：支持透明背景 PNG，用户基数大——通用兜底（输出伪像素，恰好展示清洗管线价值）。
  - Google Imagen 系强制 SynthID 水印且透明背景支持不明，v1 不接。
  - **ComfyUI 桥接**：REST /prompt + WebSocket 进度在 2026 仍是标准协议，桥接技术成熟（详见功能 3c 与第三部分 M7）。

**你设想的"ComfyUI 联动 + 内置画布保底易用性"双轨制完全成立**，且行业有空白：现有 AI 像素工具（PixelLab、Retro Diffusion 网页版）全是"生成器"，没有谁做了"生成后的批量加工工作台"。

### 功能 4：地图拼接 + 多层动效合成 —— ★ 可行（Godot 主场优势区）

- **拼图画板**：独立于无限画布的"有边界网格文档"（Board），双层模型：`tile 层`（网格吸附铺地形）+ `free 层`（像素级自由摆道具/角色/特效，可设混合模式 add/multiply）。
- **地形自动拼接**：采用行业标准 blob-47 邻接模型（256 种邻接态归并为 47 块）。注意：**不用** Godot 内置 Terrain 系统（调研确认其运行时 API 难用且有 bug），自实现映射表（256 case 可全量单测）。亮点流水线：AI 生成整张 tileset → 功能 2 切分 → 一键标注成地形组 → 笔刷涂抹自动选块。
- **多层动效**：动画素材 = 帧序列引用（轻量 JSON），free 层条目可引用动画素材，全板统一时钟 + 每实例相位偏移（20 个火把错峰摇曳的自然感）。特效层与人物层分离正是行业 spritesheet 惯例。
- **导出**：整图 PNG / 分层 PNG + 混合描述 JSON / 动效层逐帧序列 / spritesheet+坐标清单。

### 功能 5 拆解（详见第二部分）

前半（风格体系 + 手绘修补）**纳入 v1 主线**：风格研究转化为"风格预设"数据对象（产品灵魂），手绘工具做"Aseprite-lite"定位（修 AI 产出的最后 30%）。后半（体素 3D）**推迟立项**：2026-06 调研显示 AI 文本生成体素仍处早期（开源方案质量不稳定），且架构已预留接入位——推迟无技术代价（详见 M8 研究简报）。

## 第二部分 · 功能 5 深拆：风格体系、手绘工具与体素远期

### 2.1 像素风格体系 → 产品化为「风格预设」

调研提炼出六维风格参数模型（历史硬件限制塑造的风格语言）：

| 维度 | 取值示例 | 风格学依据 |
|---|---|---|
| 分辨率档位 | 8-bit / 16-bit / Hi-bit / HD-2D / 1-bit / GB | NES 每精灵 3 色+透明、GB 4 色绿——限制即风格 |
| 调色板 | DB32、PICO-8、Endesga64、自定义 | Lospec 社区文化，经典板 CC0 可内置 |
| 基准尺寸 | 16/32/48/64px | Celeste 角色 16×16、Stardew NPC 16×32、HLD 32×32 |
| 描边 | 无 / 黑 1px / 彩色 / 选择性 | 平台跳跃游戏多带描边，写实风多无描边 |
| 抖动 | 无 / Bayer / 误差扩散 | 复古档强抖动，现代档基本不用 |
| 透视 | 侧视 / 俯视 / 3⁄4 / 等距 | RPG 惯用 3⁄4，策略惯用等距 |

**预设的杀手锏在于"双端夹击"**：同一预设既作用于生成端（拼接提示词模板、映射 provider 风格参数），又作用于清洗端（量化目标调色板、网格先验、描边后处理），还作用于编辑端（默认色板、画布尺寸）。这是对"AI 风格一致性差"这一最大用户痛点的系统性回答——单点工具做不到，一体化工作台可以。

内置 6 档预设（GB/NES/16-bit·DB32/Hi-bit/1-bit/HD-2D 素材）开箱即用，用户可派生微调。远期可借鉴 Dead Cells 的 3D 转 2D 管线思路做"3D 参考辅助"（姿势参考生成多方向精灵）。

### 2.2 手绘修补工具（Aseprite-lite）

定位铁律：**修补 AI 产出，不替代 Aseprite**。功能取舍：

- 取：铅笔（像素完美模式，消 L 形冗余角）/橡皮/油漆桶/形状/选区移动、调色板面板、图层（可见性/不透明度/锁定）、动画时间线（帧管理/洋葱皮/帧标签/实时预览）、镜像绘制。
- 舍（v1）：cel 链接、骨骼、切片、瓷砖模式、Lua 级脚本。
- **独有的 AI 语境增强**：①约束模式——只能用色板内颜色作画，从机制上杜绝修图引入风格漂移；②杂色清扫——一键吸附 AI 图常见孤立噪点像素；③断线检测——高亮 1px 轮廓断点辅助人眼（不自动改，像素画轮廓是命脉）；④选区发送节点图局部重生成（inpainting 闭环）。

法律注意：Aseprite 自 2016 年起非开源（源码可见但禁再分发），交互范式可参考，代码不可挪用；其 $19.99 定价与 Retro Diffusion 插件 $65 的销量证明该品类付费意愿真实存在。

### 2.3 体素 3D（远期 M8，决策记录）

现在不做的理由：AI 文本生成体素 2026-06 仍处早期（text2vox 质量不稳、VoxAI 商业刚起步）；产品焦点风险大于收益。架构预留：新文档 Tab 类型 + Provider 能力位即可接入，无需推翻任何 v1 设计。

届时的推荐路径（已存档进执行包 M8）：内部体素网格与 2D 共用调色板体系；.vox（MagicaVoxel）互操作必做；**最有趣的差异化方向是"多视图像素图→体素重建"**——用已建成的 2D 管线生成三视图，启发式雕刻出体素粗胚，再配"逐层切片=一张像素画"的切片编辑视图，完整复用 M6 编辑器工具组。这条"像素师视角的体素工作流"路线比追逐 text-to-voxel 模型更稳，且竞品没有。立项触发条件已量化（用户需求占比/模型质量线/竞品压力，满足任一重评）。

## 第三部分 · 总体架构（摘要，权威定义在执行包）

### 四层洋葱 + 插件接入

```
UI 层（画布/节点图/拼图板/编辑器） → 应用服务层（项目/任务队列/撤销/素材库） →
领域核心层（像素算法/图模型/风格预设；纯逻辑零场景树依赖） → 基础设施层（IO/HTTP/日志）
插件通过唯一 PluginAPI 注入：节点类型/Provider/管线步骤/调色板/导出器/菜单
```

三条架构铁律保证"可扩展、易修改、可升级"：

1. **core 纯函数化**——算法不碰 UI 与场景树，可 headless 测试，性能不济时可整体平移 Rust GDExtension 而不动调用方。
2. **契约文件即法律**——项目文件格式（ZIP+JSON+PNG，人类可检查）、节点图 schema、Provider 接口、插件清单四大契约独立成文，全部带版本号与迁移链；未知节点降级为"幽灵节点"保数据不丢。
3. **内置能力插件化（自食狗粮）**——出厂的 RetroDiffusion/OpenAI provider 本身就以插件形态实现，CI 验证"拔掉任一内置插件主程序照常启动"，确保第三方扩展点真实可用。

### 关键机制速览

- **任务队列**：一切耗时操作（AI 请求/批量清洗/导出）异步化，画布永不冻结，进度在元素上原位角标显示。
- **撤销**：图像修改存"区域快照"而非全图，内存可控；AI 生成永远产出新元素，不覆盖原图。
- **凭据安全**：API key 本地 AES 加密存储，绝不进项目文件与日志（自动化 grep 测试看守）。
- **费用透明**：节点图运行前估算总费用，月度分桶累计，可设预算拦截。
- **素材溯源**：每个素材记录 provider/模型/提示词/种子/父素材链——既是可重现性，也是 AI 合规留痕。

## 第五部分 · 执行方式

配套的 `pixelforge-plan/` 文档包共 16 个文件，专为"交给更低级 AI 执行"设计：

- 每张任务卡含**目标/技术实现指导/涉及文件/验收标准**四段，验收标准可直接转化为测试，一次会话执行一张卡。
- 跨模块接口固化在 5 份契约文件中，执行 AI 不得擅改（发现缺陷走修订流程），从机制上防止多会话执行漂移。
- 调研结论（含 ⚠ 标注的易变项：API 字段、价格、版本号）存档于 RESEARCH-NOTES，落地前需重新核实的点已逐一标明。
- 质量底线（DoD、测试金字塔、CI 流水线、发布检查单）统一在 QUALITY.md。

建议执行节奏：M0 与 M1 由能力最强的模型执行（地基与核心算法），M2/M5 等模式化里程碑可下放更轻量模型，每里程碑出口做一次人工评审。
