# 外部交互成熟做法调研：无限画布 / 参考板 / 批量素材审阅

调研日期：2026-06-17  
用途：为 PixelForge 无限画布、参考板、批次内容节点与批量审阅交互提供可迁移设计依据。  
范围：Figma / FigJam、Miro、tldraw、PureRef、Apple Freeform、Eagle、Pinterest，以及 Lightroom / Capture One 等批量审片工具。优先采用官方帮助、产品文档、开源项目文档；Eagle 的比较模式补充采用其官方社区插件页。

## 0. 关键结论

PixelForge 的无限画布不应只是“能拖图的平面”，而应同时具备三种成熟工具的能力：

1. 参考板：快速导入、自动铺开、手动调尺寸、按视觉邻近关系比较。
2. 生产画布：语义缩放、像素清晰度、低干扰 pan/zoom、视口内高性能渲染。
3. 批量审阅器：候选图批次可横向比较、标记、过滤、拆分，并支持局部细节同步查看。

## 4. 对 PixelForge 任务拆分的建议

1. M3/M4 前置基础：先把 Camera/Input、LOD、卡片显示尺寸模型做稳。否则后续节点、批次、AI 结果落图都会建立在脆弱手感上。
2. 批次内容节点第一版不必做完整 DAM，但必须做 pick/reject/rating、批量过滤、拆分批次、side-by-side 对比。PixelForge 的“批量优先”需要这个闭环。
3. 图片卡片的“显示尺寸”不要等同于像素处理尺寸。像素处理仍基于源图/清洗输出；画布显示尺寸只负责参考板审阅和布局。
4. 语义缩放只切换显示详略，不承担进入编辑器；双击/命令进入编辑器，并用相机动画或共享元素过渡保持空间连续性。
5. 性能验收要从第一版就纳入：至少 100 张导入、1000 张卡片 pan/zoom、50 张批量审阅三类场景。
