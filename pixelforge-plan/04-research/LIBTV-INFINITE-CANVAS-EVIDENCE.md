# LibTV 无限画布产品证据与 PixelForge 取舍

> 日期：2026-07-12
> 用途：为 Beta 0.2 的产品壳提供界面与工作流依据，不推测 LibTV 内部技术实现。

## 1. 身份与证据范围

用户所指产品可确认是 LiblibAI 关联平台 [LibTV](https://www.liblib.tv/)，不是 LiblibAI 图片站。身份由 [LibTV 官方站](https://www.liblib.tv/)、[官方 CLI](https://www.liblib.tv/cli)、[官方 Agent Skills 仓库](https://github.com/libtv-labs/libtv-skills) 与 [LiblibAI 关联平台协议](https://www.liblib.art/activities/468ad794ccc7408d81757fd91be003ec?hideHeader=1) 交叉确认。

调研使用官方公开案例的只读画布、页面可访问性信息和官方 Agent 入口。创建动作的少量补充来自公开教程，只作待验证参考，不作为产品契约。第三方截图仅保存在已忽略的 `scratch/` 中，不进入仓库。

## 2. 可直接观察的产品事实

1. 一张画布同时放置长文本、图片、视频和批量媒体内容；节点不是只显示标题的流程符号。
2. 大型案例存在多个视觉分组和分支，组内与跨组连线共同表达项目关系。
3. 在约 10% 缩放下可查看完整项目结构；放大后仍能直接阅读、预览和操作节点中的内容。
4. 画布公开控件包含缩放、小地图切换和连线显隐，导航能力与内容能力处于同一工作区。
5. 官方 Agent 路径最终返回项目画布链接，自动生成结果仍回到同一可视空间，而不是落入割裂的结果页。

公开实测还显示图片节点可连接到多宫格候选或分镜结果。该证据说明内容模块既可承载单项，也可承载成组结果；它不证明具体数据结构、渲染框架或执行算法。

## 3. 对 PixelForge 的战略判断

PixelForge 当前底层画布已经具备平移、缩放、选择、撤销、持久化、视口剔除、图片元素、批次卡和轻节点连线。Beta 0.2 不应因追求相似外观而重写画布引擎；真正需要改变的是应用表层：把“摘要节点 + 外置模态参数”升级为“内容就在模块中”的工作台。

Beta 0.2 采用以下最小产品骨架：

- **输入模块**：卡内直接编辑对象列表或提示词。
- **参考模块**：卡内直接看见导入图片、名称和关键元数据。
- **生成模块**：卡内看见 Provider/预设摘要、运行入口、进度与错误状态。
- **结果批次模块**：复用现有批次审阅能力，直接容纳真实候选素材。
- **加工交接**：从批次卡进入清洗、抠图、切分、描边或导出，结果仍回到画布。

每种模块至少由标题/状态区、真实内容区、端口/关系区和就地主要动作组成。右侧检查器只承载低频或高级参数，不再是理解节点内容的唯一入口。

## 4. Beta 0.2 工作区壳

```text
┌──────────────────── 项目与全局动作 ────────────────────┐
│ ┌添加栏┐                                               │
│ │输入  │   ┌对象/提示┐ ──> ┌生成与状态┐ ──> ┌结果批次┐ │
│ │参考  │   │真实文本 │     │参数/运行 │     │真实缩略图│ │
│ │生成  │   └────────┘     └─────────┘     └─────────┘ │
│ │批次  │       └────── 参考图/风格 ────────┘            │
│ └─────┘                                                │
│ [导航/缩放/聚焦全部]                    [上下文检查器]   │
└────────────────────────────────────────────────────────┘
```

壳的优先级是“看得懂、装得下、跑得通、存得住”。Beta 0.2 不复制 LibTV 的视频节点、Agent 自动制片、协作、模板市场或完整多媒体时间线。

## 5. 不能从本次调研推出的结论

- 不知道 LibTV 使用何种画布库、空间索引、缓存或节点执行器。
- 未充分验证其框选、多选、端口类型、组折叠、吸附和对齐手势。
- 不能以竞品截图替代 PixelForge 的中文、像素清晰度、批量审阅和离线工作流验收。

因此，Beta 0.2 的实现继续遵守 PixelForge 现有契约与像素画产品目标，只迁移经过证据支持的“内容模块工作台”产品模式。
