# Beta 0.2-D — macOS 字体、弹窗与跨显示器缩放诊断

> 执行标签：**需单独开启最高思考强度的工程窗口**
> 建议分支：`codex/beta0-2-ui-diagnostics`
> 本卡是“复现—归因—实验—收口”任务，不预设最终修法。

## 1. 已知用户证据

在 macOS 的 Godot 编辑器 Debug 运行中已观察到：

- `Set Up PixelForge` 介绍文字与项目风格区域重叠；
- `Recover Autosave` 及其他 UI 文本存在底部截断或字形异常；
- 顶栏、检查器、状态栏的整体字号与可读性在当前显示环境中不可信。

源码审计已找到首启弹窗的高置信冲突链：`ConfirmationDialog.dialog_text` 与一个 `PRESET_FULL_RECT` 自建内容容器共享同一区域。但最终结构仍须在真实窗口验证。其余字体问题横跨 fallback font、Theme 字体度量、Control 最小尺寸、Dialog 自动布局、`content_scale_factor`、窗口像素倍率和分数 stretch，禁止先选一个原因再倒推证据。

## 2. 与常规开发的边界

常规模型负责 Beta 0.2 的工作区壳、内容模块、字符串迁移与中文翻译资源；本高强度分支负责：

- 真实窗口观测与诊断工具；
- UI 字体选择/加载与字体度量；
- 根窗口/跨屏缩放策略；
- 代表性弹窗的布局根因和通用解决原则；
- 中英文、编辑器 Debug 与导出包的视觉回归。

常规分支不得通过单个控件补 padding、私设 scale 或换字号来绕过本卡。高强度分支也不得顺手重做产品导航或内容模块。

## 3. 调试循环

### D-0 固化环境与复现

为每次运行记录：Godot 版本、运行方式（编辑器 Debug/导出包）、macOS 版本、屏幕型号、逻辑/物理尺寸、DPI、系统显示档、窗口尺寸、当前屏、`reported_scale`、`content_scale_factor`、窗口像素倍率、语言与字体。

至少覆盖：

1. Mac 内建 Retina；
2. 一块外接显示器（若当前无设备则保持“真机待验”，不能用模拟值替代）；
3. 窗口从内屏移到外屏、再移回；
4. English 与简体中文；
5. 编辑器 Debug 与 macOS 候选包。

首轮只采证，不改策略。保留代表性截图和 scale audit，使用自造文本/项目，不纳入受保护图像。

### D-1 将症状拆成独立链路

分别观测并记录真实 rect、最小尺寸、字体 ascent/descent 和父容器：

- 首启/设置弹窗内容相交；
- 标准 ConfirmationDialog 正文与按钮；
- 顶部工具栏、右侧检查器、底部状态栏；
- 画布自绘节点/批次文字；
- 跨屏后才出现的几何或清晰度变化。

一个症状可有多个原因；只有可重复触发且能被单变量实验改变，才进入根因结论。

### D-2 单变量实验

按证据选择实验变量，例如字体资源、Theme 字体度量、内容容器边界、弹窗尺寸协商、根窗口缩放模式或跨屏重算时机。每次实验只改变一类变量，记录：解决了什么、没有解决什么、是否破坏画布整数像素、是否影响另一种语言/显示器。

此处列的是实验维度，不是授权把它们全部实现。

### D-3 最小通用修复

选定能解释复现矩阵的最小方案，优先修复共用布局/主题/缩放层；必要的特例必须写明为什么不能由通用层解决。删除失败实验，不保留“碰巧有效”的叠加补丁。

### D-4 回归与证据收口

- 为首启弹窗补结构/几何自动测试，至少断言内容区不相交、按钮可达。
- 为代表中英文本补字体覆盖和布局压力测试；自动测试不得声称替代真机视觉。
- 复跑 UI scaling 静态守护、全量 GUT、目标平台构建。
- 用 D-0 同一矩阵复拍，形成前后对照和根因报告。

## 4. 退出条件

只有以下条件同时满足，本卡才可写“工程通过”：

1. `Set Up PixelForge`、恢复、未保存、Provider 设置、导出/错误等代表弹窗无文字相交、底部截断或不可达按钮；
2. 中英文顶栏、检查器、状态栏和画布模块文字完整可读；
3. 跨屏移动后 UI 尺寸合理，不出现累计放大/缩小；
4. 画布图像在各档缩放仍保持最近邻与设备像素整数对齐；
5. 报告包含复现条件、被证伪假设、最终根因、最小 diff、自动化结果和真实截图矩阵。

项目所有者未统一测试前，状态仍是“工程通过/人工待验”。

## 5. 禁止项

- 禁止只按当前截图写死像素高度、全局放大字号或给每个弹窗单独垫空白。
- 禁止以 headless 节点存在或 scale 数值正确证明字形视觉正确。
- 禁止在同一提交混入画布产品重构、Provider 或新功能。
- 禁止修改或提交 `test picture/`、`pixel/tests/fixtures/real/` 的图片。
