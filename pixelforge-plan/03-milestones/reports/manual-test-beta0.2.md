# PixelForge Beta 0.2 人工验收清单

> 候选：`0.2.0-beta.1`
> 分支：`codex/pixelforge-full-plan-goal`（本地，未合并、未 push）
> 本清单由项目所有者执行；完成前状态只能是“工程候选”。

## 1. 中文首次启动与语言

1. 使用一个没有 PixelForge 设置的 macOS 用户目录启动候选。
2. 确认中文系统默认显示简体中文；恢复提示与首次设置不同时重叠。
3. 检查首次设置正文、预设、开关和确认/取消按钮均完整可达。
4. 在“设置”中切换 English，再切回简体中文；关闭并重启，确认选择保持。

通过标准：顶栏、空状态、设置、恢复/未保存弹窗、模块标题、生成状态、批次清洗检查器和费用区无缺字、截断或中英混杂；Provider/模型/调色板品牌名可保留原文。

## 2. 空白画布最小链

1. 新建空白项目，确认“添加输入 / 导入参考图 / 打开示例”入口可见。
2. 选择“添加输入”，在对象模块中输入三项内容并应用。
3. 选择“导入参考图”，确认创建引用图模块并显示真实预览、素材名和来源；替换一次再清除，分别 Undo/Redo，确认引用与画布变化保持原子一致。
4. 使用画布添加入口补充尺寸、生成与结果批次模块，将引用图连接到生成模块，并连接其余有效端口。
5. 运行前分别制造引用为空、素材不存在和 PNG 无法解码的情况，确认可恢复错误归属引用图模块，旧结果不丢失；恢复引用后可继续运行。
6. 运行离线 mock，确认不同参考图产生不同确定性结果，结果进入批次且生成模块显示完成。
7. 删除被实时图或历史记录引用的素材，确认应用阻止删除并说明引用来源。

## 3. 模块、导航与批次审阅

1. 编辑对象、尺寸、Provider、批量数与 seed，确认卡内内容与检查器一致。
2. 折叠/展开模块，拖动位置；执行 Undo/Redo，确认图节点与画布视图原子变化。
3. 使用“聚焦选中 / 显示全部”和缩放控件。
4. 在结果批次中选择图片，执行 keep/reject/flag、过滤、聚焦和前后对比。
5. 对选中项执行一次“应用清洗”或“描边”，确认产生新结果且原图保留。
6. 启动一个可取消的 Provider 任务并取消，确认卡片显示已取消、旧批次不被覆盖。

## 4. 保存、恢复与导出

1. 保存到含中文目录和中文文件名的 `.pxproj`，关闭并重新打开。
2. 确认内容、引用图预览与资源状态、位置、连线、折叠状态、审阅状态、结果和 provenance 保持一致。
3. 用测试副本验证引用 PNG 缺失或损坏时项目仍可降级打开、显示结构化警告，保存后未知原始字节或画布项未被静默丢弃。
4. 制造未保存更改并触发新建/打开，分别验证保存、放弃更改和取消。
5. 验证 autosave 恢复为未保存副本，不覆盖原项目。
6. 导出到中文目录，确认 PNG/元数据存在，应用状态区显示产物位置。

## 5. 内屏、外屏与跨屏

分别在内建 Retina、外接显示器执行 English/简中代表旅程，并把窗口内屏 → 外屏 → 内屏移动：

- 顶栏、检查器、状态栏、恢复/未保存/设置/Provider/导出弹窗无重叠或截断；
- UI 尺寸不会每次跨屏累计放大或缩小；
- 画布图片保持最近邻和设备像素整数对齐；
- 记录 macOS、Godot、显示器、分辨率/缩放档、语言、运行方式和观察结果。
