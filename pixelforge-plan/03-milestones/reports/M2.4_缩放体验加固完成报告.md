## 结论

M2.4 的代码侧补足已完成：画布平移位置现在吸附到物理像素网格，FileDialog 统一使用 Godot 自绘路径以跟随 `content_scale_factor`，主窗口新增跨屏 live re-scale 防抖与可关闭设置，`--scale-audit` 可输出顶层 Control 和画布吸附日志，M2.4 出口脚本已落地。

## 实现内容

| 模块 | 路径 | 说明 |
|---|---|---|
| 界面缩放策略 | `pixel/ui/shell/interface_scale_policy.gd` | 抽出自动缩放检测、mac Retina fallback、屏幕快照比较和 `Window.content_scale_factor` 应用。 |
| 窗口尺寸策略 | `pixel/ui/shell/window_scale_policy.gd` | 抽出启动窗口尺寸、最小尺寸和 mac 点/像素换算，避免 `main.gd` 超 lint 行数。 |
| live re-scale | `pixel/ui/shell/main.gd` | 轮询当前屏幕快照，防抖后重算界面缩放、主题、窗口最小尺寸与画布变换；默认开启，可由 `ui.live_rescale=false` 关闭。 |
| 缩放审计 | `pixel/ui/shell/scale_audit.gd` | 支持 `--scale-audit` 输出顶层 Control rect、`F_canvas`、`item_layer_pos_physical` 与吸附状态。 |
| FileDialog 策略 | `pixel/ui/shell/dialog_scale_policy.gd` | 统一 `use_native_dialog=false`，让项目内 FileDialog 跟随 Godot UI 缩放。 |
| 画布吸附 | `pixel/ui/canvas/canvas_scale_policy.gd`、`pixel/ui/canvas/infinite_canvas.gd` | 新增物理像素网格吸附、相机中心反推；`world_to_screen/screen_to_world` 改为读取实际 layer transform。 |
| 画布瘦身 | `pixel/ui/canvas/canvas_pixel_grid_renderer.gd`、`pixel/ui/canvas/canvas_tool_input_policy.gd` | 拆出像素网格绘制与工具输入分流，保持主画布文件低于 900 行 lint 上限。 |
| 对话框接线 | `pixel/ui/shell/main.gd`、`pixel/ui/shell/m2_1_ui_controller.gd`、`pixel/ui/inspector/cleanup_inspector.gd` | 打开/保存/导出、导入图片、导入调色板 FileDialog 全部接入统一策略。 |
| 设置默认值 | `pixel/services/settings_service.gd` | 新增 `ui.live_rescale=true` 默认设置。 |
| 测试 | `pixel/tests/smoke/test_infinite_canvas.gd`、`pixel/tests/smoke/test_main_window_ui.gd` | 覆盖物理像素吸附、坐标 roundtrip、屏幕快照变化检测、FileDialog 自绘策略。 |
| 守护脚本 | `pixel/scripts/check_ui_scaling.sh`、`pixel/scripts/verify_m2_4.sh` | 新增 `item_layer.position` 必须经吸附函数的弱守护；M2.4 出口脚本覆盖 lint/test/scaling/export/图片红线。 |
