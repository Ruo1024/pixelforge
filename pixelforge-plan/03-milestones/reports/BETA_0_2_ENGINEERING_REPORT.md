## B2-0 产品壳与验收图冻结

### 用户旅程与界面区域

Beta 0.2 的唯一主旅程是：空白或示例起步 → 编辑对象列表 → 导入参考图 → 确认生成摘要并运行离线 mock → 在批次中审阅 → 对选中项执行代表加工 → 保存重开 → 导出并看到位置。

工作区固定为五个职责区：

1. 紧凑全局动作区：项目、新建、打开、保存、导出、设置；
2. 画布创作入口：添加输入、导入参考、打开示例与画布内添加菜单；
3. 主画布：对象输入、参考图、生成、批次与连线共用同一坐标；
4. 上下文检查器：按当前选择显示低频参数或批次/素材操作；
5. 导航与状态区：缩放、聚焦选中、聚焦全部、任务/费用/错误与导出结果。

### 内容模块层级与状态

| 模块 | 卡内核心内容 | 卡内主动作 | 检查器内容 | 稳定持久化 |
|---|---|---|---|---|
| 对象/提示词 | 多行条目、条目数 | 编辑条目 | 低频文本参数 | `object_list.params.items` + `canvas.collapsed` |
| 参考图 | 真实预览、名称、资源状态 | 导入/替换 | 素材元数据 | 待批准的 `image_input.params.asset_id` + `canvas.collapsed` |
| 生成 | Provider、尺寸、批量摘要、运行状态 | 运行/取消/重试 | seed、费用与 Provider 设置 | 图参数与稳定结果；运行进度/错误为瞬态 |
| 结果批次 | 真实缩略图、选择、审阅标记 | keep/reject/flag、加工、导出 | 过滤、对比与低频批次动作 | 既有 batch params、provenance 与 canvas review state |

共同状态为默认、选中、运行、失败、完成和失效引用。选择/运行/失败通过一致的边框、状态条与原位动作呈现；运行进度和最后一次瞬态错误不写项目文件，成功物化结果、provenance、内容、布局、折叠与审阅状态写项目文件。

### 空白与示例路径

- 空白项目首屏同时提供“添加对象输入”“导入参考图”“打开离线示例”；
- 画布右键/Tab 的添加入口必须能从零创建第一张图，不要求已有 graph binding；
- 离线示例随包可重建为自包含项目，默认包含对象、参考、生成、批次与有效连线；
- 示例和空白路径最终调用同一图模型、AssetLibrary、TaskQueue、UndoService 与项目保存服务。

### 契约复核

`PROJECT-FORMAT` 与 `GRAPH-SCHEMA` 已覆盖图/画布分离、节点参数、折叠、批次物化、审阅状态和 provenance，不需要新格式版本。

发现一个实现前必须批准的最小契约缺口：`GRAPH-SCHEMA` 已列出 `image_input`，但未定义其参数、执行时的 AssetLibrary 解析与失效资源语义；项目引用完整性也未覆盖图参数中的素材引用。提案为：`image_input.params.asset_id` 是唯一资源引用；运行器从受控图上下文解析为 `Image`；缺失/失效返回可恢复错误；保存与删除引用检查覆盖 `image_input` 与 `batch.params.asset_ids`。批准前不实现或静默旁路。
