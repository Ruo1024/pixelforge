## B2-0 产品壳与验收图冻结

### 用户旅程与界面区域

Beta 0.2 的唯一主旅程是：空白或示例起步 → 编辑对象列表 → 导入参考图 → 确认生成摘要并运行离线 mock → 在批次中审阅 → 对选中项执行代表加工 → 保存重开 → 导出并看到位置。

工作区固定为五个职责区：

1. 紧凑全局动作区：项目、新建、打开、保存、导出、设置；
2. 画布创作入口：添加输入、导入参考、打开示例与画布内添加菜单；
3. 主画布：对象输入、参考图、生成、批次与连线共用同一坐标；
4. 上下文检查器：按当前选择显示低频参数或批次/素材操作；
5. 导航与状态区：缩放、聚焦选中、聚焦全部、任务/费用/错误与导出结果。

### 内容模块层级与状态

| 模块 | 卡内核心内容 | 卡内主动作 | 检查器内容 | 稳定持久化 |
|---|---|---|---|---|
| 对象/提示词 | 多行条目、条目数 | 编辑条目 | 低频文本参数 | `object_list.params.items` + `canvas.collapsed` |
| 参考图 | 真实预览、名称、来源与资源状态 | 导入/替换/清除 | 素材元数据 | `image_input.params.asset_id` + `canvas.collapsed` |
| 生成 | Provider、尺寸、批量摘要、运行状态 | 运行/取消/重试 | seed、费用与 Provider 设置 | 图参数与稳定结果；运行进度/错误为瞬态 |
| 结果批次 | 真实缩略图、选择、审阅标记 | keep/reject/flag、加工、导出 | 过滤、对比与低频批次动作 | 既有 batch params、provenance 与 canvas review state |

共同状态为默认、选中、运行、失败、完成和失效引用。选择/运行/失败通过一致的边框、状态条与原位动作呈现；运行进度和最后一次瞬态错误不写项目文件，成功物化结果、provenance、内容、布局、折叠与审阅状态写项目文件。

### 空白与示例路径

- 空白项目首屏同时提供“添加对象输入”“导入参考图”“打开离线示例”；
- 画布右键/Tab 的添加入口必须能从零创建第一张图，不要求已有 graph binding；
- 离线示例随包可重建为自包含项目，默认包含对象、参考、生成、批次与有效连线；
- 示例和空白路径最终调用同一图模型、AssetLibrary、TaskQueue、UndoService 与项目保存服务。

### 契约复核

`PROJECT-FORMAT` 与 `GRAPH-SCHEMA` 已覆盖图/画布分离、节点参数、折叠、批次物化、审阅状态和 provenance，不需要新格式版本。

项目所有者已批准优化后的 [`IMAGE_INPUT_CONTRACT_PROPOSAL.md`](IMAGE_INPUT_CONTRACT_PROPOSAL.md)，并已同步到 `GRAPH-SCHEMA` 与 `PROJECT-FORMAT`。`image_input.params.asset_id` 是唯一实时资源引用；运行器只解析目标节点上游闭包，通过受控 `PFGraphContext` 向下游传递图像副本；缺失、未找到与解码失败均返回归属引用节点的可恢复错误。项目引用完整性同时覆盖实时图参数、历史记录和稳定 provenance，损坏或缺失 PNG 允许降级打开且不会静默丢弃原始字节或画布项。

## B2-D 显示诊断矩阵与根因清算

环境：macOS 26.5.1（25F80）、Apple M5、Godot 4.6.3 stable。外接 BenQ SW242Q 的 macOS 逻辑区域为 2560×1600，Godot 设备像素为 5120×3200、DPI 126、reported scale 1.0；内建 Liquid Retina 的逻辑区域为 1470×956，Godot 设备像素为 2940×1912、DPI 257、reported scale 2.0。

| 运行态/步骤 | 语言 | 缩放证据 | 可见结果 | 状态 |
|---|---|---|---|---|
| 候选包，BenQ 启动 | English | `content_scale_factor=2.0`，window pixel scale 1.0 | 首屏、离线示例、参考图、生成与批次可读；像素对齐守护为真 | agent 通过 |
| 候选包，移入内建 Retina | English / 简中 | 实时重算为 1.25，window pixel scale 2.0 | 无累计缩放；示例链、结果批次折叠、中文设置与模块列表可读 | agent 通过 |
| 候选包，移回 BenQ | 简中 | 恢复 2.0 / 1.0 | 中文“添加流程模块 → 结果批次”可见；未出现二次放大/缩小 | agent 通过 |
| Godot 编辑器 Debug 跨屏 | 待复验 | 4.6.3 编辑器进程启动后 Computer Use 屏幕捕获持续失败 | 无可靠截图，不以候选包结果替代 | 待工程复验 |

被证伪的假设：单纯提高全局字号不能解释弹窗内容相交；单纯依赖 `screen_get_scale` 不能解释外接屏 reported scale 1.0；headless 的节点存在也不能证明字体视觉正确。最终根因为首启恢复/onboarding 同时争用 exclusive child、autowrap Label 在未稳定宽度时参与循环最小尺寸协商，以及 macOS 逻辑点/设备像素与界面可读倍率被混为一套数值。最小修复分别落在弹窗顺序、稳定内容宽度与根窗口统一 live rescale；画布仍独立按设备像素整数对齐。
