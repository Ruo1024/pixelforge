# M2.4 编辑器嵌入式调试修复完成报告

完成日期：2026-06-17
分支：`codex/m2-debug-viewport-diagnosis`
对应反馈：从 Godot 编辑器 Debug/Test Window 拉大窗口后，PixelForge 工作区仍按固定区域居中显示，外圈出现黑色盲区；从独立运行窗口启动时则可随窗口正常缩放。

## 1. 结论

本轮问题最终定位为 Godot 4.6 编辑器 Game embedding 调试视图配置问题，而不是 PixelForge 产品窗口布局问题。

关键证据：

- 用户截图 p1：无顶部 Game bar 的独立运行窗口能自由随窗口缩放，PixelForge 工作区填满窗口。
- 用户截图 p2/p3：带 `输入 / 2D / 3D` 顶部 Game bar 的嵌入式 Debug/Test Window 仍居中显示固定基准区域，外圈为编辑器 Game View 背景。
- 本地 `.godot/editor/project_metadata.cfg` 在测试后被 Godot 写回 `embed_size_mode=0`，说明当前编辑器会话仍处于 Fixed Size 语义。
- Godot 官方 Game embedding 文档说明，Fixed Size 会按项目窗口尺寸固定显示；Stretch to Fit 才匹配非嵌入窗口的缩放行为。

最终处理：

1. 保留现有产品窗口代码与 `project.godot` stretch 设置，不继续用应用代码补偿编辑器调试视图。
2. 新增本地配置脚本 `pixel/scripts/configure_editor_game_view.sh`：
   - 设置项目本地 `.godot/editor/project_metadata.cfg` 的 `embed_size_mode=2`，即嵌入式 Game View 使用 Stretch to Fit。
   - 设置 Godot 4.6 全局 `editor_settings-4.6.tres` 的 `run/window_placement/game_embed_mode=2`，默认禁用 Game embedding，让 Play 走独立窗口路径。
   - 若检测到 Godot 编辑器仍在运行，提示重启编辑器，避免当前会话继续覆盖本地配置。
3. `verify_m2_4.sh` 开头运行该脚本，使本地门控先修正调试环境。
4. 手动测试文档与架构规范记录：看到顶部 Game bar 时，Fixed Size 盲区是编辑器调试设置，不应在 PixelForge 产品窗口代码中兜底。
