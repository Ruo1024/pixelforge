2. 解压到仓库外的临时目录，不覆盖旧候选。
3. 准备一个无 PixelForge 设置的干净 macOS 用户目录，以及一个可安全保存测试 `.pxproj`/导出文件的临时目录。
4. 准备项目所有者自己的 OpenAI 或已支持云端 Provider 密钥；不要把密钥写进项目、截图、报告或聊天。

- [ ] 顶栏、资源抽屉、检查器、卡片、状态与错误没有异常中英混杂、缺字、底部截字或控件重叠。

## 2. 空画布创建与系统图片落点

- [ ] 新建空项目，固定创建入口、“添加输入 / 导入参考图 / 打开示例”可见。
- [ ] 双击空白处或 Tab 在指定位置添加对象/提示词模块。
- [ ] 从 Finder 拖入图片，确认在落点成为真实预览卡；再测试系统剪贴板粘贴。
- [ ] 替换、清除参考图并 Undo/Redo，确认卡片、图参数和素材引用一致。

## 3. 卡内编辑、复制、连线、成组与导航

- [ ] 在卡内编辑提示词、对象、风格、尺寸、Provider/模型、批量与 seed。
- [ ] 测试 Cmd+C/V/D、Delete/Backspace、Cmd+Z/Shift+Cmd+Z；一次动作只撤销一次完整意图。
- [ ] 从端口拖出连线；有效/无效预览、方向和失败原因清楚，移动卡片后连线跟随。
- [ ] 选择多个模块成组，移动/删除/恢复 frame；成员和内部连线保持。
- [ ] 隐藏/显示连线；隐藏时旧连线不可误选，但新建连线预览仍可见。
- [ ] 使用 100% 编辑、约 50% 分支浏览、约 10% 总览、小地图、显示全部和返回局部，确认主要 frame、模块类型、结果和运行/失败状态可辨认。

## 4. 使用项目所有者真实云端密钥生成

- [ ] 在 Provider 设置中输入自己的真实密钥；确认项目文件、候选详情和复制设置不出现密钥。
- [ ] 选择具体模型，确认只显示该模型支持的参考图数量、尺寸、批量和 seed 能力。
- [ ] 提交单分支；检查费用/预算确认、等待、运行、进度、完成和结果回到同一画布。
- [ ] 制造凭据错误、限流或录入错误并重试；错误归属目标分支，其他结果和旧批次不被覆盖。
- [ ] 启动后取消；确认仅取消当前目标/阶段范围，旧结果保留。

## 5. 多参考、候选详情与结果继续生成

- [ ] 添加至少两张有序参考图，调整顺序、替换和移除；真实请求结果符合项目所有者对顺序影响的预期。
- [ ] 打开单个候选详情，核对模型、提示词、seed、尺寸、参考摘要、费用和来源；复制提示词/设置不含密钥或完整 Provider 响应。
- [ ] 将一个结果作为单参考，再选择多个结果作为有序参考。
- [ ] 从结果创建继续分支；旧结果保留，新分支设置可编辑并可再次运行。
- [ ] 对单个和多个结果执行审阅、代表加工和导出；来源/provenance 能回到正确分支。

## 6. 结构化批量、整组运行与本地工作流

- [ ] 粘贴至少 50 行对象，逐行编辑数量和启用状态；测试筛选、全选/取消和只运行选中行。
- [ ] 运行前核对选中行数、预计请求数、最大结果数和已知费用；确认每行结果数量不与旧 `batch_size/per_subject` 相乘。
- [ ] 运行含多个生成目标的 frame；frame 外必要上游可参与，无关/不完整分支不运行；一个目标失败不抹掉其他目标结果。
- [ ] 在多请求中制造部分失败；只重试失败行，成功行结果保留；候选可定位到稳定输入行。
- [ ] 将显式 frame 保存为工作流；检查摘要中的节点、内部连线、外部连线排除和待填写素材槽。
- [ ] 在新项目从资源抽屉搜索并插入用户工作流；位置明确、新内容保持选中，一次 Undo/Redo 完整撤销/恢复。
- [ ] 填入新素材后编辑、连接并运行；原项目与模板实例互不联动。
- [ ] 测试三个内置模板；测试用户模板重命名、删除，并确认删除模板文件不会删除已插入实例。
- [ ] 如能安全操作一份临时副本，损坏一个本地模板 JSON，确认工作流库仍打开其他模板并显示可理解的警告。
