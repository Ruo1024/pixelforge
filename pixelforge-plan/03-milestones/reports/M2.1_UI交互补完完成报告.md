# M2.1 UI 交互补完完成报告

日期：2026-06-16

## 1. 完成结论

M2.1 已完成并通过本地门禁：明确图片导入入口、M2 参数预览对话框、W/M/L 像素选区工具、左下角缩放滑条、非纯色底错误引导、无连线批次卡 MVP、批次菜单处理与对应测试。

补充修复：工具管理器默认不激活 W/M/L 工具，导入图片后普通左键仍可拖动画布素材；用户按 W/M/L 或点击工具按钮后才进入像素选区模式，Esc 或再次点击当前工具按钮可回到移动模式。

追加修改：画布左下角新增离散 zoom level 缩放滑条，降低鼠标滚轮缩放过头的概率；滚轮缩放、程序缩放和滑条显示保持双向同步。

本报告只覆盖 M2.1 开发改动；工作区中接手前已存在的其他计划/研究文档改动保持原样，未纳入本报告范围。

+func _combine_with_current(new_selection: PFSelection, modifiers: int) -> PFSelection:

## 8. 后续注意

- M2.1 的 `batch_card` 是 M3 前临时画布卡，不带端口、不连线；M3 实现正式 graph 持久化时应迁到 `type:"node"` + `graphs/{graph_id}.json` 的 `type=batch` params。
- `m2_1_ui_controller.gd` 为 UI 接线层，调用了画布内部批次卡方法，目的是把暂存 MVP 留在 UI 层边界内；M3 可把这些方法替换为正式 graph 节点 API。
