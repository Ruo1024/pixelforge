# M2.4 缩放终因追加修复完成报告

完成日期：2026-06-17
对应反馈：无缩放 2K 屏 UI 偏小；macOS 带缩放内屏 UI 过大裁切，拉大窗口不能解决。

---

## 1. 结论

本次追加修复已完成并通过 `pixel/scripts/verify_m2_4.sh`。

终极原因不是单个 Control 或 Godot 弹窗，而是 M2.4 前一版把同一个 `_interface_scale` 同时用于三件事：

1. `Window.content_scale_factor` 的 UI 视觉可读倍率。
2. 启动窗口默认尺寸 / 最小尺寸的窗口像素倍率。
3. macOS 上 `screen_get_usable_rect()` 与 `Window.size` 之间的点/像素换算倍率。

这会在两类屏幕上反向出错：

- 无缩放 2K 外屏：`screen_get_scale()` 为 1.0，旧阈值又只从 3200/1800 才抬档，导致 `2560x1440 -> 1.0`，界面偏小。
- macOS 缩放内屏：Retina/缩放信号被直接当成 UI 视觉 2.0，右侧检查器和 Debug 运行工具栏叠加后很容易过大裁切；同时窗口几何也跟同一倍率耦合，放大窗口不能从根上解决。

本次改为拆分：

- `content_scale_factor`：面向人眼可读性的 UI 视觉档，自动值为 1.0 / 1.25 / 1.5 / 2.0。
- `window_pixel_scale`：屏幕/Retina backing 像素档，只用于窗口几何和 mac 点/像素换算。
- `window_geometry_scale`：启动默认窗口尺寸使用 `max(content_scale_factor, window_pixel_scale)`，保证 Retina 内屏不会因为 UI 视觉降到 1.25 而打开成太小窗口。

官方依据：

- Godot `Window.content_scale_factor` 是窗口内容基础缩放；`content_scale_size` 是虚拟像素，`Window.size` 是窗口物理像素：https://docs.godotengine.org/en/stable/classes/class_window.html
- Godot `DisplayServer.screen_get_scale()` 返回屏幕缩放因子；`screen_get_size()` / `screen_get_usable_rect()` 文档口径为像素：https://docs.godotengine.org/en/stable/classes/class_displayserver.html
- Godot 非游戏应用推荐 `disabled` stretch mode，并在运行时用 `root.content_scale_factor` 配置 UI 缩放；Windows 等平台需要按分辨率猜测并允许用户覆盖：https://docs.godotengine.org/en/stable/tutorials/rendering/multiple_resolutions.html

## 3. 行为变更

自动档位调整：

- `2560x1440 / scale=1.0 -> content_scale_factor=1.25`
- `3840x2160 / scale=1.0 -> content_scale_factor=1.5`
- `5120x3140 / scale=1.0 -> content_scale_factor=2.0`
- macOS 点口径 `1244x778` / `1334x834` / `1440x900 -> content_scale_factor=1.25`
- macOS Retina 内屏 `3024x1964 / reported_scale=2.0 -> content_scale_factor=1.25`，但 `window_pixel_scale=2.0`

窗口几何调整：

- 启动窗口默认尺寸使用 `window_geometry_scale=max(content_scale_factor, window_pixel_scale)`。
- macOS `usable_rect` 只有在 `<= 2200x1400` 且 `window_pixel_scale>1` 时按点口径转换；`3024x1964` 这类已经像物理像素的可用区域不再二次乘。
- `Window defaults applied`、`Interface scale resolved`、`Interface scale reapplied`、`Scale audit` 日志都能区分 UI 视觉档和窗口像素档。
