## 0. 候选准备

1. 对 `pixel/build/PixelForge-0.6.0-beta.2-macOS.zip` 运行 `shasum -a 256`，确认与上方值一致。
2. 优先解压到仓库外临时目录并独立启动候选，不覆盖旧候选。
3. 准备可安全保存临时 `.pxproj`、模板和导出结果的目录，以及项目所有者有权使用的测试图片。
4. 准备内建 Retina 和外接屏；记录 macOS 版本、机器、分辨率/缩放档、测试时间和候选路径。
5. 真实云端步骤仅使用项目所有者自己的密钥；不要把密钥写入项目、截图、报告或聊天。
6. 如果必须从 Godot 编辑器运行，先在仓库根目录执行 `./pixel/scripts/configure_editor_game_view.sh`，完全重启 Godot，并确认 PixelForge 在独立原生窗口中打开；嵌入式 Game 视图不能作为本轮验收环境。

## 1. 工作区与双语入口

- [ ] 在 1080×560、1280×720、1440×900 或最接近的可用窗口尺寸检查 English 与简中；顶栏、48px 左侧栏、状态栏、缩放、小地图和主要动作完整可达。
- [ ] 长项目标题在固定槽中正确省略，悬停可看到完整标题；不挤掉运行、导出、检查器和 More。
- [ ] 打开检查器：1440 宽及以上靠右停靠；更窄窗口在画布右侧不透明覆盖；开关前后相机中心和倍率不跳变。
- [ ] 空项目能从添加输入、导入参考图和打开示例开始；过程反馈、结果和下一步均在应用内连通。
- [ ] English/简中切换后，顶栏、卡片、Frame、检查器、状态与菜单无裸字符串、缺字、截字、重叠或不可达动作。

## 2. 1A 混合卡片内容层级

- [ ] 提示词、对象列表、风格、尺寸、参考集合与生成卡呈现结构化数据，而不是只有类型标题和摘要。
- [ ] 单图输入、独立 Sprite 和结果批次以媒体/内容优先展示，预览足够辨认且不被工具控件压缩。
- [ ] 生成卡能辨认 Provider、模型、参考、尺寸、批量、seed、费用和任务状态；内容来自真实项目数据。
- [ ] 普通、Hover、选中、聚焦、运行、完成、失败、禁用和锁定状态在卡片家族间一致且易区分。
- [ ] 使用首要动作、More、检查器和结果继续入口，确认入口、反馈、结果与下一步连通。

## 3. 2B 改名、自由缩放与 Undo

- [ ] 分别对正式 Graph 卡、兼容旧 Batch 卡和独立 Sprite 卡改名；Enter 与失焦提交，Esc 取消，空白标题回退本地化名称。
- [ ] 分别从右下角自由调整三类卡片尺寸；达到最小/最大限制时稳定，正文与预览重新排版，端口和连线同帧跟随。
- [ ] 对 Frame 改名和自由缩放；双击标题不重置尺寸，成员关系和内部连线不变。
- [ ] 对标题和尺寸执行 Cmd+Z、Cmd+Shift+Z；每个完整意图只撤销/重做一次。
- [ ] 折叠、复制/粘贴、Duplicate 后标题和尺寸正确；副本可独立编辑，不联动原卡。
- [ ] 锁定卡片不能拖动、改名或缩放，但仍可选中查看；10%/25%/50% 下不出现误触缩放手柄。

## 4. 3C 全结果展开

- [ ] 生成或构造含 1、12、13、50 张结果的批次，在 All 下结果卡按宽度换列并自动增高。
- [ ] 50 张结果序号 1–50 全部存在；滚动外部画布可到达最后一张，没有分页、结果卡内部滚动、`+N` 或被隐藏尾项。
- [ ] 首张、中间和第 50 张都可选中、标记审阅、打开详情、作为参考、继续生成、加工或导出；动作定位到正确结果。
- [ ] 切换审阅筛选时只显示符合条件的真实结果；切回 All 后 50 张全部恢复且结果身份不变。
- [ ] 任务进行中占位数量与预期一致；真实结果逐步替换占位时卡片外框不无故跳动，旧结果不丢失。
- [ ] 调整结果卡宽度后列数和高度立即重排，选择、序号、命中区域与连线锚点保持正确。

## 5. 语义缩放、Frame 与连线

- [ ] 10% Map：主要空间关系、Frame 和状态可辨认；正文、端口、菜单与缩放手柄不会误触。
- [ ] 25% Overview：Frame 大标题、卡片类型/状态与主要关系可读；没有把完整正文缩成不可读小字。
- [ ] 50% Browse：大号单行标题/摘要、图片和结果预览可读；端口、编辑器和缩放手柄隐藏。
- [ ] 75%–300% Edit：标题、正文、端口、主要操作和缩放手柄完整恢复。
- [ ] 400% Inspect：像素检查层级清楚，画布图片保持最近邻与设备像素对齐。
- [ ] 在各档位往返时卡片/Frame 世界边界、端口锚点、连线端点和 Frame 归属不变化。
- [ ] 端口只在 75% 及以上显示并可开始连线；可见点约 12px 直径、命中热区约 40px 直径，缩放后屏幕尺寸稳定。
- [ ] 在低倍率双击卡片后居中进入 100%；双击 Frame 后以不超过适配值的最近离散倍率居中，Frame 尺寸不被重置。
- [ ] 保存项目为 50%，关闭重开后首帧画布、倍率标签、滑块和索引都显示 50%，不先闪成 100%。

## 6. 保存、旧项目、剪贴板与模板往返

- [ ] 打开 Beta 0.5 或更早的临时旧项目副本；旧 Batch、Graph、Sprite、Frame、结果、连线和素材无数据丢失。
- [ ] 为旧卡新增标题和尺寸并保存到新副本，关闭重开后保持；未识别字段仍保留。
- [ ] 复制粘贴含自定义标题、尺寸、Frame 和内部连线的选区；ID 正确重映射，样式和几何保持，一次 Undo 完整撤销。
- [ ] 将选区保存为本地工作流再插入；标题/尺寸保持，素材槽与结果仍按模板安全规则清空，原项目和实例互不联动。
- [ ] 在中文路径和中文项目名下保存重开；检查标题、尺寸、折叠、相机倍率、结果筛选和 provenance。
- [ ] 对创建、移动、删除、改名、缩放、粘贴和模板插入执行 Undo/Redo，确认既有 Graph、Provider、TaskQueue 和项目格式行为未退化。

## 7. 生成状态与真实云端代表旅程

- [ ] 使用项目所有者自己的 Provider 密钥运行一个代表分支；密钥不出现在项目、日志、候选详情、复制设置或错误文案中。
- [ ] 检查等待、运行、进度、完成、失败、取消和重试状态；反馈归属正确卡片/批次，旧结果不被覆盖。
- [ ] 制造一次可恢复错误并重试；其他分支和已有结果保持不变。
- [ ] 从结果创建继续分支并再次运行；旧结果保留，新分支可编辑、连接、保存和重开。

## 8. macOS 内屏、外屏与跨屏

分别以 English/简中在内建 Retina、外接屏执行代表旅程，并移动窗口内屏 → 外屏 → 内屏：

- [ ] 使用独立 PixelForge 项目窗口执行；若从 Godot 编辑器启动，先确认项目没有运行在嵌入式 Game 视图中。嵌入式视图结果不能替代本节。

- [ ] 顶栏、左侧栏、检查器、卡片、状态栏与主要弹窗没有重叠、截断或不可达按钮。
- [ ] UI 不随跨屏累计放大/缩小，画布相机、选择和 10/25/50/100/400% 档位保持稳定。
- [ ] 卡片文字和控件遵循统一 UI 缩放；画布美术保持最近邻与设备像素整数对齐。
- [ ] 50 张结果、检查器覆盖层、小地图与缩放控件在各屏幕上不遮挡关键操作。
- [ ] 拖动、缩放、连线、标题编辑、滚动画布和结果命中手感可接受。
