# Beta 0.6 — 卡片产品化与画布可读性执行书

> 状态：计划已冻结，尚未开发
>
> 前置：Beta 0.3–0.5 工程候选已完成，项目所有者认为当前产品仍然原始，未给人工 `go`
>
> 唯一视觉规格：[BETA-0.6-CARD-DESIGN-SPEC.md](BETA-0.6-CARD-DESIGN-SPEC.md)
>
> 执行方式：一个 Goal 连续完成 B6-0 至 B6-8；开发中不使用 Computer Use、不请求分段人工测试

## 1. 本轮只完成一件事

把已经能工作的无限画布原型，做成第一眼像产品、连续使用也说得通的卡片工作台。

用户从空画布加入提示词和参考图，整理风格与尺寸，运行云端生成，在原位看见状态和全部结果，筛选后继续加工或下一轮生成。整个过程不应依赖右侧检查器猜参数，也不应在按钮堆、默认 Godot 控件和摘要卡之间来回找入口。

本轮不是继续堆功能。已有 Graph、Provider、运行、结果、处理、模板、保存和撤销能力应当保留，Beta 0.6 负责把它们重新组织成清楚的产品表面。

## 2. 硬性产品决定

下面已经由项目所有者选定，执行 agent 不得自行换方案：

1. **混合卡片语言**：图片/结果采用 LibTV 式内容优先，输入/生成采用 Infinite-Canvas 式结构化分组；PixelForge 使用独立 token 和实现。
2. **标题可改**：所有正式 graph 卡、兼容旧 batch 卡和独立 sprite 内容卡支持自定义显示标题；标题只影响画布显示，不改 Graph 和执行。
3. **卡片自由缩放**：展开卡宽高可分别调整，正文重新排版；不使用整体拉伸。
4. **结果全部展开**：结果卡不分页、不内滚动、不显示 `+N`；All 筛选下 50 张必须全部存在并可操作。
5. **结果卡双尺寸**：用户保存请求尺寸；有效高度至少容纳全部当前结果。用户可以缩小偏好，不能把内容缩没。
6. **语义缩放不改几何**：LOD 只改变信息详略，不能改变世界边界、端口或连线。
7. **检查器退居次要**：默认关闭；卡片正面必须完成理解和主操作。
8. **云端模型为唯一生成路线**：不恢复本地模型和 ComfyUI。

如执行中发现这些决定与现有契约冲突，先以本计划已经批准的 [PROJECT-FORMAT](../02-contracts/PROJECT-FORMAT.md) 最小补全为准；不得另造第二套字段或把显示数据塞进 graph params。

## 3. 必做和明确延后

### 必做

- 工作区顶栏、左侧主入口、检查器抽屉、空画布入口和选中工具条重排；
- 集中卡片视觉 token、共享卡片外壳和统一状态；
- 八类 graph 卡与独立 sprite 内容卡的内容层次和视觉重做；
- graph 卡 `display_title / size` 的项目/剪贴板/模板/Undo 完整闭环；独立 sprite 完成项目/复制/Undo 闭环但不进入 graph 工作流模板；
- 结果卡全结果自动增长、Focus/Compare 下的完整候选网格；
- frame、端口、连线和 10/25/50/100/400% 语义缩放收口；
- English/简中、三种窗口和三种 UI scale 的自动化几何矩阵；
- 固定截图、总验证脚本、唯一 macOS 候选、工程报告和最终人工清单。

### 明确延后

- 新卡片、新模型、新 Provider 或新图像算法；
- 自动排布、对齐辅助线、嵌套/折叠 frame、画布搜索和书签；
- 账号、权限、协作、共享、在线发布和版本历史；
- 本地模型、ComfyUI、视频、音频和时间线；
- 通用 UI 框架重写、万级节点优化、跨平台打包完善；
- 主题切换、用户自定义主题、复杂动效和高成本动态模板预览；
- 为了代码整洁而改动与本轮旅程无关的服务。

## 4. 依赖和执行顺序

```text
B6-0 基线、契约和测试骨架
   └─> B6-1 共用卡片外壳、改名、缩放与持久化
          ├─> B6-2 工作区壳和响应式入口
          ├─> B6-3 结构化输入卡
          └─> B6-4 图片与参考卡
                  └─> B6-5 生成卡和任务状态
```

B6-2、B6-3、B6-4 在 B6-1 稳定后可由独立 worktree 并行；主 agent 必须逐个审核并合流。若只有一个执行 agent，则按编号顺序完成，不为了并行临时拆第二套主题或卡片外壳。

### B6-1 共用卡片外壳、改名、缩放与持久化

**服务旅程**：所有卡片先获得同一种可读、可选、可改名、可调整大小的产品外壳，后续正文不再各自发明样式。

**交付**：

- 在现有 `app_theme.gd` 基础上建立卡片集中 token；颜色、间距、圆角、字号、边框和状态只能从该入口读取；
- 按视觉规格 §4.4 建立最小原创单色 SVG/代码图标集；禁止继续用 emoji 或特殊 Unicode 当图标，也不引入第三方图标包；
- 抽出共用职责：Header、标题编辑、状态位、折叠、More、选中/焦点、缩放手柄和请求/有效尺寸计算；
- `canvas_node_card.gd` 与 `canvas_batch_card.gd` 保持协调职责；因本轮新增而拆出的正文/网格/几何逻辑按职责成文件，禁止继续把两文件无限扩长，也禁止为了行数拆成无意义包装层；
- 实现视觉规格 §6 的结构卡和内容卡外壳；正文组件不得各自重画一套标题栏；
- 实现 `display_title`：双击/菜单进入，Enter/失焦提交，Esc 取消，空白清除，最多 80 个 Unicode code point、换行/Tab 处理、默认标题本地化；
- 实现 `size`：右下手柄、宽高独立变化、最小/最大钳制、双击复位、Esc 取消；使用内容重新布局，禁止 `Node2D.scale` 拉伸；
- 一次 rename 或一次 resize drag 各只有一条 Undo；编辑期间画布快捷键隔离；
- 折叠高度固定 56，折叠不覆盖展开请求高度；折叠时只允许改宽；
- ProjectService 保存/打开兼容旧项目：缺字段用默认；形态错误用安全默认并规范化；未知字段继续往返；format_version 仍为 1；
- Graph clipboard 对 node 保留 `display_title / size / collapsed`，粘贴只重映射 ID 和位置；独立 sprite 的现有 Duplicate/Undo 保留标题、尺寸和 `scale_factor`；
- Workflow template v1 捕获、校验、插入三个可选字段；内置模板显式写尺寸；非法标题/尺寸 fail-closed；
- 改名、缩放、折叠前后 Graph JSON 必须字节语义等价，不得用 graph params 存视觉状态。

**完成条件**：任一正式卡片都可改名和缩放；保存重开、复制粘贴、模板插入和 Undo 后一致；旧项目无字段仍可打开；卡片 token 没有第二份。

**自动化**：八类 graph 卡与独立 sprite 的默认/最小/最大值、标题清理和语言切换、75/100/200% 拖动换算、一次 Undo、Esc、折叠恢复、Graph 不变、项目往返、复制/模板旧版兼容。

**集成边界**：B6-1 完成后跑一次全量测试；不构建候选、不请求人工。

### B6-3 结构化输入卡

**服务旅程**：用户不用打开检查器，就能读懂并填写生成所需的文本、对象、风格和尺寸。

**交付**：

- `size_spec`：`宽 × 高 px` 主视觉、数值输入和 16/24/32/48/64 快捷预设；

- 删除原始节点 ID、重复字段标签和无意义 Apply 按钮；提交语义按视觉规格执行；

**完成条件**：四卡在默认/最小/放大尺寸下都能直接编辑；中文与 English 不挤压；输入错误贴近字段并告诉用户下一步。

**自动化**：参数提交、焦点与快捷键隔离、Undo、50 行、长标题/长模型文案几何、最小尺寸重排、旧 Graph 行为回归。

### B6-4 图片与参考卡

**服务旅程**：参考内容成为画布上的视觉主体，用户看图就能整理，而不是先读 asset ID。

**交付**：

- `image_input` 与独立 `sprite` 改为浅色 32 高标题轨 + 大图主体 + 简短元数据；最近邻、棋盘格、完整显示、无拉伸；
- 默认标题回退素材名；正面隐藏 asset ID；详情仍可在检查器找到；
- 缺失、解码失败、空输入分别给出不同图标、原因和修复动作；
- Replace 保持节点/连线并进入一次 Undo；Remove 遵守现有引用规则；
- `reference_set` 改为缩略图网格、顺序号、Add tile、Replace/Remove 和拖动排序；格子最小 96；
- Provider 参考数量上限从现有模型能力读取并显示，不在 UI 写死第二套模型表；
- sprite 继续是非 Graph 图片卡，不强制转换为 graph 节点；缺少 `size` 的旧数据按 PROJECT-FORMAT 的图片尺寸公式推导，保留 `scale_factor`，保存后写显式请求尺寸。

**完成条件**：image_input 和独立 sprite 都可改名、自由缩放且图片占主要面积；多参考顺序一眼可见；缺失素材不会伪装成功，也不会破坏保存。

**自动化**：像素过滤、preview bounds、透明棋盘、替换/顺序/删除 Undo、缺失与坏素材、参考上限、标题和尺寸保存重开。

**集成边界**：B6-2 至 B6-4 合流后跑一次全量测试和一次临时脚本截图采证；截图只用于发现结构错误，不写人工结论。

### B6-5 生成卡和任务状态

**服务旅程**：用户提交前知道会用什么、生成多少、可能花多少；提交后在原位置看见排队、运行、取消、失败和重试。

**交付**：

- 按视觉规格 §7.7 固定三段内容：模型与能力、输入摘要、请求/费用；Advanced 默认收起；
- 卡底只保留一个主按钮，根据状态变为修复输入/生成/取消/重试/再次生成；
- 完整覆盖 Incomplete、Ready、Queued、Running、Canceling、Complete、Partial、Failed、Canceled；
- 每个状态明确图标、短文字、按钮是否启用和下一步；不整卡染红；
- 没有真实进度时使用不确定动画，不伪造百分比；
- 模型特有控件继续来自 Provider model descriptor；本轮不新增 Provider 能力；
- 费用未知时明确写未知，不显示 `$0`；
- 失败原因经过现有 presenter/i18n 转成短说明，日志不泄漏完整响应和密钥。

**完成条件**：所有任务状态在卡片正面可理解；用户不打开检查器也能运行、取消、修复和重试。

**自动化**：录制 Provider 响应、无费用、部分失败、取消竞态、重复点击、Retry、i18n、主按钮唯一性和状态几何。

### B6-6 结果卡全结果展开

**服务旅程**：生成完成后，用户直接在画布上审阅全部候选并继续处理，不被分页、隐藏结果和菜单迷路打断。

**交付**：

- 按视觉规格 §7.8 实现 Header、Action row、128 方格、角标、选择、双击编辑和高频动作；
- Contact 下铺完整网格；Focus/Compare 的主预览下方仍铺完整网格；删除现有只显示前若干项的限制；
- 严格使用公式计算列数、行数、内容需要高度和有效高度；默认 600 宽为 4 列；
- `0/1/4/5/12/13/50` 张都创建真实 slot；All 下 slot 数等于全部实际结果，运行中至少等于 expected_count；
- 不允许 `ScrollContainer`、页码、Next/Previous、`+N` 或只绘制视口前 N 张；视口剔除可以跳过屏外绘制工作，但数据、几何和命中槽必须完整；
- 用户缩窄时列数减少且卡变高；加宽时卡可变矮；请求高度始终是下限；
- 50 个 placeholder 逐步替换成 50 个真实结果时外框不跳；
- 最后一个结果仍可选择、Keep/Reject/Flag、Focus、Continue、Process 和 Export；
- 筛选造成的隐藏必须显示 `可见/总数` 和 Clear filter；All 下不能隐藏；
- 选框、命中、视口剔除、小地图、端口、Fit All 使用有效尺寸；自动增高不改 frame。

**完成条件**：600 宽、50 张时为 4 列 13 行，第 50 张有可见矩形和命中区域；无分页、无内滚动、无尾部裁切；保存重开后顺序和审核状态不变。

**自动化**：公式边界、不同宽度、结果增减、placeholder 替换、Focus/Compare、筛选、最后一项动作、有效 bounds、项目往返和 Undo。

### B6-7 frame、连线与语义缩放

**服务旅程**：用户远看能理解项目结构，近看能编辑内容，放大能检查像素；任何倍率切换都不会让节点和线跳位。

**交付**：

- 实现视觉规格 §8–§11 的状态、frame、端口、连线和五档 LOD；
- 10% 只保留结构色块和状态，25% 保留类型/frame 标题，50% 保留可读摘要，75%–300% 完整编辑，400%+ 增加像素检查信息；
- 删除 overview 改写卡片为 `220×116` 的旧逻辑；所有 LOD 使用同一世界 bounds；
- 端口可见 12、屏幕命中 40；低于 75% 隐藏且不能拉线；
- 兼容/不兼容连接同时有颜色、图标/文字原因；临时线、选中相关线和无关线按规格显示；
- frame 使用已有 title/size 实现改名、缩放和 Undo：标题复用 80 code point 单行规则，尺寸 `320×240` 至 `32768×32768`，Esc 取消且一次拖动只进一条 Undo；不提供双击尺寸复位，不新增 frame 锁定/自动布局/嵌套/折叠；
- 低倍率双击卡片回到 100% 并居中；双击 frame 适配该 frame；
- 50 张自动增高后，端口、小地图、Fit All、选框和连线同帧更新。

## 6. 固定自动化矩阵

### 6.1 工作区几何

参数化运行：

- locale：`en`、`zh_CN`；
- logical window：`1080×560`、`1280×720`、`1440×900`；
- UI scale：`1.0`、`1.25`、`1.5`。

共 18 组。每组必须断言：

1. 所有可见控件在父内容区内，容差不超过 0.5 logical px；
2. 顶栏按钮不重叠；每个 action id 在按钮与菜单中恰好一次；
3. File、Add/Import、Run、Export 和检查器开关始终可达；
4. 标题、状态、折叠和 More 之间至少 8；长文字省略/换行规则生效；
5. 文本高度包含真实字体高度与样式 margin，没有底部裁切；
6. 抽屉 dock/overlay/closed 几何正确；开关不改变 camera；
7. 八类 graph 卡与独立 sprite 的默认、最小和放大尺寸不越界，输入控件不互相覆盖；
8. 加载 zoom=0.5 后 canvas/index/slider/label 同帧为 50%；UI scale 不参与相机百分比；
9. 10/25/50/100/400% 切换只改变可见内容，不改变 bounds/端口/线；
10. 结果卡 `0/1/4/5/12/13/50` 的 slot 数、行列、最后项命中和有效高度正确。

### 6.2 改名与缩放

- 75%/100%/200% 相机下拖动同一屏幕距离，换算出的世界尺寸正确；
- 宽高独立，不锁比例；小于最小/大于最大精确钳制；
- 锁定、折叠纵向、10/25/50% 时手柄不能启动；
- 多次 mouse motion 只产生一条 Undo；Esc 无 Undo；Redo 精确恢复；
- 自定义标题语言切换不变，清空后默认标题随语言切换；
- 标题/尺寸保存、复制、模板和旧项目兼容；Graph 不变。

### 6.3 结果自动增长

- 默认宽 600：0/1/4/12/13/50 对应 0/1/1/3/4/13 行；
- 宽度 360/600/880/1600 的列数、高度和边界按同一公式；
- 五列临界宽 `W5 = 2×16 + 5×128 + 4×12 = 720`；719 仍为 4 列，达到 720 才变 5 列，避免临界抖动；
- 请求高度低于内容需要时不裁切；结果减少后只收回自动增长部分；
- 50 placeholders → 50 assets 外框不跳；索引 49 全部动作有效；
- 节点树不存在结果区 ScrollContainer 和分页按钮。

## 8. 回归与候选阻断条件

以下任一项出现，Beta 0.6 不得构建候选：

- 1080/1280/1440 任一 English/简中矩阵存在重叠、底部截断或关键 action 不可达；
- All 筛选下 50 张有任一结果未创建、未绘制、未命中或被分页/滚动隐藏；
- 项目加载 50% 而 UI 显示 100%；
- LOD 使卡片、端口或连线改变世界几何；
- rename/resize 修改 Graph，或保存/复制/模板丢失标题和请求尺寸；
- 旧项目、坏素材降级、Provider 录制、Undo、autosave、处理或导出旅程回归；
- 自动化红灯、lint/i18n/scaling/export 守护失败、orphan/resource 提示增长；
- 引入外部受限代码/资产、受保护图片进入 staged，或日志泄漏密钥/素材/完整响应。

次要的阴影细腻度、过渡动画、更多主题、极端超长标题美观度和非目标平台打包可以进入 Beta 0.7 候选池，但不能用来替代上述硬门。
