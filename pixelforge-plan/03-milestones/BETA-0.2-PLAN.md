# Beta 0.2 — 内容模块工作台与真实可用性重构

> 计划日期：2026-07-12
> 前置基线：[Beta 0.1 既有工程基线](BETA-0.1-BASELINE.md)
> 产品参照：[LibTV 无限画布产品证据](../04-research/LIBTV-INFINITE-CANVAS-EVIDENCE.md)
> 当前状态：**待开发**

## 1. 本轮目标

Beta 0.2 不再扩张功能清单，而是把 Beta 0.1 已有能力重新组织成一个实际可理解、可操作的产品：

1. 无限画布上的模块能直接容纳和编辑真实内容，而不是只显示流程摘要；
2. 用户能在同一画布完成“输入 → 生成 → 批次审阅 → 加工/导出”的最小闭环；
3. 支持 English 与简体中文，在中文 macOS 上可直接使用；
4. 对弹窗重叠、字体截断和不同显示器缩放建立真实根因与回归证据；
5. 完成一个整体切片后再统一冒烟，Beta 0.2 候选收口后由项目所有者一次人工验收。

本轮的成功标准不是功能数量，而是首次打开后能看懂画布上的内容、知道从哪里开始，并能无 API key 跑通一个可保存重开的完整示例。

## 2. Beta 0.2 验收旅程

> 中文系统首次启动 PixelForge，界面自动使用简体中文，也可切换 English。用户新建项目或打开示例工作区，在画布中直接看到“对象/提示词”“参考图”“生成”“结果批次”这些有真实内容的模块。他在输入模块中填写三种像素道具，导入一张参考图，确认生成设置并运行离线 mock；进度和结果原位出现，候选图直接装入批次模块。他筛选候选，对选中项执行一次清洗或描边，保存、关闭、重新打开，内容、位置、连线、状态和结果都保持一致，最后导出并看见产物位置。

使用真实 Provider 是同一路径的增强项，不是该旅程的启动前提。

## 3. 本轮必要范围

### 必做

- 工作区信息架构和首屏空状态；
- 可承载真实内容的输入、参考、生成和批次模块；
- 模块内主要编辑/运行/状态反馈；
- 现有图关系、Undo/Redo、项目保存重开与离线执行的应用整合；
- 简体中文/English、语言选择与持久化、可再分发 CJK 字体；
- macOS 弹窗/字体/缩放独立高强度诊断与修复；
- 阶段级自动化、两次整体验证和一次最终人工验收包。

### 明确不做

- 新 Provider、新像素算法、M8 体素、多人协作、云同步；
- LibTV 的视频/音频/脚本生态、Agent 自动制片、模板市场；
- 全量节点类型动物园、复杂调度、通用嵌套组；
- 签名公证、自动更新、遥测、正式发布站；
- 为追求外观相似而替换现有无限画布引擎；
- 未经新设计验证重做已被否决的 UX-4 overview/LOD。

这些项目只有在 Beta 0.2 统一验收后，才可从 Beta 0.3 候选池选择。

## 5. 执行卡

### B2-0 冻结产品壳与验收图

**目标**：在写实现前把“模块里装什么、用户如何走完旅程”变成唯一验收依据。

**产出**：

- 工作区区域：紧凑全局动作、画布内添加入口、主画布、上下文检查器、导航/状态区；
- 四类模块的内容层级、默认/选中/运行/失败/完成状态；
- 一个随包离线示例工作区和空白项目的首次操作路径；
- 复核 `GRAPH-SCHEMA`/`PROJECT-FORMAT` 能否承载所需数据，只在确有缺口时提交最小契约修订。

**出口**：实现者无需自行猜测模块职责；任何高级功能都有明确延期位置。本卡不做 UI 冒烟或人工验收。

### B2-1 搭建工作区壳

**目标**：先形成像样、清晰的无限画布应用外壳，不在此卡重写后端能力。

**必要结果**：

- 项目/保存/导出等全局动作与画布创作动作分层，不再把所有功能等权塞满顶栏；
- 空画布提供“添加输入、导入参考图、打开示例”的明确起点；
- 画布内有统一的添加入口，Tab/右键快捷入口可继续复用；
- 右侧区域根据选中内容展示上下文，高级参数可放这里，但核心内容不从卡片中消失；
- 导航区保留缩放、聚焦选中/全部等基本能力。高级 minimap、语义 LOD 不作为本卡出口。

**约束**：壳使用真实项目数据和现有服务，不允许做一套无法接入现有流程的静态 mock UI。

**自动门**：工作区实例化、关键入口可达、空状态随项目状态切换、现有项目动作无回归。完成后继续 B2-2，不单独冒烟或请求用户测试。

### B2-2 把轻节点升级为内容模块

**目标**：模块本身能装下并操作用户关心的东西。

| 模块 | 卡内必须可见/可操作 | 复用的真实数据 |
|---|---|---|
| 对象/提示词输入 | 多行内容、增删/编辑、条目数 | `object_list` 参数 |
| 参考图输入 | 真实预览、名称/来源、替换入口 | `image_input` + AssetLibrary |
| 生成 | Provider/风格/尺寸的核心摘要、运行/取消、进度、错误与费用提示 | `ai_generate`、Provider/Task/Cost 服务 |
| 结果批次 | 真实缩略图、选择、keep/reject/flag、focus/compare、批量动作 | 既有 `batch` 内容卡 |

风格与尺寸可以使用独立模块，也可作为生成模块的就地摘要；必须继续遵守现有 Graph 契约，不能在 UI 中造第二份逻辑真相。

**共同行为**：

- 默认尺寸能看懂核心内容；需要时可展开/折叠，折叠不丢状态；
- 卡内输入、按钮和缩略图操作不会误触发拖卡、连线或画布平移；
- 位置、内容、展开状态和引用按契约保存重开；
- 逻辑节点与画布视图的增删、移动和编辑保持原子 Undo/Redo；
- 低频高级参数仍可进入检查器，不强迫首版把所有字段塞进卡内。

**自动门**：每类模块的数据映射、编辑、输入路由、保存重开、Undo/Redo 和无资源/失效资源状态。完成后继续 B2-3，不单独冒烟或请求用户测试。

### B2-3 跑通最小画布工作流

**目标**：把四类模块串成真实纵向闭环，而不是四个孤立组件。

**必要结果**：

1. 从空画布或示例创建一条有效链；
2. 运行前能看见缺失输入，错误在对应模块原位显示；
3. 离线 mock 无 key 可运行，结果进入批次；真实 Provider 继续沿用相同结果落点；
4. 批次选中项能调用既有清洗/描边等一个代表性加工动作并得到新结果；
5. 保存重开后内容、关系、状态、结果与 provenance 一致；
6. 失败、取消与重跑不会覆盖原图或破坏画布。

**整体切片 A 验证**：B2-1～B2-3 完成后才统一运行一次真实窗口 agent 冒烟，覆盖空白起步和示例起步两条路径；同时运行全量自动化。结果写入单一 Beta 0.2 报告，不逐模块生成体验报告，也不请求项目所有者测试。

### B2-4 English / 简体中文

**目标**：中文不只是翻译几项菜单，而是可完成整条 Beta 0.2 旅程。

**必要结果**：

- 语言默认跟随系统；中文系统使用简体中文，用户可在设置中切换 English/简体中文并在重启后保持；
- 用户可见字符串迁入稳定 key 与翻译目录，迁移期允许 `strings.gd` 作为兼容访问层；
- 覆盖工作区、菜单、检查器、弹窗、状态/错误、节点名称/参数/枚举和 onboarding；Provider/模型品牌名与用户数据不强制翻译；
- 随包提供许可证清晰、覆盖简体中文的 UI 字体和 fallback 规则；最终字体度量由高强度诊断线验证；
- 中文文件名、项目名、提示词和导出路径 round-trip 不损坏。

**自动门**：翻译 key 完整性、两种语言格式占位符一致、语言切换/持久化、中文路径 round-trip、代表长文布局压力。完成后交给 B2-D 在真实显示矩阵收口，不单独请求人工测试。

### B2-D UI/字体/显示器专项

完整任务见 [Beta 0.2-D 诊断卡](BETA-0.2-UI-DIAGNOSTICS.md)。该卡先采证，再在稳定壳与中文资源上做单变量实验和最小修复；不得由常规模型预写具体解决办法。

### B2-5 合流与 Beta 0.2 候选

**目标**：把产品主线和显示诊断线合成一个可供项目所有者统一验收的候选。

**工程门**：

- 定向测试 + 全量 GUT + lint/format + UI scaling 守护 + `git diff --check`；
- 中英文离线完整旅程；
- macOS 编辑器 Debug 与候选包、内屏/外屏/跨屏矩阵；
- 项目保存重开、autosave 恢复、Undo/Redo、导出；
- 受保护图片/PCK 路径、凭据与日志安全审计；
- 候选版本标记为 `0.2.0-beta.1` 或后续递增候选，不宣称发布。

**整体切片 B 冒烟**：由 agent 使用真实窗口一次完成中英文两条主旅程和代表异常恢复，形成截图/日志/已知问题证据。

**人工验收包**：只提供一份 `manual-test-beta0.2.md`，覆盖：

1. 中文首次启动与语言切换；
2. 空白画布搭建并运行最小链；
3. 模块内编辑、画布导航和批次审阅；
4. 保存重开、autosave 恢复和导出；
5. 内屏、外屏和跨屏移动的弹窗/字体/缩放观察。

项目所有者统一执行后决定：`go`、在本地分支修复后复验，或整个未推送 Beta 0.2 不合并。只有明确允许后才合并 `main` 和 push。

## 6. 状态与报告规则

- 每张卡可有本地可回退提交；同一集成切片连续开发，不因小模块暂停。
- 完成报告只建一份 `reports/BETA_0_2_ENGINEERING_REPORT.md`，按卡追加 diff、测试与失败记录。
- `CURRENT-STATE.md` 只在整体切片、专项根因或 go/no-go 发生变化时更新。
- 工程通过、agent 冒烟通过、项目所有者人工通过和发布通过四者禁止混写。

## 7. Beta 0.2 出口定义

Beta 0.2 可称“人工通过”的最低条件：项目所有者能在自己的 macOS 环境中，用中文独立完成验收旅程；内容模块清楚可用；代表弹窗和文字没有重叠/截断；跨屏缩放不破坏 UI 或像素画；没有数据丢失、密钥泄漏或核心流程不可达。

未满足时保留为 `Beta 0.2 工程候选`，不得靠延期登记升级版本状态。
