## 0. 如何使用本指导

## 1. 护栏（必须遵守，违反即返工）

- **core 算法与图模型本轮不动**（画布/节点模型已于 2026-06-16 锁定为"统一画布自绘轻节点 + 批次内容节点"）。本轮只动 UI 表现层、缩放/布局、批次卡渲染、交互细节；唯一可碰的 core 是 outliner 的前景阈值（P1-1，且加测试守护）。
- **所有新增/改动 UI 尺寸、字号必须经 `_scaled_int()` / `ui_scale`**（M1.1 复盘血泪：硬编码像素值会绕过缩放体系，在 Retina 上显示成一半）。
- UI 字符串集中在 `ui/shell/strings.gd`；**禁止裸 `print`**（用 `log_util`）；`gdformat`/`gdlint` 零告警。
- **绝不 commit/push 图片资源**（`test picture/`、`pixel/tests/fixtures/real/` 是未授权 AI 图，已 gitignore）。
- 改契约前走"提议待批准"流程；本轮预计不动契约（批次卡 schema 已在 PROJECT-FORMAT 登记临时例外）。

## 2. 一个关键认知：多处"看不见"是同源的

用户 6 处"看不见 Apply / Report / Export 反应 / 状态栏"**不是独立 bug**，根在两处：

1. **缩放被钳到过小**：`fit_interface_scale_to_startup_screen` 用 `MIN_WINDOW(1280×800)` 反推，在 1470×956 可用区把 ui_scale 从应有 **2.0 钳成 ~1.09**。
2. **清洗台无滚动容器**：~20 个控件竖排，矮窗下底部 Apply/Cancel/Report 被裁且无法滚动。

**P0-1 + P0-2 一并修，会一次性解锁清洗台 13 项、批次 8 项、Export 2 项的复测。** 这是本轮的主线。

## 3. P0 · 阻断性（先做）

### P0-1　界面整体过小 + 缩放钳制反噬
- **现象（QA 0-4）**：mac 上整体界面偏小（字清晰但控件物理尺寸约一半）；调窗口大小时主工作区不随之缩放。
- **根因**：`ui/shell/main.gd`
  - `fit_interface_scale_to_startup_screen(scale, usable_size)`（~L140）：`fit = min((1470−80)/1280,(956−80)/800) ≈ 1.086`，把 `compute_auto_interface_scale` 得到的 2.0 钳到 1.086。
  - 即"最小窗口 1280×800（`MIN_WINDOW_*` L22-23）× ui_scale 必须塞进可用区"的约束，在小逻辑屏上逼着 ui_scale 变小。
- **改法方向**：
  - 让最小布局能更小（**依赖 P0-2 给清洗台加滚动**后，窗口最小高度需求大幅下降）→ 下调 `MIN_WINDOW_HEIGHT`（如 800→560）/`MIN_WINDOW_WIDTH`；
  - 或把"窗口能否装下"与"ui_scale 是否要钳"解耦：macOS Retina 优先**保 2.0**，靠响应式 + 滚动适配小屏，而不是缩小 ui_scale；
  - 复核 `_apply_window_defaults`（~L264）里 `usable_rect`(点) 与 `window.min_size`(像素) 的单位口径，避免再次 mac 点/像素混算。
- **涉及文件**：`ui/shell/main.gd`（缩放/窗口几族函数 L104-230、L264-320）。
- **验收**：1470×956 可用区下 ui_scale=2.0、控件物理尺寸正常、清晰；窗口拉伸时 canvas 跟随填充无固定小区域；保留既有 smoke 测（`test_main_window_ui.gd` 的内屏夹取/Retina 检测）不回归——**新增**一条"1470×956 点屏不被钳到 <2.0"的断言。
- **回归注意**：别让"内屏黑窗"复发（当初加钳制就是为此）。验证矮屏（如 1366×768）仍能正常开窗不黑屏（靠滚动 + 较小 min window，而非缩 ui_scale）。

### P0-2　清洗台 Apply/Cancel/Report 看不见
- **现象（QA 3-1/3-11/3-12/3-13，连带 3-2~3-5 全部 N.A.）**：选中图有反应，但找不到 Apply Cleanup、看不到 Report；调参也"无变化"（其实是够不到 Apply、也可能预览没注意）。
- **根因**：`ui/inspector/cleanup_inspector.gd` 的 `_build_ui` 把 ~20 个控件直接竖排进 PanelContainer 的 VBox，**无 `ScrollContainer`**；`main.gd` L362-367 又把它 `SIZE_EXPAND_FILL` 塞进 `HSplitContainer`。窗口不够高→底部 Apply/Cancel/Report 被裁、无法滚动。
- **改法方向**：把 inspector 内容 VBox 包进 `ScrollContainer`（纵向滚动、横向 fit）；确保 Apply/Cancel 始终可达（可考虑把 Apply/Cancel 固定在面板底部 sticky，Report 在滚动区内）。
- **涉及文件**：`ui/inspector/cleanup_inspector.gd`（`_build_ui`）。
- **验收**：任意窗口高度都能滚到并点到 Apply/Report；矮窗也能完整跑"选图→调参→预览→Apply→看 Report"。
- **联动**：本项落地后回填 P0-1（最小高度需求下降）。

### P0-3　批次卡过小 / 缩略失真 / 缩放到 1600%
- **现象（QA 8-5，连带 8-6~8-12 全部搁置）**：单图导入 25% 缩略能正常预览；但批次卡要放大到 ~1600% 才看清，且缩略默认是"小像素图"不是原图，clean batch 前后看不出区别。
- **根因**：
  - `ui/canvas/canvas_batch_card.gd`：卡是 world 空间 `Node2D`，`CARD_WIDTH=280`、`THUMB_SIZE=52`、`_rebuild_thumbnails` 把每张降采样到 52px **NEAREST** → 放大即大色块。
  - `ui/canvas/infinite_canvas.gd::_add_batch_card`（~L180）加到 `item_layer`（随 `camera_zoom` 缩放）；导入流程 `ui/shell/m2_1_ui_controller.gd::_import_image_files` **无自动取景**，用户只能手动滚轮放大（叠加 P0-4 过冲→1600%）。
- **改法方向**：
  - 缩略图提分辨率（存/画 128–256px，或按当前显示尺寸动态出图）；适当加大 `CARD_WIDTH`/`THUMB_SIZE`；
  - 卡片创建后**自动取景**到合适 zoom（fit 卡片到约 60% 视口），或把卡片改**屏幕空间固定尺寸**（不随 zoom 变大色块）；
  - 缩略图滤镜可保留 NEAREST（像素风），但源分辨率要够。
- **涉及文件**：`ui/canvas/canvas_batch_card.gd`、`ui/canvas/infinite_canvas.gd`（`_add_batch_card`、可加 fit 取景）、`ui/shell/m2_1_ui_controller.gd`（导入后取景）。
- **验收**：导入多图后默认就能看清每张内容；clean/matte batch 前后肉眼可辨；缩放卡片不再退化为糊块。

### P0-4　滚轮缩放过冲、无中间档
- **现象（QA 9-3）**：mac 滚轮缩放倍率跳太大、无中间态（疑与 mousefix 等高分滚动软件叠加；其他软件也有类似问题；Windows 未测）。
- **根因**：`ui/canvas/infinite_canvas.gd`
  - `ZOOM_LEVELS`（L13）是 ×2 档（0.125…32），无 1.5x 等中间档；
  - `_handle_mouse_button`（~L543）每个 `MOUSE_BUTTON_WHEEL_*` 事件调一次 `zoom_by_steps(±1)`（~L354）；
  - trackpad/高分鼠标/mousefix 一次手势触发多次 WHEEL → 飞档；**未用 `event.factor`、未累计 delta、未限频**。
- **改法方向**：
  - 累计 wheel delta 或用 `InputEventMouseButton.factor`，跨阈值才进一档；或对 WHEEL 事件限频（同一帧/短时窗只进一档）；
  - 加密 `ZOOM_LEVELS` 中间档（如插入 1.5/3/6…）让停档更自然；
  - 与左下缩放滑条保持双向同步不回归。
- **涉及文件**：`ui/canvas/infinite_canvas.gd`、`ui/canvas/canvas_zoom_control.gd`（如改档位表需同步）。
- **验收**：mac 一格滚轮 ≈ 一档、可停在中间倍率；滑条与滚轮仍同步。
- **回归注意**：触控板双指缩放、`pan_gesture` 不受影响。

### P0-5　Export PNG 点击无反应
- **现象（QA 1-5/1-6）**：选中图（绿框）后点 Export PNG 无反应；多图同样。
- **根因（待确认）**：`ui/shell/main.gd::_export_selected_png`（~L780）逻辑本身正常（取选区→设 `current_file`→`_export_dialog.popup_centered_ratio`→`_export_png_path`→`Exporter.export_png`=`save_png` 无损）。疑点二选一：① `get_selected_sprite_snapshots()` 对当前选择返回空 → 走 `STATUS_EXPORT_EMPTY`（提示只在底部状态栏，可能被裁/没注意）；② 异常缩放下保存框未正常弹出。
- **改法方向**：修好 P0-1/P0-2 后**先复测**；确认单选 sprite 时 `get_selected_sprite_snapshots`（`infinite_canvas.gd`，要求 `CanvasItemSpriteScript`）非空；**为空/失败给显眼提示**（弹窗或顶部 toast，不只状态栏）。
- **涉及文件**：`ui/shell/main.gd`（导出族）、`ui/canvas/infinite_canvas.gd`（选区快照）。
- **验收**：选 1 张能弹保存框并导出 PNG；选 ≥2 张导出 spritesheet；为空时有明确提示。

## 4. P1 · 重要功能与质量

### P1-1　Outline 导出后色彩裂化/噪点
- **现象（QA 6-1）**：描边后导出，色彩出现裂化（banding）和噪点，尤其边缘。
- **根因（推断）**：`core/pixel/outliner.gd` `ALPHA_THRESHOLD=0.004`（L22）极低 → 把 AI 图抗锯齿/半透明边缘都判为前景 → 描边沿锯齿边界走；`colored` 模式 `_compute_stroke_color`（取邻色加深30%）逐像素取邻色 → 多色噪点；`blit_rect` 保留半透明边 → fringe。
- **改法方向**：
  - 提高/自适应前景 alpha 阈值，或描边前对 alpha 二值化；
  - 文档/交互提示"描边前建议先 matte/cleanup"；
  - `colored` 边色做平滑或取主色（减少逐像素跳色）。
  - **这是唯一允许动的 core**，须加回归测试守护。
- **涉及文件**：`core/pixel/outliner.gd`；测试 `tests/unit/`（新增）。
- **验收**：对干净（已 matte）素材描边导出无明显噪点；新增测试断言 input→output 颜色数/边缘 alpha 在合理范围。

### P1-2　首次启动引导不弹
- **现象（QA 0-1）**：编辑器中启动没有快速入门提示。
- **根因（疑）**：`ui/shell/m2_1_ui_controller.gd::show_onboarding_if_needed` **先把 `m2_1_onboarding_seen=true` 写入再 `call_deferred` 弹窗** → 一次弹失败即永久抑制；或此前测试已置 seen。`main.gd:76` 在 `_ready` 末尾调用。
- **改法方向**：把 `seen=true` 移到弹窗确认（关闭）后再写；编辑器与真实窗口都验证首启弹一次；加一个"重看引导"入口（如 File 菜单或帮助）。
- **涉及文件**：`ui/shell/m2_1_ui_controller.gd`、`ui/dialogs/onboarding.gd`。
- **验收**：清 profile（删 `m2_1_onboarding_seen`）后首启必弹一次；重看入口可用。

### P1-3　手动选区不进算法 + W/M/L 实测损坏（用户已降级）
- **现象**：我方评审发现 W/M/L 选区**从不进入** matte/slice/outline（`ui/shell/m2_action_controller.gd` 全程只用整张 `item["image"]`，零处读选区）；用户实测（QA 7-2/7-3/7-5/7-6/7-7）连选区都选不出、Esc/L/Ctrl+Z 异常。用户明确"没那么重要，后续再修"。
- **本轮决策（二选一，推荐 A）**：
  - **A（推荐，省力避坑）**：M2.2 先**隐藏/禁用 W/M/L 工具入口**（顶栏按钮 + 快捷键），状态栏/引导不再宣传，避免用户点了无效困惑。把完整修复列为独立后续任务（M2.3 或并入 M3 选区子任务）。
  - **B（一次做对）**：修好"能选区 + 喂给 matte/outline + 光标随工具 + 蚂蚁线画真实轮廓"。注：`outliner` 已支持 `mask` 入参（`core/pixel/outliner.gd` add_outline 的 `mask` 键），outline 接入无需改 core；matte 需要新增 mask 支持（属 core 改动，须评审）。
- **涉及文件**：`ui/shell/m2_1_ui_controller.gd`（入口）、`ui/shell/main.gd`（快捷键）、（若选 B）`ui/tools/*`、`m2_action_controller.gd`、`core/pixel/matting.gd`。
- **验收**：A=入口隐藏干净、无残留宣传；B=上述四点一次修齐并有测试。
