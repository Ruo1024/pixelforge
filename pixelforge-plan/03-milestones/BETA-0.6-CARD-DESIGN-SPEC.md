# Beta 0.6 — 卡片、画布与工作区视觉规格

> 状态：已冻结，供 Beta 0.6 执行 agent 直接实现
>
> 冻结日期：2026-07-14
>
> 产品选择：`1A` 混合卡片语言、`2B` 标题可改且卡片可自由缩放、`3C` 结果卡展示全部结果
>
> 数据契约：[PROJECT-FORMAT.md](../02-contracts/PROJECT-FORMAT.md) 与 [WORKFLOW-TEMPLATE.md](../02-contracts/WORKFLOW-TEMPLATE.md)

## 1. 本规格解决什么

Beta 0.3–0.5 已经把创建、连接、生成、结果回流、分组、模板、保存和撤销做成工程候选，但当前界面仍像“功能已经接上、产品外壳还没完成”的开发工具。

Beta 0.6 只解决这一层：让用户一眼看懂画布上是什么、现在能做什么、运行到哪一步、结果在哪里，以及下一步怎么继续。它不重写画布引擎、Graph、Provider 或像素算法。

完成后的直接体验应当是：

1. 空画布第一步清楚；
2. 输入、参考、风格、尺寸、生成和结果都在卡片正面可读、可操作；
3. 图片和结果成为视觉主体，表单不再压过内容；
4. 卡片可改名、可调整大小，保存、复制、模板插入后保持不变；
5. 结果卡无论有 1 张还是 50 张，都把当前筛选下的结果完整铺开；
6. 10%、50%、100% 和 400% 各有明确用途，缩放不改变卡片位置与连线；
7. English、简体中文和常见窗口尺寸下无文字重叠、底部截断或关键入口丢失。

## 2. 参考方式和原创边界

本轮复刻的是成熟产品的**信息组织和交互关系**，不是逐像素复制外观：

- 从 LibTV 采用“媒体和结果优先、标题栏克制、远看结构近看内容”的卡片思路；
- 从 `hero8152/Infinite-Canvas` 采用“结构化输入、参数分组、卡内主动作、状态和结果原位反馈”的卡片思路；
- PixelForge 独立定义颜色、间距、字体、图标、组件代码和工作流细节。

外部证据见 [LibTV 调研](../04-research/LIBTV-INFINITE-CANVAS-EVIDENCE.md) 与 [90% 差距评估](../04-research/INFINITE-CANVAS-90-PARITY-ASSESSMENT.md)。`hero8152/Infinite-Canvas` 只允许作为只读行为参考；禁止复制其代码、CSS、截图、资产、文案或工作流文件。Beta 0.6 已把产品决定写清，执行中不得重新浏览竞品来替代实现；只有本规格与现有代码出现无法判断的真实冲突时，才允许低上下文子 agent 核对单一行为点。

## 3. 本轮不做

- 不新增账号、权限、协作、共享、项目版本历史；
- 不恢复本地模型或 ComfyUI 主路径；
- 不新增视频、音频、时间轴、Agent 自动建图或模型市场；
- 不重写无限画布、Graph、Provider、TaskQueue、项目格式容器或 Undo 服务；
- 不新增卡片种类，不扩展生成能力；
- 不做自动排布、嵌套 frame、frame 自动吸附或万级节点优化；
- 不建设完整设计系统编辑器、主题市场或动画特效系统；
- 不以清理旧代码、追求统一架构或压文件行数为理由扩大重构；
- 不在开发中使用 Computer Use，也不请求项目所有者逐卡测试。

## 4. 整体视觉方向

### 4.1 两种卡片语言

采用用户选择的 `1A` 混合方案：

| 卡片语言 | 使用对象 | 视觉重点 |
|---|---|---|
| 结构卡 | 提示词、对象列表、风格、尺寸、参考集合、生成 | 清楚的标题、分组、摘要和一个主要动作 |
| 内容卡 | `image_input`、独立 `sprite`、结果批次 | 图片占主要面积，外壳退后，状态和操作贴近内容 |

两者共用同一套边框、选中态、端口、菜单、标题编辑和缩放手柄。不得把每种卡做成互不相关的独立小工具。

### 4.2 颜色 token

颜色只能从集中主题/卡片 token 访问，不得散落在各组件中：

| 名称 | 值 | 用途 |
|---|---|---|
| Canvas | `#0D121C` | 画布背景 |
| Grid | `#283246` | 网格，低对比显示 |
| Card | `#151C29` | 结构卡主体 |
| Elevated | `#1B2535` | 悬停、浮层、较高层级 |
| Section | `#101722` | 输入区和参数分组 |
| Border | `#2B3850` | 普通边框 |
| Border Hover | `#455773` | 悬停边框 |
| Text Primary | `#F2F5FA` | 标题和主要内容 |
| Text Secondary | `#AAB4C4` | 说明和次要信息 |
| Text Muted | `#7B879B` | 元数据和禁用说明 |
| Selection / Info | `#6FA8FF` | 选中、焦点、有效连接 |
| Success | `#45D6A3` | 成功、完成、保留 |
| Warning | `#F4B45F` | 等待、部分完成、需注意 |
| Error | `#F06B6B` | 失败、无效连接、拒绝 |
| Media Rail | `#DDE8F7` | 单图内容卡的浅色标题轨 |
| Media Rail Text | `#111827` | 浅色标题轨文字 |

正文文字与背景对比度至少 4.5:1；大图标、粗线和大字号至少 3:1。颜色不能是唯一状态信号，必须同时有图标或文字。

### 4.3 尺寸 token

以下全部是逻辑像素/画布世界尺寸，不是设备像素：

| 项目 | 固定值 |
|---|---:|
| 外层圆角 | 14 |
| 内部分组圆角 | 10 |
| 控件圆角 | 8 |
| 普通边框 | 1 |
| 选中/连接边框 | 2 |
| 结构卡 Header | 44 |
| 内容卡标题轨 | 32 |
| 卡片内边距 | 16 |
| 分组间距 | 12 |
| 行间距 | 8 |
| 标题字号 | 15 |
| 正文字号 | 14 |
| 状态/元数据字号 | 12 |
| 普通控件最小高度 | 32 |
| 主按钮高度 | 42 |
| 图标按钮点击区 | 32×32 |
| 可见端口直径 | 12 |
| 端口屏幕命中区 | 40×40 |
| 折叠高度 | 56 |

间距只使用 `4 / 8 / 12 / 16 / 24`。字体沿用项目现有 Noto Sans CJK；本轮不引入新字体资产。组件不得另设第二套 UI 缩放倍率，也不得用 `Node2D.scale` 拉伸文字和控件。

- 普通卡不加大面积软阴影，只靠背景和边框分层；Hover/Selected 可使用 `#00000055`、size 4、offset `(0, 2)` 的轻阴影；菜单/抽屉可用 size 12、offset `(0, 6)`。
- Hover、选中和按钮颜色过渡固定 100ms；抽屉开合 160ms；拖动、缩放、LOD 切换和结果自动增高立即生效，不做妨碍命中的补间动画。

### 4.4 图标

- 禁止用 emoji、特殊 Unicode 字符或字体里的偶然字形充当界面图标，避免 macOS/不同字体出现“奇怪符号”和基线截断。
- 本轮只使用 PixelForge 自行绘制的单色 SVG 或简单代码绘制图标；不得下载/临摹参考项目图标包。
- SVG 使用 24×24 viewBox、1.75 线宽、round cap/join；常态图形占 18×18，颜色由主题调制，不在每个 SVG 写多套状态色。
- 至少覆盖：File、Undo、Redo、Run、Export、Inspector、More、Add、Import、Library、Collapse、八类 graph 卡与独立 sprite 类型、Duplicate、Group、Delete、Lock、Replace、Link、Keep、Reject、Flag，以及 Incomplete/Ready/Queued/Running/Canceling/Complete/Partial/Failed/Canceled。
- 视觉尺寸 16–20，点击区仍为 32；每个图标必须有 tooltip、可访问名称和菜单文字兜底。
- 原创 SVG 在资产说明中标记为 PixelForge 原创；本轮不新增第三方图标许可证依赖。

卡片类型图标和 10%/25% 类型色固定如下，不由执行 agent 自选：

| 类型 | 图标几何 | 类型色 |
|---|---|---|
| `text_prompt` | 方框内三条左对齐文本线 | `#6FA8FF` |
| `object_list` | 三个勾选框 + 三条文本线 | `#6FA8FF` |
| `style_preset` | 四个并排色块 | `#B993FF` |
| `size_spec` | 两个相对直角 + 双向尺寸箭头 | `#5BC6C8` |
| `image_input` / `sprite` | 方框内山形与圆点 | `#F0B45F` |
| `reference_set` | 两个错位图片框 | `#F0B45F` |
| `ai_generate` | 四角星 + 小圆点 | `#C77DFF` |
| `batch` | 2×2 方格 | `#45D6A3` |
| 幽灵/未知 | 虚线方框 + 问号 | `#F06B6B` |

类型色只用于图标、3px 标记线和低倍率轮廓，不能铺满正文背景。状态图标也固定：Incomplete=空心圆、Ready=播放三角、Queued=时钟、Running=旋转圆弧、Canceling=停止方块+圆弧、Complete=勾、Partial=半实心圆+感叹号、Failed=感叹号三角、Canceled=停止方块。Keep/Reject/Flag 分别为勾、斜线圆、旗帜。

## 5. 工作区壳

### 5.1 固定布局

```text
┌──────────────────────── 顶栏 52 ─────────────────────────┐
│ File  项目名     Undo Redo         Run Selection Export  ◫ ⋯ │
├─48─┬────────────────────── 画布 ────────────────────┬──────┤
│ Add│                                                │检查器│
│Import      [选中对象的就近工具条]                    │抽屉  │
│Library                                             │360  │
│    [卡片 / frame / 连线 / 结果]                     │      │
│    [缩放]                                  [小地图] │      │
└────┴───────────────────────────────────────────────┴──────┘
```

### 5.2 顶栏

- 高度固定 52；控件垂直居中；左右内边距 12；相邻控件间距 8。
- 左侧顺序：`File`、项目标题、Undo、Redo。
- 右侧顺序：`Run Selection`、`Export`、检查器开关、`More`。
- 项目标题最小宽 120、最大宽 280；超长省略并提供完整 tooltip；未保存状态在标题后显示圆点，不改变标题宽度。
- `New / Open / Save / Save As / Quit` 只放入 File 菜单，不再横铺成一排文字按钮。
- 当前选择无可运行内容时，`Run Selection` 禁用并在 tooltip 解释；不得隐藏后让用户找不到入口。
- 低频全局动作进入 More；同一个 action id 在直接按钮和菜单中只能出现一次。
- 1080 宽仍放不下时，Undo/Redo 与检查器仅显示图标，Run/Export 保留文字；不能把 New/Open/Save 或 Run/Export 彻底藏掉。

### 5.3 左侧栏

- 宽 48；每个按钮 40×40；从上到下为 Add、Import、Library。
- Add 打开现有节点添加面板；Import 打开现有图片导入；Library 打开现有素材/风格/工作流资源抽屉。
- 只重排已有入口，不新建第二套素材库或工作流服务。
- 当前按钮用 Selection 蓝色 2px 左边标记，同时有 tooltip 和可访问名称。

### 5.4 检查器

- 宽 360；默认关闭，选择卡片不会强制弹出。
- 用户显式打开后：窗口宽度 `>=1440` 时靠右停靠；更窄时覆盖在画布右侧，不改变相机中心和缩放。
- 抽屉开关状态只保留在当前应用会话，不写进项目文件。
- 检查器只承载来源、完整参数、低频高级设置和详细错误；理解卡片与完成主要动作不能依赖它。
- 抽屉关闭后焦点回到原卡片；Esc 先关闭抽屉，再回到画布。

### 5.5 空画布和就近工具条

- 空画布中央显示一句短说明和三个动作：`从模板开始`、`导入图片`、`添加输入`。
- 三个动作在 English/简中均不换成两行；空间不足时改成纵向排列。
- 单选卡片时，在卡片上方显示 Duplicate、Group、Run（适用时）、More；多选时显示 Duplicate、Group、Run Selection、Delete。
- 就近工具条使用屏幕空间定位，不能跟随画布缩放变得过小；不得遮住卡片标题或端口。
- Delete 放 More 中并使用文字和图标；不把危险动作做成默认主按钮。

## 6. 共用卡片外壳

### 6.1 结构卡

```text
┌─ [类型图标] 可编辑标题      [状态] [折叠] [⋯] ─┐ 44
│ 简短摘要；只保留本卡最重要的信息                 │
│ ┌─ 分组 / 输入 / 预览 ───────────────────────┐ │
│ │                                            │ │
│ └────────────────────────────────────────────┘ │
│ [次要说明]                                      │
│ [唯一主要动作；仅需要主动作的卡片显示]           │
└─────────────────────────────────────────────────┘
```

- Header 左侧依次为 20×20 类型图标和标题；右侧依次为状态、折叠、More。
- 标题与右侧控件至少留 8；空间不足时标题省略，不得挤压状态和按钮。
- 正文按摘要、内容、反馈、动作排序。高级参数默认收起。
- 只有生成卡等真正需要提交动作的卡片使用全宽主按钮；普通编辑卡不堆 Apply 按钮。

### 6.2 内容卡

```text
┌─ 浅色标题轨：可编辑标题                    [⋯] ┐ 32
│                                                 │
│                 图片 / 像素内容                 │
│                                                 │
├─ 尺寸 · 来源 · 状态 ───────────────────────────┤
└─────────────────────────────────────────────────┘
```

- `image_input` 与独立 `sprite` 使用内容卡；图片区域至少占默认卡片高度的 65%。
- 标题轨固定 32，底部元数据栏固定 28；其余高度全部给图片区。
- 图片保持最近邻、完整显示、居中；透明区域使用棋盘格；空白不拉伸填满。
- 原始 asset ID 不在卡片正面显示，只在检查器/详情中出现。
- 内容卡仍使用统一选中框、焦点环、端口、标题编辑和缩放手柄。

### 6.3 标题编辑

- 双击标题或 `More → Rename` 进入单行编辑。
- Enter 或失焦提交；Esc 恢复原值且不进 Undo。
- 换行和 Tab 转为空格，去掉首尾空白，最多 80 个 Unicode code point；按 Godot `String.length()` 计数并在 code point 边界截断，不按 UTF-8 字节计数，组合符号分别计数。
- 提交全空白表示清除自定义标题，恢复本地化默认标题。
- 自定义标题是用户数据，不随语言切换；默认标题随 English/简中切换。
- 一次提交只产生一条 Undo；编辑期间 Space、Delete、字母快捷键不得传给画布。
- graph batch 默认标题回退顺序：自定义标题 → `params.label` → 本地化“结果”；`image_input` 和独立 `sprite` 为自定义标题 → 可解析素材名 → 本地化“图片”；其他节点为自定义标题 → 本地化节点类型。

### 6.4 自由缩放

- 选中或悬停卡片时，右下角显示 12×12 视觉手柄，实际屏幕命中区 32×32。
- 普通展开卡可独立改变宽、高，不锁比例；拖动时正文实时重新排版，文字和图像不拉伸。
- 鼠标多次移动只预览；释放时写入一条 Undo。Esc 取消并恢复原尺寸，不产生 Undo。
- 双击手柄恢复该类型默认尺寸，并产生一条 Undo。
- 用户请求尺寸钳制到本规格的最小值和 `1600×1200` 上限；batch 的内容自动高度不受 1200 上限限制。
- 折叠卡高度固定 56，仅允许横向改宽；展开恢复此前请求高度。
- 锁定、10%/25%/50% 语义缩放下不显示手柄，也不能启动缩放。
- 选框、命中、视口剔除、小地图、连线端口和 Fit All 必须使用当前有效尺寸。

### 6.5 默认与最小尺寸

| 卡片 | 默认尺寸 | 最小尺寸 |
|---|---:|---:|
| `text_prompt` | 360×300 | 320×240 |
| `object_list` | 400×520 | 360×360 |
| `style_preset` | 320×280 | 280×220 |
| `size_spec` | 320×260 | 280×220 |
| `image_input` | 320×380 | 280×300 |
| `sprite` 独立图片卡 | 按 `PROJECT-FORMAT §4.1` 的旧图片公式；不可解析时 320×380 | 200×188 |
| `reference_set` | 400×480 | 360×320 |
| `ai_generate` | 400×520 | 360×400 |
| graph `batch` / 旧 `batch_card` | 600×240 | 360×240 |
| 幽灵、未知和其他轻节点 | 320×180 | 240×144 |

数值是用户未保存尺寸时的唯一默认。不得由各组件再声明另一组魔法数字。

## 7. 每种卡片的固定内容

### 7.1 提示词 `text_prompt`

- Header：提示词类型、可编辑标题、有效/空白状态、折叠、More。
- 正文：至少可见 5 行的 TextEdit；空状态直接显示示例 placeholder；底部右侧显示字符数。
- 编辑期间保留草稿；`Cmd/Ctrl+Enter` 或失焦作为一次提交，Esc 还原。
- 有未提交草稿时显示“有未保存更改”，但不弹窗。
- 不在卡片正面显示节点 ID、端口类型名或重复的“Prompt”字段标签。

### 7.2 对象列表 `object_list`

- 列固定为：启用、对象/提示词、数量、行菜单。
- Header 下方显示 `已启用 12/50 · 预计 48 个结果`；数字由现有执行语义计算，不另造估算真相。
- 每行高度 36；数量输入宽 64；行菜单 32；提示词占剩余宽度。
- 底部固定 `添加一行`；多行粘贴放在次级展开区，默认收起。
- 50 行允许正文区域内部纵向滚动并显示滚动条；滚动只发生在列表区域。结果卡的“禁止内滚动”不扩展到长表单。
- 行错误贴在对应行下方，不把整张卡染红。

### 7.3 风格 `style_preset`

- 首屏显示预设名和 12 色以内的调色板条；更多颜色在 tooltip/检查器说明。
- 下一行显示基础尺寸、描边、抖动、透视四类短标签，例如 `32px · 1px outline · no dither · top-down`。
- 缺失或失效的自定义预设显示 Warning 和“选择替代风格”，不静默换预设。
- 不把完整 JSON、调色板 ID 或长提示词模板放在卡片正面。

### 7.4 尺寸 `size_spec`

- `32 × 32 px` 是正文最大、最先读到的信息。
- 宽、高使用并排数值输入；下面显示 16、24、32、48、64 的快捷预设。
- 每主体数量只在现有契约仍需兼容时以次要信息显示，不能与结构化行数量造成双重主入口。
- 无效尺寸在输入旁说明范围和修复动作。

### 7.5 单图输入 `image_input` 与独立 `sprite`

- 图片占主体，使用最近邻和棋盘格；不得只显示小缩略图和大段 ID。
- 标题默认使用素材名；底栏只显示 `宽×高 · imported/generated/edited · 状态`。
- More 包含 Replace、Reveal details、Remove；Replace 保持节点和连线，只替换素材参数并进入 Undo。
- 素材缺失保留卡片尺寸和来源说明，显示 `重新链接 / 从节点移除`，不得画成正常图片。
- 独立 `sprite` 使用同一内容卡外壳和改名/自由缩放；它没有 Graph 端口，More 保留现有 Duplicate、Batch selected、Remove。旧 `scale_factor` 只参与缺少 `size` 时的默认推导，之后原样保留但不再决定外框。

### 7.6 参考集合 `reference_set`

- 正文使用缩略图网格；每张图左上显示顺序号，选中时出现 Replace、Remove，拖动可调整顺序。
- 最后一格是明确的 Add tile；不得继续使用“ID + 上移 + 下移 + 删除”的文字按钮行。
- 网格根据卡宽自适应，每格最小 96；参考数超出正文高度时允许内部滚动。
- 卡片摘要显示 `3 references · order matters`；Provider 上限不足时直接说明允许数量。
- 顺序、替换、删除和新增继续使用现有 Graph 参数与 Undo，不另存视图副本。

### 7.7 生成 `ai_generate`

- Header 显示可编辑标题和 `Incomplete / Ready / Queued / Running / Canceling / Complete / Partial / Failed / Canceled` 之一。
- 第一分组只显示 Provider、模型和能力短标签；模型名超长省略并有 tooltip。
- 第二分组显示输入摘要：提示词/对象行数、参考图数、风格、尺寸、候选数。
- 第三分组显示已知费用和请求数；未知费用写“提交前无法估算”，不得显示 `$0` 冒充免费。
- Advanced 默认收起，放 seed 与模型特有的低频参数；只显示当前模型实际支持项。
- 卡底始终只有一个强主按钮：Incomplete=`修复输入`，Ready=`生成`，Queued/Running=`取消`，Canceling=`正在停止…` 且禁用，Partial=`重试失败项`，Failed=`重试`，Complete/Canceled=`再次生成`。
- 没有真实百分比时用不确定进度动画；禁止伪造百分比。
- 失败区显示短原因和下一步；完整 Provider 响应不直接展示或写日志。

### 7.8 结果批次 `batch`

- Header 显示可编辑标题、`已选/总数`、任务状态、折叠、More。
- Action row 在卡片正面：
  - 无选择：All / Pending / Keep / Reject / Flag 筛选，Process All，More；
  - 有选择：Keep、Reject、Flag、Process、Continue、Export、More；
  - 宽度不足时只保留最常用动作，其余进入 More，同一 action 不重复。
- 候选格固定 128×128，间距 12；图片完整显示、最近邻、棋盘格；顺序号、Keep/Reject/Flag、选中和失败状态使用角标/边框叠加，不增加格高。
- 单击选择；双击进入已有编辑器；hover 可显示少量快捷动作，但主要动作必须仍可从 Action row 找到。
- Contact 模式直接铺网格。Focus/Compare 模式先显示主预览，再在下方铺开完整网格；不得只保留前 7 张。
- 筛选可以隐藏不匹配结果，但 Header 必须显示 `当前可见/总数` 并提供 Clear filter；`All` 下任何实际结果都不能隐藏。
- 0 个结果显示明确空状态、来源和下一步；缺失素材保留格位并显示修复状态。

#### 全结果自动高度

用户保存的是“请求尺寸”，实际绘制使用“有效尺寸”：

```text
header = 44
padding = 16
thumbnail = 128
gap = 12
action_row = 40
focus_preview_height = clamp(round((width - 2*padding) * 9/16), 240, 480)

columns = max(1, floor((width - 2*padding + gap) / (thumbnail + gap)))
rows = ceil(slot_count / columns)
grid_height = rows*thumbnail + max(0, rows-1)*gap
focus_active = review_layout == "focus" and focus_asset_id resolves to a visible slot
action_y = header + padding
preview_y = action_y + action_row + gap
grid_y = preview_y + (focus_preview_height + gap if focus_active else 0)

required_content_height = max(
  240,
  grid_y + grid_height + padding
)

effective_width = requested_width
effective_height = max(requested_height, required_content_height)
```

- 默认宽 600 时为 4 列；50 张为 13 行，索引 0–49 全部真实创建、绘制并可命中。
- 任务未完成且筛选为 All 时，`slot_count=max(已有结果数, expected_count)`，空位显示占位格，避免 50 个占位变成真实结果时外框跳动。
- 垂直顺序固定为 Header → 16 padding → Action row → 12 gap → 可选 Focus/Compare 主预览 → 12 gap → 完整网格 → 16 padding。Action row 不得放在长网格底部。
- 有效 Focus/Compare 在网格上方增加一个高为 `focus_preview_height` 的主预览区；Compare 在同一区内并排显示 A/B，下面的完整网格不减少。focus 引用失效或被当前筛选隐藏时回退 Contact，并显示修复/清除筛选入口。
- 用户缩窄时列数减少、卡片自动变高；加宽时列数增加、自动增长部分可收回，但不得低于用户请求高度。
- 用户把请求高度拖到内容所需高度以下时，只改变最小高度偏好，不能裁切、分页或引入内部滚动。
- 保存、剪贴板和模板只记录请求尺寸，不记录派生高度。
- 结果变少或筛选改变时重算有效高度；自动重算不单独进入 Undo。
- 自动增长不得改变 frame 大小、frame 归属或相邻卡位置。

### 7.9 幽灵/未知卡

- 保留原尺寸、标题、端口位置和未知数据；不可执行。
- 正文只显示“缺少什么 / 数据仍保留 / 可以做什么”：安装来源（若可信）、移除卡片、查看详情。
- 不伪造正常内容，不静默替换节点类型。

## 8. 状态视觉

### 8.1 交互状态

| 状态 | 固定表现 |
|---|---|
| 默认 | Card 背景、1px Border |
| Hover | Elevated 背景、Border Hover；手柄和 More 可见 |
| Selected | 2px Selection 外框；就近工具条出现 |
| Keyboard focus | 选中框外再加 2px 高对比焦点环；不能只靠颜色 |
| Dragging | 卡片 92% 不透明，原位显示轻轮廓；不改变内容状态 |
| Resizing | 显示当前 `宽×高` 浮标，正文实时重排 |
| Renaming | 标题换成单行编辑框，其他画布快捷键暂停 |
| Collapsed | 保留标题、类型、数量/执行状态和端口，高度 56 |
| Locked | 显示锁图标；不能拖动、缩放或改名，仍可选中查看 |

### 8.2 数据与执行状态

| 状态 | 边框/图标 | 文字与动作 |
|---|---|---|
| Empty / Incomplete | 普通边框 + 空心圆 | 说明缺什么，主动作是补输入 |
| Ready | Info 图标 | 显示将运行的摘要，动作是生成/运行 |
| Queued | Warning 时钟 | 显示队列中，可取消 |
| Running | Info 旋转标记 | 显示当前步骤；无真实数值不显示百分比 |
| Canceling | Warning | 显示“正在停止”，禁用重复取消 |
| Complete | Success 勾 | 显示结果数量和继续动作 |
| Partial | Warning 三角 | 显示成功/失败数量，可只重试失败项 |
| Failed | Error 图标与 2px 局部边线 | 显示短原因、Retry/修复入口；不把整卡染红 |
| Canceled | Muted 停止图标 | 已有结果保留，可再次生成 |
| Missing / Decode failed | Warning/Error 文件图标 | 保留格位，提供重新链接或移除 |

### 8.3 候选状态

- Selected：Selection 2px 外框 + 左上勾选标记。
- Keep：Success 角标和文字 tooltip。
- Reject：Error 斜线角标，图片降低到 55% 不透明；仍可恢复。
- Flag：Warning 旗帜角标。
- Focus：外框加粗，并在主预览与网格中同步。
- Placeholder：棋盘/骨架，不显示假图或假完成状态。
- Compare：两项分别标 A/B，不能只用左右位置表达。
- Filtered empty：显示当前筛选无结果与 Clear filter，不冒充批次为空。

## 9. 端口与连线

- 输入端口在卡片左侧中线，输出端口在右侧中线；有效尺寸改变后重新计算。
- 可见端口直径 12；命中区按屏幕 40×40 保持，不随画布倍率缩小。
- 10%、25%、50% 不显示端口，也不能开始拉线；75% 及以上显示。
- 拉线时：临时线使用虚线；兼容目标以 Info 环标出；不兼容目标显示 Error 环和一句可理解原因。
- 普通线为 2px 低对比曲线；选中卡片相关线变亮；无关线降低到 25%–35% 不透明。
- 线的透明命中宽度 16 屏幕像素，视觉宽度仍为 2。
- `显示/隐藏连线` 继续保留；隐藏只影响显示，不改变 Graph。
- Resize、LOD、折叠和 batch 自动高度后，端点必须在同一帧更新，不允许线悬空一帧。

## 10. Frame

- Frame 使用低对比半透明底、1px 边框和 36 高标题轨；颜色来自现有 frame color，但正文透明度保持克制。
- 标题轨显示 frame 标题、成员数、汇总运行状态、Run、Save Workflow、More。
- 使用现有 `title` 和 `size` 支持改名、右下缩放和 Undo；不新增 frame schema。双击标题/More 进入编辑，复用卡片的单行清理、80 code point、Enter/失焦提交和 Esc 取消；空白标题回退本地化“阶段”。
- Frame 右下手柄复用卡片的屏幕命中、拖动预览、一次 Undo 和 Esc 取消；尺寸钳制到 `320×240` 至 `32768×32768`。Frame 没有类型默认尺寸，因此不提供双击手柄复位。
- 低倍率时 frame 是主要阅读单位；成员轮廓和运行/失败数量比单卡正文更重要。
- Frame 不自动吸附成员、不自动调整大小、不嵌套、不折叠，本轮也不新增锁定；移动穿过边界不改变 `frame_id`。

## 11. 语义缩放

PixelForge 使用现有离散倍率，分成五档：

| 相机倍率 | 模式 | 必须显示 | 必须隐藏 |
|---:|---|---|---|
| 10% | Map | frame 形状、卡片轮廓/类型色、结果马赛克、运行/失败标记 | 全部正文、文字、端口、按钮、手柄 |
| 25% | Overview | 48 world-px 的 frame 标题、卡片类型/状态图标、主要关系 | 卡片正文文字、表单、端口、菜单、缩放手柄 |
| 50% | Browse | 大号摘要标题、单行摘要、图片/结果预览、状态 | 编辑器、端口、复杂动作、缩放手柄 |
| 75%–300% | Edit | 完整正文、标题编辑、端口、操作、缩放手柄 | 像素检查专属信息 |
| 400%–3200% | Inspect | Edit 全部 + 图片尺寸、透明信息、空间允许时的像素网格 | 无额外隐藏 |

硬规则：

- LOD 只决定“画什么”，不能改变卡片世界尺寸、请求尺寸、有效尺寸、端口锚点、连线端点或 frame 归属。
- 删除现有 overview 把节点改成 `220×116` 的行为。
- 25% 的 frame 标题使用独立 48 world-px 样式；50% 的卡片摘要使用独立 24 world-px 样式。不得把 14px 正文直接缩到不可读后仍称为摘要。
- 10% 卡片轮廓用对应类型色 18% 透明填充；运行/失败状态点直径 64 world-px。25% 类型图标为 48×48 world-px，状态图标为 40×40 world-px。
- 50% 标题字号 24、行高 30，只显示一行；摘要字号 20、行高 26，只显示一行；都按实际宽度省略并提供在 75%+ 可见的完整 tooltip，不按字符数硬截断。
- 低倍率单击只选择；双击卡片居中并切到 100%，双击 frame 适配该 frame。
- 加载保存为 50% 的项目时，画布、倍率索引、滑块和标签必须同帧显示 `50%`，不得先显示 `100%`。

50% 的摘要内容固定如下：

| 卡片 | 一行摘要 |
|---|---|
| `text_prompt` | 第一条非空提示词 |
| `object_list` | 已启用行数 / 总行数 · 预计结果数 |
| `style_preset` | 预设名 · 基础尺寸 · 调色板色数 |
| `size_spec` | `宽 × 高 px` |
| `image_input` / `sprite` | 图片预览 + `宽×高` |
| `reference_set` | 缩略马赛克 + 参考图数 |
| `ai_generate` | Provider / 模型 · 当前任务状态 |
| `batch` | 结果马赛克 + 可见数/总数 · 审阅状态 |
| 幽灵/未知 | 缺失类型名 · 不可执行 |

## 12. 本地化、可读性和键盘

- 所有用户文字进入集中 i18n；组件中不得新增 English 裸字符串。
- 自定义卡片/frame 标题是用户数据，不翻译；默认类型标题和系统状态必须翻译。
- 单行标题、模型名和项目名使用省略 + tooltip；说明文字最多两行，超出进入详情/检查器。
- 所有文字控件最小高度必须包含字体真实高度和上下 margin；禁止依赖 `clip_contents` 掩盖底部截断。
- Tab 顺序：标题/正文 → 次要控件 → 主动作 → More；焦点不得跳到不可见 LOD 内容。
- Enter 提交标题；Esc 取消当前编辑/缩放；Space 只在非文本焦点时平移画布。
- `Cmd/Ctrl+C/V/D/Z/Shift+Z`、Delete/Backspace 保持已有语义；文本焦点内优先处理文本。
- 图标必须有 tooltip、可访问名称和键盘入口；不能只靠生僻图标表达危险动作。

## 13. 完成判定

Beta 0.6 只有同时满足以下条件才算工程完成：

1. 八类 graph 卡与独立 sprite 内容卡全部使用本规格的外壳和内容层次；
2. 改名、自由缩放、折叠、复制粘贴、模板、保存重开和 Undo 语义一致；
3. 50 张结果在 All 筛选下全部存在、可见、可命中，无分页、无内部滚动、无 `+N`；
4. 10/25/50/100/400% 的信息层级符合表格，LOD 不改变几何；
5. 顶栏、左侧栏、检查器、空状态和就近工具条形成完整入口；
6. English/简中与固定窗口/缩放矩阵通过，无重叠、截断或关键动作丢失；
7. 现有创建、连接、运行、失败、结果回流、审阅、处理、导出和保存旅程没有回归；
8. 自动化和脚本截图通过后只写“工程通过”；项目所有者统一人工测试前仍是“人工待验、发布未通过”。

任何一个结果被隐藏、加载 50% 却显示 100%、LOD 改变卡片/端口几何、旧项目丢数据或关键按钮在 1080 宽不可达，都属于阻断候选的问题，不能登记为“以后优化”。
