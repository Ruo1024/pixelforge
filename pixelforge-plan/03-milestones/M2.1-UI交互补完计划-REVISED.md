# M2.1 — UI 交互补完计划（修订版）

> 基于：开发 Agent 审阅建议
> 原则：不重做画布、不提前做节点编辑器、只补"让 M2 真正可用"的缺失交互
> 目标：从"算法完成"走向"可用 alpha"

---

## 审阅建议总结

### 原计划的主要问题

1. **误判现状**：计划认为无限画布缺失，但 `PFInfiniteCanvas` 已有平移/缩放/网格/框选/撤销
2. **架构冲突**：推荐用 GraphEdit 替换当前画布，但这会推翻已有模型，且与 M3 职责重叠
3. **契约冲突**：自定义端口类型与 `GRAPH-SCHEMA.md` 已定义的类型不一致
4. **工期低估**：节点编辑器压缩到 3 天，实际是 M3 完整里程碑（3000 行 + 3-4 周）
5. **范围过大**：16 天计划实际需 4-6 周，且会引发画布/图模型/执行器连锁改造

### 正确的 M2.1 定位

**M2.1 应该做什么**：
- ✅ 让抠图/切分/描边可以**调参数、看预览**
- ✅ 让魔棒/矩形/套索可以**点击操作**（修正选区）
- ✅ 让图片导入有**明确入口**（不是 New 按钮）
- ✅ 错误反馈、撤销、跨平台验证

**M2.1 不应该做什么**：
- ❌ 重做无限画布（已有 `PFInfiniteCanvas` 工作良好）
- ❌ 节点编辑器（属于 M3，有正式契约和架构设计）
- ❌ GraphEdit 替换（M3 负责；且 M3 已定为画布原生自绘节点，不做独立 GraphEdit 面板）
- ❌ 复杂执行引擎、类型系统（M3 职责）

```gdscript
# ui/tools/magic_wand_tool.gd
class_name MagicWandTool
extends PFTool

var tolerance: int = 32
var mode: String = "flood"  # flood/global
var current_selection: PFSelection = null
var _source_image: Image

func get_name() -> String:
    return "Magic Wand"

func on_mouse_press(pos: Vector2i, button: MouseButton, modifiers: int) -> void:
    if button != MOUSE_BUTTON_LEFT or _source_image == null:
        return
    
    var params := {
        "tolerance": tolerance,
        "mode": mode
    }
    
    var new_selection := PFSelection.magic_wand(_source_image, pos, params)
    
    # 布尔操作（Shift加选、Alt减选）
    if modifiers & KEY_MASK_SHIFT and current_selection != null:
        current_selection = current_selection.union_with(new_selection)
    elif modifiers & KEY_MASK_ALT and current_selection != null:
        current_selection = current_selection.subtract(new_selection)
    else:
        current_selection = new_selection
    
    selection_changed.emit(current_selection)

func draw_overlay(canvas: CanvasItem) -> void:
    if current_selection == null:
        return
    
    # 绘制蚂蚁线（marching ants）
    var bbox := current_selection.get_bounding_box()
    _draw_marching_ants(canvas, bbox)

func _draw_marching_ants(canvas: CanvasItem, rect: Rect2i) -> void:
    # 使用虚线 + 时间偏移模拟蚂蚁线动画
    var time_offset := Time.get_ticks_msec() * 0.001
    var dash_length := 4.0
    var dash_offset := fmod(time_offset * 10.0, dash_length * 2)
    
    # Godot 4.x 使用 draw_dashed_line
    for i in range(0, int(rect.size.x), int(dash_length * 2)):
        canvas.draw_line(
            Vector2(rect.position.x + i, rect.position.y),
            Vector2(rect.position.x + i + dash_length, rect.position.y),
            Color.WHITE
        )
    # ... 四条边都绘制
```

#### 3.3 矩形选区工具

```gdscript
# ui/tools/rectangle_tool.gd
class_name RectangleTool
extends PFTool

var _start_pos: Vector2i
var _end_pos: Vector2i
var _is_dragging := false
var current_selection: PFSelection = null

func on_mouse_press(pos: Vector2i, button: MouseButton, modifiers: int) -> void:
    if button == MOUSE_BUTTON_LEFT:
        _start_pos = pos
        _end_pos = pos
        _is_dragging = true

func on_mouse_move(pos: Vector2i) -> void:
    if _is_dragging:
        _end_pos = pos

func on_mouse_release(pos: Vector2i, button: MouseButton) -> void:
    if button == MOUSE_BUTTON_LEFT and _is_dragging:
        _is_dragging = false
        _end_pos = pos
        
        var rect := Rect2i(_start_pos, _end_pos - _start_pos).abs()
        current_selection = PFSelection.rectangle(_get_canvas_size(), rect)
        selection_changed.emit(current_selection)

func draw_overlay(canvas: CanvasItem) -> void:
    if _is_dragging:
        var rect := Rect2(_start_pos, _end_pos - _start_pos)
        canvas.draw_rect(rect, Color(1, 1, 1, 0.3), false, 2.0)
        # 显示尺寸提示
        var size_text := "%d×%d" % [abs(rect.size.x), abs(rect.size.y)]
        canvas.draw_string(ThemeDB.fallback_font, rect.position + Vector2(5, -5), size_text)
```

#### 3.4 套索工具

```gdscript
# ui/tools/lasso_tool.gd
class_name LassoTool
extends PFTool

var _points: Array[Vector2i] = []
var _is_closed := false
var current_selection: PFSelection = null

func on_mouse_press(pos: Vector2i, button: MouseButton, modifiers: int) -> void:
    if button == MOUSE_BUTTON_LEFT:
        _points.append(pos)
    elif button == MOUSE_BUTTON_RIGHT:
        # 双击或右键闭合
        _close_polygon()

func _close_polygon() -> void:
    if _points.size() < 3:
        return
    
    _is_closed = true
    current_selection = PFSelection.polygon(_get_canvas_size(), _points)
    selection_changed.emit(current_selection)
    _points.clear()
    _is_closed = false

func draw_overlay(canvas: CanvasItem) -> void:
    if _points.is_empty():
        return
    
    # 绘制路径
    for i in range(_points.size() - 1):
        canvas.draw_line(_points[i], _points[i+1], Color.YELLOW, 2.0)
    
    # 绘制临时线（鼠标到最后一点）
    if not _is_closed:
        var mouse_pos := canvas.get_local_mouse_position()
        canvas.draw_line(_points[-1], mouse_pos, Color.YELLOW_GREEN, 1.0, true)
```

#### 3.5 工具管理器

```gdscript
# ui/tools/tool_manager.gd
class_name PFToolManager
extends RefCounted

signal tool_changed(tool: PFTool)

var _tools: Dictionary = {}  # String -> PFTool
var _current_tool: PFTool = null

func register_tool(id: String, tool: PFTool) -> void:
    _tools[id] = tool

func set_active_tool(id: String) -> void:
    if _current_tool != null:
        _current_tool.on_deactivate()
    
    _current_tool = _tools.get(id)
    if _current_tool != null:
        _current_tool.on_activate()
        tool_changed.emit(_current_tool)

func get_current_tool() -> PFTool:
    return _current_tool

func handle_input(event: InputEvent) -> bool:
    if _current_tool == null:
        return false
    
    if event is InputEventMouseButton:
        var mouse_event := event as InputEventMouseButton
        if mouse_event.pressed:
            _current_tool.on_mouse_press(
                Vector2i(mouse_event.position),
                mouse_event.button_index,
                mouse_event.get_modifiers_mask()
            )
        else:
            _current_tool.on_mouse_release(
                Vector2i(mouse_event.position),
                mouse_event.button_index
            )
        return true
    
    elif event is InputEventMouseMotion:
        _current_tool.on_mouse_move(Vector2i(event.position))
        return true
    
    return false
```

#### 3.6 集成到画布

```gdscript
# ui/canvas/infinite_canvas.gd 修改

var tool_manager: PFToolManager = null

func _gui_input(event: InputEvent) -> void:
    # 如果工具管理器处理了输入，则不走其他逻辑
    if tool_manager != null and tool_manager.handle_input(event):
        queue_redraw()
        return
    
    # 原有的平移/缩放/框选逻辑
    super._gui_input(event)

func _draw() -> void:
    super._draw()
    
    # 绘制工具 overlay
    if tool_manager != null:
        var tool := tool_manager.get_current_tool()
        if tool != null:
            tool.draw_overlay(self)
```

## 任务 6：批次内容卡 + 整批菜单（MVP · 无连线）

> 范围决策（2026-06-16，选项 c）：M2.1 只做**无连线的批次卡 + 菜单整批**，验证「批次 + 菜单」交互与 v0.1「无 API key 清洗台」定位；**连线、节点执行器、双身份图持久化留 M3**。

### 问题诊断
- M2 已有逐张「选中素材 → 对话框 → 执行」，但缺「导入一批 → 整批一键处理」的批量爽点。
- 无 AI 也能跑通：导入一批图 → 批次卡 → 菜单一键清洗整批（调真 M1）→ 结果留画布。

### 解决方案（最小批次卡）
- `ui/canvas/canvas_batch_card.gd`：画布上一张容器卡，持有一队 asset_id（导入/框选成批），内部缩略网格。**不带端口、不连线**（与 GRAPH-SCHEMA §5a 的 batch 节点同一视觉/交互，但本阶段不接图执行器）。
- 边框菜单：整批清洗 / 整批抠图 / 整批描边 / 拆小批次 / 导出整批。每项对批内每张调 core（pipeline/matting/outliner，与 M1/M2 同函数），结果作为新素材版本写回批次，进 undo 栈、记 provenance（origin="cleaned"/"edited"，parent_asset 链，graph_id 空）。
- 拆小批次：卡内勾选子集 → 分离出独立子批次卡。
- 持久化：本阶段批次卡暂存于 canvas.json 的轻量结构（asset_id 列表 + 位置）；M3 再升级为「graphs 节点 + canvas node 引用」双身份（预发布期 schema 就地演进，无迁移）。

### 验收标准
- [ ] 导入/框选一批图 → 成批次卡，缩略网格显示
- [ ] 「整批清洗」对 ≥20 张一键处理，进度可见、可整体撤销（Ctrl+Z 回退全批）
- [ ] provenance 正确（parent_asset、origin、graph_id 空）
- [ ] 拆小批次得到独立子批次卡，原批次不受影响
- [ ] 无 AI 全程跑通（不依赖任何 provider/API key）

## 与 M3 的边界

**M2.1 做什么**：
- ✅ 让用户能调参数、看预览、手动选区
- ✅ 批次内容卡（无连线）+ 整批菜单：一键对整批调真 M1/M2，记 undo+provenance（验证批次交互；连线/执行器/双身份持久化留 M3）
- ✅ 保持"选中素材 -> 点按钮 -> 对话框 -> 执行 -> 生成新素材"的批量流程

**M3 做什么**（不在 M2.1 范围）：
- ❌ 画布原生节点编辑器（自绘端口/连线，M3；非 GraphEdit 独立面板）
- ❌ 节点连线与数据流
- ❌ 节点执行引擎与拓扑排序
- ❌ 按 `GRAPH-SCHEMA.md` 定义的正式端口类型
- ❌ 可保存的工作流模板

**为什么这样划分**：
- M2.1 补齐"可用性"缺失，让 alpha 版本能对外
- M3 是架构升级，引入图计算范式，有正式契约和更长开发周期
- 避免 M2.1 做"临时节点系统"导致 M3 推倒重来

## 总结

### 核心改进
1. **图片导入明确** - File > Import Images 替代含糊的 New
2. **参数可视化** - 对话框 + 实时预览替代黑盒按钮
3. **手动选区工具** - 魔棒/矩形/套索让用户能精确控制
4. **错误引导** - 失败时给建议而非抛 warning

### 开发难度
- **总工期**：7-10 天（单人/Agent 连续）
- **风险等级**：中等（无架构重做，基于现有画布扩展）
- **测试负担**：适中（单元测试 + 手动跨平台验证）

### 预期效果
用户可以：
1. 拖入图 -> 选中素材 -> 点 Matte
2. 弹出对话框 -> 拖动容差滑块 -> 实时预览
3. 不满意 -> 按 W 用魔棒手动选背景 -> Shift 加选
4. 满意 -> 点 OK -> 生成新素材
5. 继续 Slice/Outline -> 导出

这是一个"从算法完成到真正可用"的 alpha 体验，而不是跳过这步直接做节点编辑器。

---

**结论**：M2.1 修订版是务实的 UI 补完计划，范围可控，风险可控，为对外 alpha 铺路。节点编辑器回归 M3，按正式契约实施。
