# M2 — 抠图与切分（功能2）

> 目标：白底/纯色底自动抠图；魔棒式手动选取抠图；一图多素材自动切分；描边处理。
> 依赖：M1（清洗后的真像素图是最佳输入，但算法须兼容任意 RGBA 图）。

---

## M2-1 色键抠图（matting.gd）

**目标**：自动识别并移除纯色背景。

**技术实现指导**：
- 背景色推断：统计四角 + 四边中点共 8 个采样点（各取 3×3 均值），聚类（容差内合并），最大簇为背景色候选；候选占边界像素比 < 60% 时报"非纯色底"（UI 转手动）。
- 移除策略两种：
  - `flood`（默认）：从全部边界背景色像素种子 BFS 泛洪（容差 OKLab 距离，默认阈值可调 0–100 映射），只清外部连通区——**保留物体内部同色区**（如白色高光），这是比全局色键更正确的默认。
  - `global`：全图色键（物体内同色也透明化），处理"背景透过镂空可见"场景。
- 边缘处理：泛洪边界 1px 内做半透明判定（与背景距离线性映射 alpha），随后按像素画规范二值化（默认）或保留（参数）。
- 真像素图优化路径：若图已清洗（逻辑分辨率小），算法天然快；高分图先警告建议先清洗。

**验收标准**：
1. 单测：白底放一个含白色高光的红苹果（fixture 程序绘制）→ flood 模式高光保留、背景全透明；global 模式高光也透明。
2. 渐变底（非纯色）→ 正确报告"非纯色底"不误删。
3. 容差 0 时仅精确同色被移除。

## M2-2 魔棒与手动选区抠图

**目标**：在完整像素画中选取部分素材抠出（用户痛点：从 AI 生成的整张场景里挖单个物体）。

**技术实现指导**：
- 选区模型 `PFSelection`（core）：位掩码（PackedByteArray，每像素 1 字节）+ bbox 缓存；支持并/差/交布尔（Shift 加选 / Alt 减选）。
- 工具集（画布 OverlayLayer 交互，工具状态机模式——后续 M6 编辑器复用同一状态机框架）：
  - 魔棒 W：点击处 flood 选择（容差可调，滚轮微调实时更新——性能：增量 BFS 或重跑均可，逻辑分辨率图很小）。
  - 矩形选区 M、套索 L（多边形点击 + 双击闭合，光栅化用扫描线填充）。
- 选区视觉：蚂蚁线 shader（时间偏移虚线描边掩码边缘）。
- 抠出动作：`提取为新素材`（拷贝选区 bbox 内掩码像素 → 新素材 + 新画布元素；provenance.parent_asset 记录来源）与 `剪切提取`（原图选区清为透明，进 undo）。

**验收标准**：
1. 单测：构造已知连通区图，魔棒各容差选区掩码与期望一致；布尔运算正确。
2. 集成：场景图中套索圈出房子→提取→新素材尺寸=掩码 bbox、外部全透明、provenance 链接正确。
3. 魔棒在 256×256 逻辑分辨率图上响应 < 50ms。

## M2-3 连通域切分（segmenter.gd）

**目标**：一张图内多个独立素材（如 AI 一次生成 6 个道具铺在画面上）自动切成多个素材。

**技术实现指导**：
- 前提：图已具透明背景（或先跑 M2-1）。alpha > 0 为前景。
- two-pass 连通标记或 BFS（图小，BFS 够），**8-连通**默认（像素画对角相连常见）。
- 合并启发（关键体验点）：纯连通域会把"飘在头顶的火焰"切离本体。提供 `merge_distance`（默认 2px）：bbox 间距 ≤ 该值的组件合并为一个素材。UI 实时预览切分框，用户可调距离、手动框选合并/拆分（点击两框连线合并）。
- 排序：输出按"先行后列"（top-left 栅格顺序）编号命名 `{原名}_01..`。
- 过滤：面积 < min_area（默认 4px）的碎屑忽略（防噪点成素材）。
- 节点化预留：本算法即 M3 `slice` 节点的实现体。

**验收标准**：
1. 单测：fixture 摆 6 个已知 bbox 图形 + 2px 噪点 → 恰切 6 个、bbox 正确、噪点被滤。
2. "本体+1px 间隙飘浮部件"样本：merge_distance=0 切 2 个，=2 合 1 个。
3. 集成：白底 AI 多物体图 → 抠图 → 切分 → 批量入库，每素材可独立拖动。

## M2-4 描边工具（outliner.gd）

**目标**：为素材添加/移除 1px 描边（风格预设关键字段的执行端）。

**技术实现指导**：
- 添加外描边：alpha 通道形态学膨胀（3×3 十字核或方核=参数 corner 连通性）减原 alpha = 描边掩码 → 填 outline 颜色。注意画布扩边 1px（素材尺寸 +2）。
- 内描边：原 alpha 减腐蚀结果。
- 移除描边：检测"与透明相邻且色值≈描边色（深色启发）"的 1px 壳层并腐蚀掉；提供预览，因有损（移除是启发式的，UI 标注 beta）。
- 彩色描边（colored_1px）：取相邻内部像素色加深 30%（HSL L 通道）作为该段描边色。

**验收标准**：
1. 单测：圆形 sprite 加黑外描边 → 周长壳层均黑、尺寸+2、再移除 → 与原图 IoU ≥ 95%。
2. selective 模式（仅下半部描边，平台游戏惯例）按掩码参数正确生效。

## M2-5 导出器 v1

**目标**：素材正式出口：单图 PNG、批量 PNG、spritesheet 打包。

**技术实现指导**：
- `services/exporter.gd` + UI 导出对话框：选素材集合（当前选中/标签筛选）→ 模式：逐文件（命名模板 `{name}.png`）/ 网格 spritesheet（自动行列或指定列数，padding 参数，附带 JSON 坐标清单——格式对齐通用 TexturePacker JSON hash 子集）/ 放大导出（×2/×4/×8 最近邻，引擎外使用场景）。
- 导出走 task_queue；注册为 `register_exporter` 插件点的首个内部消费者（PLUGIN-API 对齐）。

**验收标准**：
1. spritesheet JSON 被 Godot `AtlasTexture` 手工还原验证坐标正确。
2. 放大导出像素完全锐利（颜色集合不增）。
3. 100 素材批量导出 < 10s。
