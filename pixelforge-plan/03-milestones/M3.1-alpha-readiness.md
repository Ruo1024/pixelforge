## 1. v0.1 最小旅程

陌生测试者无需口头指导，在 15 分钟内完成：

```
启动
  → 导入 1–5 张真实 AI 图且立即看见内容
  → 批次清洗/抠图；必要时对单图切分或描边
  → 查看前后结果并筛选
  → 保存、关闭、重新打开
  → 导出 PNG / spritesheet，并找到产物
```

“完成”必须同时满足入口可发现、过程有反馈、结果可判断、失败可恢复、产物可找到；只证明算法或按钮存在不算完成。

## 2. AR-1 数据安全底线

**目标**：任何常规操作都不能静默丢失未保存工作，恢复与文件失败对用户可见。

**工作**：
- New / Open / Quit 遇到 dirty 项目统一提供 Save / Discard / Cancel；Save 失败时不得继续破坏性动作。
- 调整恢复通知时序，确保主 UI 建立后仍能可靠收到；恢复副本不得静默覆盖原项目，后续保存语义明确。
- 保存、打开和自动保存失败显示简短原因与可执行下一步；日志只作诊断证据，不作唯一反馈。

**自动化**：覆盖三个 dirty 入口的三分支、保存失败不继续、恢复通知到达、恢复后保存目标。

**人工**：真实窗口分别验证 New / Open / Quit；强杀后重启恢复一次；模拟无写权限路径确认错误可操作。

## 3. AR-2 首次任务与关键反馈

**目标**：真实 AI 图导入后立即可见，耗时处理和导出不再让用户猜测状态。

**工作**：
- 空画布只增加一个低干扰导入提示与明确入口，不建设完整欢迎页或示例项目。
- 文件对话框导入使用稳定的画布落点；不得使用弹窗关闭时鼠标位置。空画布自动聚焦新内容，已有工作区只提供轻提示/聚焦入口，不强行打断上下文。
- 导入失败必须指出失败文件与可重试动作，不产生半成品素材，不得只写日志后留下空画布。
- 真实 1254×1254 输入的清洗显示进行中、完成、失败；消除完成后仍停留 `Preview queued` 等陈旧状态。现有协作式取消路径应正确反馈；若取消需要重构算法/队列，则明确显示当前任务不可取消并登记首版后处理，不为本卡扩张底层范围。
- 导出前处理覆盖冲突；导出成功后显示完整路径并提供打开所在目录入口；失败说明未产出哪些文件。

**自动化**：覆盖导入落点与聚焦策略、状态转换、取消、覆盖选择和导出结果摘要。

**人工**：使用本地真实测试图执行最小旅程前半段；测试图只用于本机体验，不 commit、不 push。

## 4. AR-3 候选构建与受邀验收

**目标**：用真实候选构建证明 v0.1 最小旅程可由陌生测试者独立完成。

**工作**：
- 同步应用版本、项目 README 和当前状态；候选版本使用 `0.1.0-alpha.N`；生成 macOS 可运行候选构建，在干净用户配置下启动。
- 构建包不得包含 `test picture/` 或 `pixel/tests/fixtures/real/` 的未授权真实测试图；出口自查必须覆盖该红线。
- 由至少 3 名未参与开发的目标用户执行同一份最小旅程，只给应用和一页任务说明，不口头教学。
- 记录完成/失败点、首次导出时间和是否愿意继续处理下一批素材；失败必须回填报告或设计债。

**出口**：
- 至少 2/3 测试者无需口头指导在 15 分钟内完成最小旅程。
- P0 数据丢失、找不到导入内容、任务无反馈问题为零。
- 全量自动化、当前主平台候选构建、保存重开与强杀恢复均通过。
- 报告明确写“受邀 alpha”，不得写成公开发布或跨平台完成。
