# Beta 0.5 — 工作流复用与 90% 收口

> 状态：Beta 0.4 工程门通过后连续执行
>
> 总路线：[Beta 0.3–0.5 原型冲刺](BETA-0.3-0.5-ROADMAP.md)
>
> 版本出口：图像向无限画布至少 103/114 的工程候选，随后停止增加原型功能

## 1. 本轮用户旅程

> 用户把几十条对象提示放进结构化输入，选择其中一部分，先预览请求数量与费用，再运行一个阶段组。独立分支并行执行，失败项可以单独重试，结果保持输入来源。用户将已经验证的阶段组保存成“参考图批量生成”工作流；新建另一个项目，从工作流库在指定位置插入，填入新的素材后继续编辑和运行。最后打开一个接近真实规模的项目，总览、局部工作、保存重开和结果定位都成立。

本轮完成后不再增加新的竞品模块，进入 UI、手感、性能和真实模型兼容的打磨阶段。

## 2. 必做与明确不做

### 必做

- 单节点/目标分支和阶段组的明确运行范围；
- 结构化批量输入，结果能定位到原输入行；
- 把阶段组保存为本地工作流模板；
- 工作流再次插入后仍可编辑、连接和运行；
- 最近项目、内置快速起步模板和用户工作流入口；
- 连线显隐、快捷键、低倍率状态和真实规模自动化样例；
- 48 个核心项和至少 7 个次要项的固定矩阵；
- 三轮唯一候选包与统一人工验收单。

### 明确不做

- 通用循环节点、脚本语言和复杂变量系统；
- 携带素材的工作流 ZIP、跨项目剪贴板、在线模板市场；
- 持久任务中心、项目版本历史、共享和协作；
- 自动布局、书签和导航历史；
- Computer Use 或 agent 自主完整体验测试；
- 人工验收前合并 `main` 或 push。

工作流模板可以跨项目使用，但不打包真实素材：参考图节点保存为待填写输入槽，避免把素材复制、许可证和引用迁移问题带进原型阶段。

## 3. 依赖与集成顺序

```text
B5-0 结构化输入、组运行与模板契约
   ├─> B5-1 运行节点、分支和阶段组
   └─> B5-2 结构化批量输入
           └─> B5-3 保存本地工作流模板
                    └─> B5-4 工作流库与快速起步
                             └─> B5-5 大画布操作收口
                                      └─> B5-6 90%工程矩阵与唯一候选
                                               └─> B5-7 统一人工验收交接
```

B5-1 与 B5-2 可在 B5-0 后并行。B5-3 依赖分组和运行语义稳定；B5-4 至 B5-7 顺序合流。

## 4. 执行卡

### B5-0 结构化输入、组运行与模板契约冻结

**服务旅程**：在最后一轮开工前固定计分、运行范围和工作流模板含义，防止收口时再换分母或临时扩格式。

**交付**：

- 复核开工前已冻结的 [90% 受控矩阵](BETA-0.3-0.5-PARITY-MATRIX.md)：48 核心项、18 次要项和四个 PixelForge 硬门；不得在本轮更换分母或主卡；
- 固定运行范围：节点/目标分支和显式 frame，不增加“一键运行整张图”；
- 固定组运行语义：有效终点只指 frame 内“输入有效且可提交”的 `ai_generate` 节点；运行它们及必要上游，组外但被依赖的上游可作为前置，其他内容不运行。结果进入已有空 batch 或按 B4-3 新建 batch；不含可运行生成节点的 frame 禁止运行并说明原因；
- 在 `02-contracts/WORKFLOW-TEMPLATE.md` 固定本地工作流模板最小内容：独立 schema 版本、名称、白名单节点参数、内部连线、frame 和相对布局；
- 模板保存采用 fail-closed：只允许计划列出的内置节点和明确参数键；幽灵、插件、历史 Comfy 或未知节点拒绝保存；
- 自定义风格只能安全内嵌完整 StylePreset/调色板数据，否则转为待填写槽；
- 工作流模板不保存 API key、运行中状态、结果素材/审阅状态、绝对路径或真实图片；允许保留已清空素材的 batch 节点作为结果槽，`image_input/reference_set` 转为待填写输入槽；
- 插入时重映射所有节点和 frame ID，放到指定位置，作为一次 Undo；
- 图节点 position、canvas item、frame_id 和内部 edge 必须一起重映射，不能留下双位置真相；
- 模型 ID 和风格可以保留；当前环境没有该模型时显示可修复警告，不丢参数；
- 模板保存在本机用户数据目录，不进入账号、云同步或项目版本历史；
- 模板原子写；损坏文件跳过并给用户可见警告，不阻止其他模板加载；
- `object_list.params.rows` 最小结构为 `{id, text, count, enabled}`；旧 `items` 只读转换，`rows` 存在后成为执行真相；公开端口仍输出 `text_list`，运行协调器另行携带 `source_row_id`；
- rows 存在时，行 `count` 是该行候选数唯一真相；超过模型 `max_batch` 时拆成多次请求但保持同一 `source_row_id`。`ai_generate.batch_size` 只服务单条 `text_prompt`/旧 items 的默认数量，`size_spec.per_subject` 只作旧兼容读取，不与 row count 相乘。

**完成条件**：矩阵分母固定；旧项目不受模板格式影响；后续卡不再临时设计素材打包或通用子图。

**自动化**：模板 schema/白名单、敏感字段禁止、原子写/损坏跳过、ID/位置重映射、未知节点/模型降级、旧 `items` 转 rows 和共用兼容矩阵。

**外部参考触发点**：只有组选区保存或再次插入的用户行为仍模糊时，派子 agent 只读外部工作流保存/插入模块；不复制其文件格式。

### B5-1 运行节点、分支和阶段组

**服务旅程**：用户一次启动明确范围，不必逐张生成卡重复点击，也不会意外运行无关付费分支。

**固定语义**：

| 动作 | 实际运行范围 |
|---|---|
| 运行节点/分支 | 该目标的上游依赖闭包 |
| 运行阶段组 | frame 内输入有效、可提交的 `ai_generate` 节点及其必要上游；不运行无关分支 |

**交付**：

- 选中工具条提供与当前选择匹配的运行入口；
- 提交前高亮即将运行的节点和连线，并显示预计请求数、结果数和已知费用；
- 阶段组把每个有效 `ai_generate` 交给 Beta 0.4 的目标感知运行协调器；不同目标的状态和结果保持隔离；仅含输入/结果卡的组不可运行并解释原因；
- 多目标允许有限并行，受 TaskQueue 和模型能力上限约束；一个目标失败不抹掉其他目标结果；
- 取消只作用于本次运行范围；
- 失败后只重试失败目标；不在本轮承诺通用缓存复用或智能跳过付费上游；
- 结果继续遵守 Beta 0.4 的新结果卡、不覆盖旧结果语义。

**完成条件**：运行前范围清楚，运行中能定位当前和失败节点，运行后各结果回到正确分支。

**自动化**：线性链、菱形依赖、多个目标、frame 内外依赖、不完整无关节点、失败隔离、取消、失败目标重试、并发上限和成本汇总。

**外部参考触发点**：级联顺序仍有歧义时，只读参考外部项目的级联运行可见行为；PixelForge 的无环图和目标闭包契约优先。

### B5-2 结构化批量输入

**服务旅程**：用户一次管理几十个对象，选择其中一部分生成，并知道每个结果来自哪一行。

**交付**：

- `object_list` 展开后显示最小结构化行：稳定行 ID、启用状态、对象/提示词、数量；
- 支持粘贴多行、逐行编辑、全选/取消、筛选和只运行选中项；
- 同一个分支的风格、尺寸和参考素材作用于选中行，不在本轮增加每行独立参考图；
- 每行 `count` 决定该行候选总数；模型批量上限只负责拆请求，不改变总数。`batch_size/per_subject` 不与行 count 相乘；
- 运行前显示选中行数、预计请求数、最大结果数和已知费用；
- 复用现有 `text_list` 自动展开语义，不新增循环节点或图中的环；
- 运行协调器在不改变公开 `text_list` 端口的前提下携带 `source_node_id/source_row_id`；结果 provenance 写入二者，不能只依赖会变化的行索引；
- 结果批次可按输入行查看；
- 部分失败时只重试失败行，不重跑已成功行。

**完成条件**：50 行仍可编辑、筛选和选择；生成结果能定位回行；保存重开后条目和选择保持合理的稳定状态。

**自动化**：旧 `items` 转换、稳定行 ID、多行粘贴、行编辑/筛选/选择、rows/旧 items/单条 prompt 三种数量优先级、跨 `max_batch` 拆请求、50 行录制任务、并发上限、部分失败、只重试失败行和 provenance 映射。

**不做**：通用变量、脚本表达式、任意循环、每行独立工作流。

### B5-3 把阶段组保存为本地工作流

**服务旅程**：已经验证的流程不必每次从零搭建。

**交付**：

- 对一个显式 frame 提供“保存为工作流”；保存前显示节点、内部连线和待填写素材槽数量；
- 保存前按 B5-0 白名单检查；存在插件、幽灵、历史 Comfy 或未知参数时拒绝并列出问题，不尝试猜测或静默剥离；
- 只保存 frame 成员和选区内部边；指向外部的边不保存，并在摘要中说明；
- 运行中状态、结果素材/审阅状态和瞬态错误不进入模板；batch 只保留空结果槽结构；
- 提示词、对象列表、尺寸、风格和生成设置保留；真实参考素材转换为空输入槽；
- 插入时生成新 ID、保留相对布局和内部连线，整体放在鼠标或当前视野中心；
- 插入后的节点可正常编辑、连接、分组和运行；
- 用户模板支持命名、重命名和删除；更新流程采用“另存为新模板”，不建立模板版本历史；
- 模板文件使用原子写；损坏模板跳过并提示，不阻止工作流库打开；
- 插入和删除画布实例都可撤销，删除模板文件不删除已插入实例。

**完成条件**：同一模板可在新项目插入并填入新素材；原项目和模板实例彼此不联动；没有密钥或用户图片泄漏。

**自动化**：模板保存/读取、内部边过滤、输入槽转换、ID 重映射、相对布局、插入 Undo、未知模型/节点降级和敏感字段扫描。

**不做**：ZIP 导入导出、素材打包、临时跨项目剪贴板、共享和市场。模板库允许“显式保存无素材模板 → 在另一项目插入”，但不计为 L08/S05 的直接跨项目复制。

### B5-4 工作流库、最近项目与快速起步

**服务旅程**：新用户和回访用户都能快速进入可运行的工作空间。

**交付**：

- 在 Beta 0.4 资源抽屉增加“工作流”分类；
- 支持搜索、用户模板和三个内置模板：文本批量生成、参考图继续生成、生成后进入批量加工/导出；
- 工作流预览只显示名称、用途、节点数、所需模型和待填写输入，不制作高成本动态预览；
- 从空白画布两步内插入模板，插入位置明确，新增内容保持选中；
- 模型缺失时仍允许插入，生成卡显示修复入口；
- 首屏/文件入口显示最近项目，失效路径可以移除，不建设项目云管理后台；
- 保留离线示例作为无密钥路径，与内置模板使用同一真实图数据。

**完成条件**：用户模板保存后立即可搜索和插入；三个内置模板都能通过离线录制响应完成各自旅程；最近项目不会阻止新建/打开。

**自动化**：搜索、模板增删、内置模板结构、插入位置和 Undo、缺失模型/路径、最近项目、空状态和中英文截图。

### B5-5 大画布操作与 UI 收口

**服务旅程**：内容增多后仍能理解结构、定位运行状态和使用高频动作。

**交付**：

- 连线显示/隐藏；
- 选中工具条统一复制、成组、运行、保存工作流和删除；
- Cmd/Ctrl+C/V/D/Z/Shift+Z、Delete/Backspace、Cmd/Ctrl+0 与菜单行为一致；
- 约 10% 总览保留 frame、模块类型、结果缩略、运行/失败状态和主要关系；
- 100% 工作倍率能直接编辑内容，不依赖外部弹窗理解模块；
- 自动构造约 200 个内容模块、多个 frame、分支和结果批次的确定性项目；
- 记录构造、打开、适配全部、保存和重开耗时；发现瓶颈只修复阻断旅程的局部问题；
- 已知基线 orphan/resource 提示不得增长，除非有明确归因和登记。

**完成条件**：大型自动化项目能打开、总览、聚焦分支、运行一个小范围、保存重开；无关内容不参与运行。

**自动化**：连线显隐、工具条/快捷键、低倍率状态、200 模块项目、保存重开和 English/简中 10%/50%/100% 固定截图。

**不做**：搜索、书签、自动布局、万级节点优化和通用渲染重写。

### B5-6 90% 工程矩阵与唯一候选

**服务旅程**：证明三轮能力已经连成一个可交给项目所有者统一验收的候选。

**必须完成**：

- 48 个核心项全部具有实际实现和相应工程证据；需要人工感受的项目另列最终人工步骤，但人工步骤不能替代缺失实现，也不能用次要项补核心项；
- 12 个计划次要项逐项记录，至少 7 个达到工程完成；
- 工程候选分数至少 103/114；
- 四个 PixelForge 硬门都有自动化几何/状态证据，并保持人工待验；
- 全量测试、lint/format、i18n、UI scaling、项目往返、Provider 录制契约、安全和受保护图片守护通过；
- 自动生成中英文、关键状态、不同窗口尺寸和大画布倍率截图；
- 只在此卡构建首次统一 macOS 候选，并记录构建产物与校验和；若项目所有者验收失败，修复后允许递增候选号重建；
- 统一工程报告区分工程通过、人工待验、发布未通过。

**自动化总旅程**：空白创建 → 图片落点 → 两分支与阶段组 → 云模型录制生成 → 候选审阅 → 结果回流 → 结构化批量 → 运行组 → 保存工作流 → 新项目插入 → 再次运行 → 加工/导出 → 保存重开/恢复。

**明确禁止**：Computer Use、真实付费 API 和用脚本截图宣称人工体验通过。

### B5-7 项目所有者统一人工验收交接

本卡不继续开发功能，只准备一份 `reports/manual-test-beta0.3-0.5.md` 和候选使用说明。人工清单覆盖：

1. 中文首次进入、English 切换和主要弹窗；
2. 空画布创建与系统图片拖入；
3. 卡内编辑、复制、连线、分组、小地图与远近导航；
4. 使用项目所有者自己的真实云端密钥生成；
5. 多参考、任务失败/取消/重试、候选详情和结果继续生成；
6. 结构化批量、整组运行和本地工作流模板；
7. 保存重开、autosave 恢复、现有加工和导出；
8. macOS 内屏、外屏和跨屏移动的字体、缩放和裁切。

项目所有者给出：`go / 修复后复验 / 整体不合并`。

- `go`：才可更新为人工通过；是否合并 `main` 和 push 仍以用户明确指令为准；
- 修复后复验：在当前未推送分支修复、复跑相关自动化并重新交付；
- 整体不合并：保留证据，破坏性回退等待用户确认。

账号、权限、协作和版本历史不出现在验收单，也不登记为缺陷。

## 5. 本轮出口

Beta 0.5 自动化完成时，只能标记：

> Beta 0.3–0.5 工程通过；图像向无限画布达到 90% 工程候选；项目所有者人工与发布均待验。

项目所有者人工验收通过后，三轮原型开发结束，下一阶段只处理打磨、真实反馈和明确批准的新方向。
