# M2.1 — UI 交互完善计划

> **【已废弃 · 2026-06-16】** 本稿是 M2.1 的**原始计划**，其中「方案 B：用 GraphEdit 作画布/节点容器」的建议**已被取代**。现行计划见 `M2.1-UI交互补完计划-REVISED.md`；节点模型见统一画布决策（GRAPH-SCHEMA §5a、M3-node-graph.md、04-research/无限画布架构审阅.md 顶部）。本文件仅作历史参考，请勿据此实现。

> 基于：M2 完成报告评估、Godot 开源工具 UI 调研
> 目标：将当前"有其形无其用"的 UI 改造为可用的交互界面
> 范围：无限画布、节点编辑器、文件导入、工具交互、参数面板

---

## 问题诊断

### 当前 UI 状态
- ✅ **已实现**：顶栏按钮（Matte/Slice/Outline）、基础画布、素材列表
- ❌ **缺失**：
  1. 无限画布交互（平移/缩放/网格）
  2. 节点式编辑器（ComfyUI风格）
  3. 文件导入UI（拖放/New按钮）
  4. 工具交互（魔棒点击、矩形拖拽、套索）
  5. 参数调整面板（实时预览）
  6. 图层/素材管理面板

### 用户体验缺陷
1. **无法导入素材**：New 按钮未连接 FileDialog
2. **无法操作画布**：不能平移/缩放查看细节
3. **无法手动选区**：魔棒/矩形/套索只有 core 算法无 UI
4. **无法调整参数**：抠图容差、切分距离无法实时调整
5. **无法预览结果**：点击按钮黑盒执行，无预览无撤销

## M2.1 设计方案

### 参考架构（基于调研报告）

#### 1. 无限画布系统（参考：Lorien + Godot GraphEdit）

**技术选型**：
- 方案A：Camera2D + CanvasLayer（Lorien风格）
  - 优点：简单直观，适合自由绘图
  - 缺点：需手动管理视口变换
- 方案B：GraphEdit 作为画布容器（Godot内置）
  - 优点：内置缩放/平移/网格，节点编辑器天然支持
  - 缺点：需适配到像素画场景

**推荐方案**：**方案B（GraphEdit）**
- 理由：M2.1 需同时实现无限画布 + 节点编辑器，GraphEdit 一箭双雕
- 渐进路线：
  - Phase 1：GraphEdit + 素材节点（拖放摆放，预览图）
  - Phase 2：添加处理节点（Matte/Slice/Outline）
  - Phase 3：节点连线（数据流可视化）

**代码框架**（基于 Material Maker 模式）：
```gdscript
# ui/canvas/infinite_canvas.gd
class_name PFInfiniteCanvas
extends GraphEdit

var _zoom_min := 0.1
var _zoom_max := 8.0

func _ready() -> void:
    # 启用内置功能
    show_grid = true
    show_zoom_label = true
    minimap_enabled = true
    # 连接信号
    connection_request.connect(_on_connection_request)
    disconnection_request.connect(_on_disconnection_request)
    # 网格配置
    grid_pattern = GraphEdit.GRID_PATTERN_DOTS
    
func add_asset_node(asset: Dictionary, pos: Vector2) -> PFAssetNode:
    var node := PFAssetNode.new()
    node.setup(asset)
    node.position_offset = pos
    add_child(node)
    return node
```

#### 2. 节点系统（参考：Material Maker + Godot Orchestrator）

**节点类型设计**：
```
输入节点：
  - AssetNode：素材源（缩略图预览）
  
处理节点：
  - MatteNode：抠图（参数：mode/tolerance/feather）
  - SliceNode：切分（参数：merge_distance/min_area）
  - OutlineNode：描边（参数：color/width/type）
  - CleanupNode：清洗（M1 能力复用）
  
输出节点：
  - ExportNode：导出（格式/路径）
  - PreviewNode：预览窗口
```

**端口类型系统**（基于 Godot Orchestrator）：
```gdscript
# core/graph/port_types.gd
enum PortType {

}

class PFGraphNode extends GraphNode:
    var input_ports: Array[PortType]
    var output_ports: Array[PortType]
    
    func execute(inputs: Array) -> Array:
        # 子类实现具体算法
        pass
```

# ui/tools/magic_wand_tool.gd
class_name MagicWandTool
extends PFTool

var tolerance: int = 32
var mode: String = "flood"  # flood/global
var current_selection: PFSelection = null

func on_mouse_press(pos: Vector2i, button: MouseButton) -> void:
    if button == MOUSE_BUTTON_LEFT:

            "mode": mode
        })

#### 4. 文件导入系统（参考：Pixelorama + Editor Assets Dock）

**拖放导入**：
```gdscript
# ui/shell/main.gd
func _ready() -> void:
    # 启用拖放
    get_viewport().files_dropped.connect(_on_files_dropped)

func _on_files_dropped(files: PackedStringArray) -> void:
    for path in files:
        if path.ends_with(".png") or path.ends_with(".jpg"):
            _import_image(path)
```

**New 按钮导入**：
```gdscript
var _file_dialog: FileDialog

func _init_file_dialog() -> void:
    _file_dialog = FileDialog.new()
    _file_dialog.file_mode = FileDialog.FILE_MODE_OPEN_FILES
    _file_dialog.access = FileDialog.ACCESS_FILESYSTEM
    _file_dialog.filters = PackedStringArray(["*.png;PNG Image", "*.jpg;JPEG Image"])
    _file_dialog.files_selected.connect(_on_files_selected)
    add_child(_file_dialog)

func _on_new_button_pressed() -> void:
    _file_dialog.popup_centered_ratio(0.8)
```

#### 5. 参数面板与实时预览（参考：Material Maker 参数系统）

**参数面板基类**：
```gdscript
# ui/panels/param_panel.gd
class_name PFParamPanel
extends VBoxContainer

signal param_changed(key: String, value: Variant)

func add_slider(label: String, key: String, min_val: float, max_val: float, default: float) -> void:
    var hbox := HBoxContainer.new()
    var label_node := Label.new()
    label_node.text = label
    var slider := HSlider.new()
    slider.min_value = min_val
    slider.max_value = max_val
    slider.value = default
    slider.value_changed.connect(func(val): param_changed.emit(key, val))
    hbox.add_child(label_node)
    hbox.add_child(slider)
    add_child(hbox)
```

**实时预览模式**（抠图工具示例）：
```gdscript
# ui/tools/matte_preview.gd
var preview_texture: ImageTexture
var preview_timer: Timer

func _on_tolerance_changed(val: float) -> void:
    tolerance = int(val)
    # 延迟 100ms 后预览（防抖）
    preview_timer.start(0.1)

func _update_preview() -> void:
    var result := PFMatting.matte(source_image, {
        "tolerance": tolerance,
        "mode": mode
    })
    preview_texture.set_image(result["image"])
```

## 总结

M2.1 将 PixelForge 从"命令行工具"升级为"可视化编辑器"，核心是：

1. **无限画布** - 让用户看得见操作空间
2. **工具交互** - 让用户摸得着选区抠图
3. **参数面板** - 让用户调得准容差距离
4. **节点编辑器** - 让用户搭得起工作流

技术债清偿：文件导入、画布操作、工具系统、实时预览全部补齐。

预期成果：用户可以"拖入图 -> 点击魔棒 -> 调节参数 -> 预览结果 -> 连接节点 -> 导出素材"完整闭环，真正做到"好用"。
