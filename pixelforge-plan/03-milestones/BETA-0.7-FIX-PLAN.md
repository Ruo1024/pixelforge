## 1. 目标与边界

本轮只修复项目所有者在 Beta 0.7 人工体验中提出的十类问题：

1. 卡片内部滚动到边界后误缩放画布、滚动位置跳变。
2. 重做生成卡，并把 API 地址、Key、Ping 和开发者模式移到顶部工具栏。
3. 把风格提示词改成真正可选择、复制和编辑的预设流程。
4. 提示词输入框自动换行。
5. 参考图支持拖动排序，删除文件名和“更多”占位。
6. Reference 和 Output 在 100% 画布倍率下显示明显大图，并随卡片尺寸动态重排。
7. 删除 AI 预算、估价和费用 UI。
8. 像素清晰恢复完整右侧设置面板，画布上只保留紧凑卡。
9. 新增 Reference 直通 Output 的零 API 调试路径。
10. 被点击的卡片临时显示在最上层。

本轮不顺手处理其他版本迁移、全局架构重写、全局 i18n 重构、新 Provider、多账号、云同步或发布工作。

### 1.1 已确认的产品决定

- 生产主路径使用 `openai_image / gpt-image-2`。
- 第三方平台通过 OpenAI-compatible `base_url` 接入，不为每个平台另写生成卡。
- 尺寸只保留 `720p / 1080p / 2K / 4K` 四个固定档位。
- 每个档位提供横向、纵向、正方形；不提供自定义宽高和比例锁。
- 单次运行上限 16 张；1–4 张直接运行，5–16 张先做一次不含金额的数量确认。
- 生成卡删除 Quality、Advanced、Seed、预算、估价和费用展示。
- 最终核心提示词默认隐藏，只在顶部“开发者模式”开启后显示。
- 风格节点只负责把预设提示词前置注入生成提示词。
- 像素清洗预设属于像素清晰节点，不改变风格节点语义。
- E 不调用真实 API。Ping、生成和编辑均用本地 mock 验证；真实平台由项目所有者后续手动测试。
- 费用系统从产品主路径删除，但 Provider 已返回的原始 `actual_cost_usd`、`charge_id` 可继续作为不可见审计字段保存。

### 1.2 不得破坏的既有边界

- PF-SEC-01 继续生效：Key、用户图片、完整提示词、base64 和原始外部响应不得进入日志或错误框。
- 生成 POST 不得自动重试。
- Queued、Running、Canceling、Partial、Failed、Complete、Canceled 等状态继续保留。
- Output 的 Retry、Preview、Edit、Download、Detach、Detach All 等已有动作必须保留。
- 1A / 2B 中未被本文件覆盖的卡片职责、尺寸持久化和 Undo 规则继续有效。
- 不为未发布的旧 Beta 0.7 行为保留兼容 alias。

## 2. 实现规则

### 2.1 卡片滚动与画布输入

- 指针位于卡片内部可滚动控件时，该控件必须消费普通滚轮和触控板滚动；即使已经滚到边界，也不能把同一事件交给画布缩放或平移。
- Prompt、风格编辑器、Reference、Output 和其他内部滚动区使用同一输入所有权规则。
- 画布缩放只在指针不受子控件管理时执行。
- 同一 LOD 档位内平移或缩放不得重建卡片内容实例。
- 内容刷新后必须保留滚动位置、输入焦点和未提交草稿。

最低回归：在滚动区末端继续滚动 10 次，画布 zoom/center、滚动位置、焦点和草稿均不变化。

### 2.2 顶部 API 设置与开发者模式

顶部工具栏新增两个入口：

1. `API 设置`：打开包含 API 地址、API Key、测试连接、保存和取消的对话框。
2. `开发者模式`：控制最终核心提示词预览是否可见，默认关闭。

API 设置规则：

- 默认地址为 `https://api.openai.com/v1`，允许填写 OpenAI-compatible 第三方地址。
- Key 使用项目现有安全凭据存储，不写入项目文件、Undo、日志、截图或错误文本。
- 地址和 Key 草稿被修改后显示“未测试”。
- Ping 使用不产生图片的模型探测请求；只执行一次，不自动重试、不跟随跨主机重定向。
- UI 至少区分：连接成功、鉴权失败、服务可达但无法确认模型、限流、超时、网络/TLS 错误、协议错误。
- 自动化只能访问本地 mock server，不得读取开发机真实 Key 或访问真实平台。

### 2.3 生成卡

生成卡取消内部滚动，固定按以下顺序显示：

1. 标题和运行状态。
2. 只读模型 `GPT Image 2` 与当前 API 主机名。
3. 分辨率：720p、1080p、2K、4K。
4. 方向：横向、纵向、正方形。
5. 数量：1–16。
6. 开发者提示词预览，仅在开发者模式开启时出现。
7. 开始/取消和运行反馈。

必须删除：`BodyScroll`、手工宽高、比例锁、Quality、Advanced、Seed、费用脚注和默认可见的核心提示词。

Graph 中 `ai_generate.params` 只保留：

```text
provider_id, model_id, resolution_preset, orientation,
batch_size, seed=-1, extra={}
```

旧 `target_width`、`target_height`、`ratio_lock`、`quality`、用户 seed 和非空 `extra` 必须拒绝，不保留别名。

固定交付尺寸：

| 档位 | 横向 | 纵向 | 正方形 |
|---|---:|---:|---:|
| 720p | 1280×720 | 720×1280 | 720×720 |
| 1080p | 1920×1080 | 1080×1920 | 1080×1080 |
| 2K | 2560×1440 | 1440×2560 | 1440×1440 |
| 4K | 3840×2160 | 2160×3840 | 2160×2160 |

GPT Image 2 请求边长必须满足 16 的倍数。1080p 请求与交付按以下规则处理：

- 横向请求 `1920×1088`，居中裁切为 `1920×1080`。
- 纵向请求 `1088×1920`，居中裁切为 `1080×1920`。
- 正方形请求 `1088×1088`，四边各裁 4 像素为 `1080×1080`。
- 其他档位不裁切。

数量与执行规则：

- 一次用户运行总结果数为 1–16。
- 5–16 张的确认必须发生在创建 pending slots 和网络任务之前；取消后项目和队列不变。
- Provider 单请求批量上限仍按 descriptor 拆分，拆分后保持顺序。
- 等待期间卡片显示状态和耗时，运行连接线显示发光推进，不伪造百分比。
- 失败使用结构化错误“原因 + 下一步”，不得显示 Key、完整 prompt、用户图片或 raw response。
- 开发者预览与真实请求共用同一个提示词构建函数，不得在 UI 复制拼接逻辑。

### 2.4 提示词与风格预设

提示词输入框：

- 使用可编辑多行 TextEdit，英文、中文、日文和无空格长串都必须在边界自动换行。
- 禁用水平滚动；视觉换行不得改变实际存储字符串。
- 保留项目现有提交、Esc 取消和单次 Undo 行为。

风格预设卡：

- 明确显示当前预设，并提供可理解的预设列表。
- 内置和插件预设只读；点击“编辑”时先复制成用户预设。
- 用户预设支持新建、复制、重命名、编辑、保存和删除。
- 切换存在未保存修改时，必须让用户选择保存、放弃或取消切换。
- 节点保存完整预设快照，用户库之后删除该预设也不能破坏项目重放。
- 运行时只把非空 `preset.prefix` 追加在自由提示词之前。

### 2.5 Reference 与 Output 图片组

两类卡共用媒体网格和图片 tile，避免两套尺寸与缓存逻辑。

- 默认卡片约 `720×520` logical px；允许用户拖动宽高。
- 100% 画布倍率下 tile 不小于 176 px，目标约 224 px，最大约 320 px。
- 根据卡片宽度动态计算列数；最多显示三行，超出后卡片内部滚动。
- tile 必须渲染真实图片，不得只显示状态色块。
- 大量图片使用复用/虚拟化，只为可见项及少量缓冲项创建 Control 和 Texture。
- Reference 删除文件名和“更多”行；左上角显示顺序号，悬停显示替换和删除。
- Reference 图片支持真实拖放排序，包括插入线、拖动 ghost、边缘自动滚动、Esc 取消和一次 Undo。
- Output 保留原有状态、重试、预览、编辑、下载、单张拆分和全部拆分能力。
- 图片顺序、操作目标和持久化都按稳定 `asset_id/slot_id`，不能依赖复用后的 UI 索引。

### 2.6 像素清晰

- 画布上只保留紧凑卡：摘要、设置、开始/取消和状态。
- 点击“设置”打开既有右侧 cleanup inspector。
- 原右侧清洗参数全部恢复，字段含义和默认值保持不变。
- inspector 修改节点参数并形成正常 Undo；运行中禁止修改。
- 设置面板不能直接开始清洗，只有卡片“开始”按钮可以进入协调器。
- 删除旧的 selection-direct-run 或 preview apply 执行入口，避免出现第二套运行路径。

### 2.7 Reference 直通 Output

新增 `reference_set.assets → batch.in` 本地直通：

- 用户选中目标 Output 后点击“运行选区”，按 Reference 当前顺序写入 Output。
- 不复制图片，不创建 Provider request/task，不访问 HTTP，不进入排队或生成状态。
- 输出 slot 标记为本地素材成功态，并保留原 `asset_id`。
- 输入存在缺失、重复或无效资产时整次失败，Output 保持原样，不能写一半。
- 只替换目标 Output 的素材结果，不改变卡片位置、尺寸、标题和连接线。
- 已带真实生成审计记录的 Output 不允许被本地直通静默覆盖，应要求使用空白 Output。

### 2.8 点击卡片置顶

- 点击或框选后的主选中卡片进入临时 SelectedItemLayer，在视觉和命中测试中都位于其他卡片之上。
- 切换选择、取消选择或删除节点时恢复正常层级。
- 临时置顶不写入项目、不产生 Undo、不改变基础 z-order，也不影响导出和重开结果。

### 2.9 删除费用系统与补齐文案

- 删除预算设置、估价、月累计、费用确认、费用脚注以及生产路径中的 CostService/estimate_cost。
- 保留 Provider 原样返回的结算标识作为隐藏审计数据；不得据此恢复用户费用 UI。
- 本轮新增或修改的所有可见文案进入集中 i18n 目录，至少覆盖 `zh_CN` 和 `en`。
- 本轮只补相关键，不做全局 i18n 架构重写。

## 4. 测试与效率规则

### 4.1 测试阶梯

每张卡依次执行：

1. 解析、changed-file lint 和 touched-files 格式检查。
2. 当前卡定向单元测试。
3. 受影响的集成测试簇。
4. 每个完整合流波次最多一次全量测试。
5. 最终只运行一次 `./pixel/scripts/verify_beta_0_7.sh`；修复其红灯后才重新完整运行。

固定截图使用 `./pixel/scripts/capture_beta_0_7.sh`。该脚本内部调用 evidence verifier，不再单独重复执行 verifier。场景脚本是 `capture_beta_0_7.gd`。

### 4.2 必须具备的证据

| 问题 | 最低证据 |
|---|---|
| 滚动泄漏 | 边界后继续滚动不改变画布；offset、焦点、草稿保留 |
| 生成卡 | 无废弃控件；12 种尺寸映射、1080p 裁切和 1–16 上限通过 |
| API 设置 | 本地 mock 覆盖成功与主要错误；Key 不进入项目、日志和截图 |
| 风格预设 | 只读来源、用户副本、未保存切换和项目重放通过 |
| 提示词 | 中英日与无空格长串换行，存储文本不变 |
| Reference | 真实图片、动态布局、拖放排序、Undo 和无文件名行通过 |
| Output | 真实图片及所有原动作通过；大量图片只创建有限可见项 |
| 费用删除 | UI 与生产调用路径不存在，隐藏审计字段仍可读取 |
| 像素清晰 | 紧凑卡、完整 inspector、参数 roundtrip 和唯一运行入口通过 |
| 本地直通 | 顺序正确、原子失败、零 Provider/HTTP/task |
| 点击置顶 | 视觉与命中一致；不改变项目、Undo 和基础 z-order |

### 4.3 防止再次低效执行

- 一张卡只处理一条用户旅程和一个主要故障域；预计超过 90 分钟或触碰超过 20 个 runtime 文件时先拆卡。
- 最多两个写代码 agent 加一个只读审查 agent；共享文件由集成人独占。
- 每卡最多三个主提交：失败证据、实现、收口；不要逐断言提交。
- 只格式化 touched files，禁止全仓格式化。
- 同一根因连续两次全量红灯后回到定向测试，不能盲跑第三次。
- 每 25 分钟报告：可见结果、触碰文件数、测试次数、阻断和下一步。
- 禁止为单一调用者新增抽象层，禁止重建全量 requirement manifest，禁止顺手迁移范围外代码。

以下情况立即暂停并回 P：

- 产品决定或跨模块契约出现本文件未定义的新含义。
- 同一根因第三次仍无法清零。
- 两个 agent 修改同一共享文件。
- 90 分钟没有可演示闭环。
- 需要真实 API、真实 Key、受保护图片、merge、push、候选构建或发布。
- 发现数据丢失、密钥泄漏或许可证风险。

## 5. 项目所有者最短人工复验

本轮人工复验不要求真实 API；真实平台由项目所有者另行手动检查。

1. 在 Output 和长 Prompt 中滚到末端后继续滚动 10 次，确认画布不缩放且滚动条不跳端。
2. 打开 API 设置，检查地址、Key、Ping 状态和保存流程；不发送图片生成请求。
3. 检查生成卡无内部滚动、比例锁、Quality、Advanced、Seed 和费用；四档尺寸、三种方向及 1–16 数量可选。
4. 切换开发者模式，确认最终提示词预览只在开启时出现。
5. 粘贴长中文、英文和无空格文本，确认全部在输入框内换行。
6. 切换内置风格，复制成用户预设后编辑、保存并重启确认。
7. 导入至少 12 张参考图，在 2560×1440、100% 倍率下确认是明显大图；拖动排序并 Undo。
8. 把 Reference 接到空白 Output，运行选区；确认图片按顺序立即出现且没有生成状态。
9. 点击像素清晰“设置”，确认完整参数在右侧；修改后只能从卡片“开始”运行。
10. 创建三张重叠卡片，依次点击下层卡片，确认被点击卡片始终在视觉和命中上最靠前。

前十项通过只能记为“Beta 0.7 Fix 本地/UI 人工通过；真实 API 由项目所有者自行验证”。这不等于候选通过、merge 通过、push 通过或发布通过。
