# Beta 0.3 — 内容工作区

> 状态：待项目所有者批准总路线；收到一次开始开发/Goal 指令后连续执行，不在本版出口暂停
>
> 总路线：[Beta 0.3–0.5 原型冲刺](BETA-0.3-0.5-ROADMAP.md)
>
> 目标区间：把当前约 50%–55% 的产品原型推进到约 65%–70%

## 1. 本轮用户旅程

> 用户新建空项目，从固定添加入口或空白处加入提示词；把一张或多张图片拖到画布指定位置，图片立即成为真实参考卡。他加入风格、尺寸和生成卡，把内容连成一条分支，复制出第二套方案，再把相关模块组成有标题和颜色的阶段组。用户缩到约 10% 看清全局，用小地图回到某个分支，运行离线生成，保存并重新打开后，内容、位置、连线、组和结果都保持。

本轮只解决“画布能否像内容工作区一样被创建、编辑、组织和导航”。具体云模型、多参考请求和结果继续生成留给 Beta 0.4；整组运行和工作流保存留给 Beta 0.5。

## 2. 必做与明确不做

### 必做

- 空画布无需先选旧节点即可开始；
- 系统图片拖入/粘贴后在落点直接成为参考卡；
- 文本、对象列表、参考图、风格、尺寸、生成和批次均是可读内容卡；
- 复制/粘贴/删除/撤销与内部连线保持；
- 连线的可用与不可用反馈清楚；
- 显式阶段组、小地图、适配全部和低倍率结构视图；
- 本地模型、ComfyUI 及其他延后入口退出主路径；
- 中英文、保存重开和自动化截图继续受门控。

### 明确不做

- 卡片自由缩放、锁定、自动排布、独立便签类型；
- 嵌套组、组折叠、整组运行、保存工作流；
- 拖线到空白处自动创建下一节点；
- 搜索、书签、视口历史；
- 云端模型扩展、真实付费调用；
- Computer Use 或项目所有者分段测试。

普通文本卡可以承担说明用途，不为“便签”再增加一套模块。卡片采用内容自适应高度、折叠和展开编辑，不做自由拖拽改尺寸。

## 3. 依赖与集成顺序

```text
B3-0 行为/契约/自动化样例冻结
   ├─> B3-1 空白创建与图片落点
   └─> B3-2 统一内容卡
           └─> B3-3 常见编辑
                    └─> B3-4 连线反馈与阶段组
                             └─> B3-5 导航与主路径收口
                                      └─> B3-6 集成出口
```

B3-1 与 B3-2 可以在契约冻结后由独立 worktree 并行。其余卡按顺序合流，避免多个 agent 同时修改画布输入路由、选择状态和 Undo。

## 4. 执行卡

### B3-0 行为、最小契约与自动化样例冻结

**服务旅程**：提前固定所有后续卡共用的行为，避免一边开发一边反复改创建、分组和截图口径。

**交付**：

- 固定创建位置、粘贴偏移、多图排布、复制范围、组成员和远景显示规则；
- 在 `PROJECT-FORMAT` 补全最小 `frame`：`id / type / graph_id / title / color / position / size / z_index`；成员唯一真相是节点画布项的 `frame_id`，frame 不另存成员数组；
- 一个 frame 只能容纳同一 graph 的节点，跨 graph 成组必须拒绝并说明原因；
- 组成员只由“成组 / 移入组 / 移出组 / 解组”改变；把卡拖过边界不会自动改变归属；
- 准备一个两分支、两个阶段、包含真实内容卡和批次的确定性自动化项目样例；
- 若现有工具无法自动产出截图，只新增一套共用截图入口，不为每张卡建脚本。

**完成条件**：旧项目安全读取；新字段保存重开；未知字段保留；后续卡不需要重新决定相同行为。

**自动化**：项目格式 round-trip、旧项目默认值、frame 引用失效降级、样例可重复构造。

**外部参考触发点**：只有多图默认排布或显式分组行为仍有歧义时，派子 agent 只读外部项目的拖放/分组模块，回传行为，不复制实现。

### B3-1 从画布任意位置开始

**服务旅程**：让空画布的第一步始终可找到。

**交付**：

- 固定添加入口、双击空白处、右键和 Tab 共用一个模块选择器；
- 模块在触发位置对应的画布世界坐标创建，不依赖当前选择；
- 空项目没有图时由同一路径创建默认图；已有多个图且没有选中节点时统一加入当前 active graph，没有 active graph 时加入默认 graph，不要求用户理解图 ID；
- 从系统拖入或粘贴图片，在落点创建 `image_input` 参考卡；
- 多图按稳定小网格排布，不重叠、不超出当前可见区域的起始方向；
- 支持文件与无效文件混合拖入：有效项继续导入，失败项给出简短原因，不留下空素材或幽灵卡；
- 一次创建或一次多图导入均为一个 Undo 动作。

**完成条件**：新项目两步内出现可编辑文本或真实图片；保存重开后素材和位置保持；创建失败不污染项目。

**自动化**：屏幕/世界坐标、四个入口一致性、剪贴板图片、多图排布、部分失败、Undo/Redo、保存重开。

**不做**：远程 URL、第三方素材协议、视频和音频。

**外部参考触发点**：创建落点、粘贴优先级或多图排布仍不明确时，只读核对外部项目对应 drop/paste 行为。

### B3-2 统一内容卡

**服务旅程**：用户不打开通用参数弹窗，也能看懂画布上是什么内容、处于什么状态、下一步能做什么。

**交付模块**：

| 模块 | 卡内直接显示 |
|---|---|
| `text_prompt` | 可编辑多行提示词、内容摘要 |
| `object_list` | 多条对象、条目数、编辑入口 |
| `image_input` | 真实预览、名称、来源、替换/清除 |
| `style_preset` | 风格名称、调色板和关键风格摘要 |
| `size_spec` | 目标像素尺寸、比例和数量 |
| `ai_generate` | 当前离线/云端服务摘要、输入完整性、运行状态 |
| `batch` | 候选宫格、审阅状态、加工与导出主动作 |

每张卡统一使用：类型/可编辑标题、状态、真实内容、端口/关系、当前阶段可用的主要动作。内容多时可展开编辑，折叠后保留能辨认模块的摘要。高度随内容在安全范围内自适应，不增加自由缩放手柄。

`text_prompt` 与 `style_preset` 虽已存在于契约，但当前注册表尚未提供真实节点。本卡必须实现并注册它们，接通执行、画布绑定、保存重开和 Undo；禁止只画卡面或留下幽灵节点。

**完成条件**：正文不退化成标题摘要；高频动作可见；输入控件不误触拖卡、连线或画布平移；中英文全部走集中 i18n；保存重开与 Undo 保持。

**自动化**：七类模块数据映射、空/有效/失效/选中/运行/失败/完成状态、折叠/展开、输入路由、保存重开、长文几何和 i18n。

**外部参考触发点**：卡片信息层级仍模糊时，只允许子 agent 比较外部普通画布和智能画布中“标题—内容—状态—动作”的可见关系，不参考视觉样式。

### B3-3 常见编辑与原子 Undo

**服务旅程**：用户能快速派生第二种方案，并安全恢复误操作。

**交付**：

- 复用现有单选、框选、多选和多节点移动；
- Cmd/Ctrl+C、Cmd/Ctrl+V、Cmd/Ctrl+D、Delete/Backspace、Undo/Redo；
- 复制多张卡时保留相对位置、选区内部连线和安全的素材引用；
- 粘贴/重复生成新节点和画布项 ID，不复制运行中瞬态状态；
- 粘贴位置以鼠标附近为优先，无鼠标位置时放在当前视野中心并固定偏移；
- 文本正在编辑时，复制粘贴优先作用于文本；
- 复制、粘贴、重复和批量删除各自作为一次原子 Undo。

**完成条件**：复制一条小分支后原分支不变，内部关系保持，新旧 ID 不冲突；保存重开后派生分支仍正确。

**自动化**：ID 重映射、内部/外部边处理、相对位置、文本焦点快捷键、原子 Undo/Redo、素材引用和项目往返。

**不做**：跨项目剪贴板、云剪贴板、复制任务状态。

**外部参考触发点**：只有内部边复制和粘贴落点仍有疑问时，派子 agent 查看外部项目的复制/粘贴可见结果。

### B3-4 连线反馈与显式阶段组

**服务旅程**：模块关系能真正传递内容，多个模块可以组成清楚的创作阶段。

**连线交付**：

- 保留现有端口类型、单输入、环检测和图模型；
- 拖线经过端口时，在放手前明确显示可连接或不可连接；
- 非法连接给出简短原因，不静默失败；
- 连线可选择、删除和撤销；
- 点击生成卡运行时，以按钮所属节点为目标，不依赖画布当前选中了谁。

**阶段组交付**：

- 多选后“成组”，生成有标题、颜色和自动边界的 frame；
- 移动 frame 带动显式成员并保持相对位置；
- 移入、移出和解组使用明确动作，空间重叠本身不改变成员；
- 解组不删除节点或内部连线；
- 删除 frame 等同解组并保留全部内容；成组、移动、解组和删除 frame 各自为一次原子 Undo；
- frame 位于内容后方，不抢卡片/连线输入；
- 远景保留阶段标题、颜色和范围；
- 本轮 frame 只组织内容，不显示不可用的“运行组”按钮。

**完成条件**：分组不是装饰矩形；移动、解组、保存重开与 Undo 不破坏关系；点击哪张生成卡就运行哪张。

**自动化**：连接兼容/非法状态、运行目标回归、成组/移入/移出/解组、frame 移动、z-order、命中、保存重开和 Undo。

**不做**：嵌套、折叠、锁定、自动布局和整组运行。

### B3-5 大画布导航与主路径收口

**服务旅程**：用户能在局部编辑和全局结构间往返，并且主界面只显示当前原型真正支持的路线。

**交付**：

- 可开关小地图，显示内容边界、frame、卡片和当前视口；
- 点击或拖动小地图视口框定位；创建、删除和移动后自动更新；
- 适配全部、聚焦选中和可见缩放百分比；
- 支持约 10% 的总览；低倍率只保留组、模块类型、缩略图/状态和主要连线，回到工作倍率恢复真实内容；
- 保持鼠标锚点缩放和像素图最近邻显示；
- 添加菜单优先显示输入、参考、风格、尺寸、生成、结果和组织；
- 本地模型、ComfyUI 和其他延期入口退出主菜单，但底层代码不删除；
- 更新离线示例为本轮统一自动化样例，不另外制作静态展示界面。

**完成条件**：用户能一键看到整张画布并回到指定阶段；小地图不遮挡主内容；延后功能不再干扰首次路径。

**自动化**：世界/小地图坐标、视口框、fit-all 边界、远近显示切换、菜单可达性、离线示例保存重开。脚本生成 English 1280×720、简中 1440×900，以及画布约 10%/50%/100% 的固定截图。

**外部参考触发点**：小地图点击/拖动或低倍率信息层级仍模糊时，只读核对外部项目的用户结果。

### B3-6 Beta 0.3 工程出口

**自动化旅程**：

1. 新建空项目并添加提示词；
2. 拖入多张图片，在落点形成参考卡；
3. 加入风格、尺寸、生成和批次并建立真实连线；
4. 复制分支并修改提示词；
5. 把两条分支放进不同阶段组；
6. 用小地图、适配全部和工作倍率导航；
7. 运行离线生成，让结果进入批次；
8. 保存、重新载入并核对内容、位置、连线、组、相机和结果。

**工程门**：定向测试、全量 GUT、lint/format、i18n、UI scaling、headless/export 检查、`git diff --check`、受保护图片检查和固定截图全部通过。

**记录**：只保留本地提交与测试日志；不写正式阶段报告，不更新 `CURRENT-STATE.md`，不构建候选包，不记录候选校验和，不使用 Computer Use，不请求项目所有者测试。自动化全绿后直接进入 Beta 0.4。

**内部状态**：`Beta 0.3 工程门通过；继续 Beta 0.4`。不单独向项目所有者交接。
