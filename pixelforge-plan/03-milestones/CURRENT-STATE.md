## 触发 Beta 0.2 的项目所有者体验证据

项目所有者在 Beta 0.2 开工前的 macOS Godot Debug 运行中确认：

1. `Set Up PixelForge` 弹窗文字/控件重叠；
2. `Recover Autosave` 及其他位置存在文字底部截断或字形异常；
3. 不同显示器和缩放环境下的 UI 可靠性尚未被真实证明；
4. 当前无可用简体中文界面；
5. 无限画布底层可操作，但普通节点主要是摘要卡，只有批次卡真正承载内容，整体工作流没有达到类似 LibTV 的内容模块工作台预期。

Beta 0.2 已为上述问题提供自动化和 agent 工程证据，但项目所有者尚未重新签收，因此不能把它们改写为人工通过。旧 M0–M7 的状态仍为：`历史工程完成 / 部分人工证据 / 当前整体人工未通过 / 未发布`。

## 项目所有者最新范围决定与执行结果

1. Beta 0.3–0.5 已连续完成图像向无限画布工程原型，但项目所有者认为当前产品仍然原始，没有给人工 `go`；
2. Beta 0.6 只做卡片、工作区入口、结果审阅和语义缩放的产品化，不继续增加新模型或新工作流能力；
3. 卡片采用混合方向：媒体/结果内容优先，输入/生成结构化；使用 PixelForge 独立视觉 token；
4. 正式 graph 卡、兼容旧 batch 卡和独立 sprite 内容卡支持自定义显示标题和自由缩放；结果卡无分页、无卡内滚动，当前筛选结果全部展开并按内容自动增高；
5. 账号、权限、协作、共享和项目版本历史永久排除；本地模型与 ComfyUI 无限期延后；
6. Beta 0.6 开发期间不使用 Computer Use，不请求分段人工测试，只运行自动化与脚本截图；
7. Beta 0.6 全部工程完成后，项目所有者一次性测试真实云端、中文、跨屏、视觉和完整旅程；
8. 人工 `go` 前不合并 `main`、不 push。

Beta 0.3–0.5 固定矩阵为 48 个核心项全部工程完成、12/18 次要项工程完成，总分 108/114；六个排除项未改写为欠债，也未用其他功能冒充得分。该结果只证明工程原型范围，不推翻项目所有者对“产品仍原始”的体验判断。Beta 0.6 的 `1A / 2B / 3C` 已冻结在视觉规格与执行书中，当前尚无 Beta 0.6 代码、截图、候选或验收结果。
