# Beta 0.3–0.5 无限画布 90% 受控矩阵

> 状态：随总路线锁定；开发中不得更换分母、主卡或用自创功能加分
>
> 总路线：[BETA-0.3-0.5-ROADMAP.md](BETA-0.3-0.5-ROADMAP.md)

## 1. 计分与证据规则

- 核心 48 项 × 2 分，次要 18 项 × 1 分，总分 114；
- 48 个核心项必须全部具有实现和工程证据；
- 再从次要项至少完成 7 项，工程候选才达到 103/114；
- 人工步骤只能追加体验签收，不能替代缺失实现；
- “已有”能力仍必须在三版最终旅程中回归，不能沿用旧绿灯直接计分；
- PixelForge 自有的失败恢复、UI 几何等硬门不计竞品分，但失败时禁止出候选。

## 2. 六个最终验收场景

| 场景 | 完整动作 |
|---|---|
| T1 空白起步 | 新建 → 添加提示词 → 拖入图片落点成卡 → 建立第一条内容分支 |
| T2 编辑与导航 | 复制分支 → 编辑 → 连线 → 成组 → 约 10% 总览 → 小地图返回局部 |
| T3 云端生成 | 选择具体云模型 → 两张参考图 → 常用参数 → 提交/失败/重试/完成 → 候选回到画布 |
| T4 审阅与继续 | 查看来源 → 审阅/加工/导出 → 结果作为参考 → 创建并运行下一分支 |
| T5 批量与复用 | 选择结构化输入行 → 运行阶段组 → 结果对应输入 → 保存工作流 → 新项目插入并运行 |
| T6 持久与质量 | 大型项目 → 保存重开/autosave → 中英文固定截图 → 唯一候选 → 项目所有者真机验收 |
