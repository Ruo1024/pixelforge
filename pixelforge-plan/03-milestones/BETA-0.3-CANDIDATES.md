# Beta 0.3 候选池（待项目所有者选择）

> 本文件不是执行计划。Beta 0.2 人工验收前，不得把下列能力顺手加入当前范围。

## 选择原则

Beta 0.3 只解决 Beta 0.2 真实使用中暴露的最主要摩擦。建议最多选择一个主主题和一个低成本配套项，避免重新进入“功能齐全但整体不好用”的节奏。

| 编号 | 候选主题 | 用户价值 | 成本/风险 | 适合在什么反馈下选择 |
|---|---|---|---|---|
| C1 | 画布分组、Frame 与可复用工作流模板 | 把一段成熟链保存、复制和复用；大型项目更像真正的创作空间 | 中；涉及组内选择、移动、保存和模板边界 | 用户已经能搭链，但反复搭同一流程或大型画布难整理 |
| C2 | 素材库、标签、搜索与变体谱系 | 让生成/清洗后的大量资产可找、可比较、可追溯 | 中；需统一元数据和批次/画布入口 | 用户最常说“产出很多，但之后找不到/不知道哪个版本好” |
| C3 | 重新设计导航：minimap、聚焦历史与语义缩放 | 在大画布中快速定位，全局与局部切换 | 中高；必须吸收 UX-4 失败证据，不能复活旧方案 | Beta 0.2 内容模块好用，但真实项目放大后明显迷路 |
| C4 | 引导式配方与模板首启 | 用“角色素材/地块/道具”等任务模板快速形成第一条链 | 低到中；需维护模板和文案，但后端新增少 | 陌生用户知道模块是什么，却仍不知该按什么顺序组合 |
| C5 | 大项目性能与后台资源管理 | 大批量图片、长工作流下保持 60 fps 和稳定内存 | 中高；必须以真实项目 profile 驱动 | Beta 0.2 在 200+ 素材或多批次下出现卡顿、内存压力 |
| C6 | 可观察的 Agent 辅助编排 | 用户描述目标后，由 Agent 创建/调整画布链，所有步骤仍可见可撤销 | 高；涉及权限、确认、失败恢复和操作审计 | 用户已经认可手动工作台，并明确需要自动搭链而非更多节点 |
| C7 | 无障碍与输入方式扩展 | 键盘遍历、焦点可见、屏幕阅读语义、色弱模式 | 中；需要系统性 UI 控件语义 | Beta 0.2 结构稳定，且目标用户或发布范围要求无障碍 |

## 初步建议

- 若 Beta 0.2 的主要问题是“内容多了难管理”，优先 `C2`，配套少量 `C1`。
- 若主要问题是“工作流会搭但画布会迷路”，优先 `C3`；必须先做新一轮原型选择，不能直接恢复旧 UX-4。
- 若主要问题是“新用户不知道怎么开始”，优先 `C4`，不要先做 Agent 自动化。
- `C6` 只有在手动工作台已被真实用户认可后才值得进入；否则 Agent 只会自动化一个尚未成立的产品流程。

## Beta 0.2 验收后需要记录的数据

1. 用户从空白画布到首次结果的耗时与求助次数；
2. 画布中模块/素材规模和迷路次数；
3. 重复搭建相同链的频率；
4. 找回历史素材/变体的耗时；
5. 性能瓶颈出现时的真实元素数、内存和帧时间；
6. 用户更想“自动搭链”还是“更好地操作/复用现有链”。

项目所有者根据这些证据勾选 Beta 0.3 范围后，再生成正式执行计划。
