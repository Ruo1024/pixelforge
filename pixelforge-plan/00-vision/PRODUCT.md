# PRODUCT.md — 产品愿景与体验原则

## 一句话定位

PixelForge 是一把「AI 像素美术瑞士军刀」：把 AI 生成的不完美像素图清洗成生产可用的真像素素材，在无限画布（既是参考板也是工作台）上用轻节点批量生成、用批次菜单批量加工，再拼接成完整的游戏场景——同时保留人工绘制介入的全部能力。

## 它不是什么

- 不是又一个"在线 AI 出图网站"。差异点在于**生成之后的全链路**：清洗→抠图→切分→风格统一→拼接→人工修补→导出，全部在一个本地工具里完成。
- 不是 Aseprite 替代品。内置编辑器目标是"修补 AI 产出的最后 30%"，不追求覆盖专业像素画家的全部工作流。
- 不是 ComfyUI 替代品。内置节点系统刻意保持简单——节点只做「AI 输入输出自由」与「批次容器」两件事，日常处理走批次菜单而非连线大图；重度工作流用户通过 ComfyUI 桥接走专业路线。

## 用户画像（按优先级）

1. **独立游戏开发者（主要）**：会用 Godot/Unity，美术能力弱，需要大量风格统一的像素素材（地块、道具、角色、特效）。痛点：AI 出图风格漂移、网格不对齐、不能直接当 tileset 用。
2. **像素美术师（次要）**：用 AI 起稿提效，但要求像素级控制权。痛点：现有 AI 工具产出"伪像素"，修起来比重画还慢。
3. **非技术爱好者（兼顾）**：不懂 SD/ComfyUI，想做小游戏或表情包素材。需要"填提示词→出可用素材"的傻瓜路径。

## 核心用户旅程（v1.0 验收场景）

> 小李在做一款 16-bit 风格的农场游戏。他打开 PixelForge，新建项目并选择「16-bit / DB32 调色板 / 32px 基准 / 无描边」风格预设。无限画布是他的参考板：他先拖入一排同类游戏的参考图，横向比对想要的观感。然后在画布上接一条**轻节点**链：风格预设节点 + 物体描述节点（填入"木桶、栅栏、稻草人"三行）+ 尺寸节点 → 云端生成节点。点击运行，结果落入一个**批次内容节点**，卡里铺开 3×4 张候选素材。他在批次卡边框点「整批清洗」，一键把整批对齐到 32×32 真像素网格并映射到 DB32（菜单路径，可撤销）。他在卡里**勾选 8 张满意的，拆出一个小批次**，再对小批次「整批描边」。对其中一张稻草人**双击——画面以无缝过渡动画放大铺满、切入编辑器**，手动修掉帽子上的杂色，加两帧摆动动画，存回。最后他把一批地块素材拖进地图拼接画板，铺出一块农田场景，导出 PNG 分层文件和 spritesheet，直接丢进游戏引擎。

这个旅程跑通且顺手，即为 MVP 达成。

### v0.1 受邀 alpha 最小旅程

v0.1 不提前承诺完整生成、地图或像素编辑能力。陌生测试者在当前主平台启动应用后，应能在无口头指导的情况下完成：

> 导入 1–5 张真实 AI 图且立即看见内容 → 批次清洗/抠图，必要时对单图切分或描边 → 查看前后结果并筛选 → 保存、关闭并重新打开 → 导出 PNG 或 spritesheet，并明确知道文件位置。

这条旅程的数据安全、过程反馈和可找到的导出缺一不可；通过 `M3.1-alpha-readiness.md` 后才称为受邀 alpha。首版只以当前主平台为发布门，三平台同时发布、签名公证、完整欢迎页、无障碍与本地化在首版后迭代。

## 体验原则（执行 AI 做 UI/交互决策时的依据）

> **统领原则 · 使用体验闭环优先**：先有「一个用户从头到尾走通一件事」的使用闭环，再有功能——闭环是**第一性设计物**，排在功能清单/模块表之上。每个功能按「服务闭环哪一步」设计（入口/反馈/出口/交接），不是先堆功能、需要体验时再补。功能「完成」＝在闭环里走通（见 ARCHITECTURE §5），孤立单测绿 ≠ 完成。里程碑规划应先立闭环、再切片实现（M3 闭环走查见 `scratch/m3-loop-walkthrough.html`）。

1. **像素清晰度优先**：任何缩放、预览、导出环节都必须最近邻插值，1:1 像素边界清晰可见。出现模糊 = P0 bug。
2. **不打断创作流**：所有 AI 任务异步执行，画布永远可操作；任务进度在画布元素上原位显示，不弹模态框。精修通过共享元素过渡动画跳转独立编辑视图，避免硬切割裂感，但不在画布坐标系内做像素编辑。
3. **画布即参考板**：无限画布的首位职责是承载海量参考做风格对比、审产出是否符合总体观感、给绘制时自由取参。生成节点、批次容器、参考卡、编组共用同一套坐标，便于横向审阅。语义缩放只切显示详略（缩略 ↔ 看清 ↔ 像素网格），不承载"放大进入编辑"。
4. **批量优先**：每个功能设计时先问"一次处理 50 张时体验如何"。单张操作是批量操作的特例。批次内容节点是批量的一等载体。
5. **处理走菜单、节点可复现（both-and）**：默认用批次卡边框菜单一键整批处理（即时、可撤销、记 provenance）；需要固定可重跑的流水线时才连处理节点。两条路调同一份 core 算法，差别只在"记不记进图里"。
6. **可撤销**：一切修改画布/图像状态的操作进 undo 栈（Ctrl+Z 全局可用）。AI 生成结果作为新元素出现，永不覆盖原图。
7. **渐进暴露复杂度**：默认界面只露核心路径（生成→批次菜单→导出）。节点参数折叠高级项；预设工具节点、插件、ComfyUI 桥、自定义脚本藏在"扩展"菜单。
8. **本地数据主权**：项目文件、素材库、API key 全部本地存储。云端调用仅发送必要的提示词与参考图，明确告知。
9. **键盘友好**：高频操作（平移空格、缩放滚轮、框选 V、抠图 W、切分 S）有快捷键，与 Aseprite/PS 习惯对齐。

## 风格预设——产品灵魂功能的设计依据

调研结论（详见 04-research）：AI 像素工具最大痛点是**风格一致性**。PixelForge 的答案是把"风格"做成一等公民数据对象（详见 contracts/STYLE-PRESETS.md）：

- 风格预设 = 分辨率档位 × 调色板 × 基准尺寸 × 描边规则 × 抖动规则 × 透视类型 + 提示词模板片段。
- 预设同时作用于**生成端**（拼接进提示词、选择 LoRA/风格参数）与**清洗端**（量化目标调色板、网格尺寸先验、描边后处理），双端夹击保证一致性。
- 内置 6 档历史风格预设（GB 4色 / NES / 16-bit / Hi-bit / 1-bit / HD-2D 素材），用户可派生自定义。

## 版本里程碑对应的产品形态

| 版本 | 形态 | 可独立交付的价值 |
|---|---|---|
| v0.1 (M0–M3 + M3.1 门禁) | 像素清洗台（受邀 alpha） | 真实图导入 → 批次清洗/抠图 → 单图切分/描边 → 保存重开 → 可找到的导出，无需 API key |
| v0.3 (M4-V1 决策后进入 M4) | 生成工作台 | 轻节点工作流 + 云端生成，端到端素材生产 |
| v0.5 (M5–M6) | 完整工作台（MVP） | + 地图拼接、动效合成、手绘修补 |
| v1.0 (M7) | 可扩展平台 | + 插件系统、ComfyUI 桥、本地模型 |
| v2.x (M8) | 远期 | 体素生成与编辑（评估后立项） |
