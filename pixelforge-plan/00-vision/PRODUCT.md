# PRODUCT.md — 产品愿景与体验原则

## 一句话定位

PixelForge 是一把「AI 像素美术瑞士军刀」：把 AI 生成的不完美像素图清洗成生产可用的真像素素材，在无限画布上用节点工作流批量生成与加工，再拼接成完整的游戏场景——同时保留人工绘制介入的全部能力。

## 它不是什么

- 不是又一个"在线 AI 出图网站"。差异点在于**生成之后的全链路**：清洗→抠图→切分→风格统一→拼接→人工修补→导出，全部在一个本地工具里完成。
- 不是 Aseprite 替代品。内置编辑器目标是"修补 AI 产出的最后 30%"，不追求覆盖专业像素画家的全部工作流。
- 不是 ComfyUI 替代品。内置节点系统刻意保持简单（线性为主、少量分支），重度工作流用户通过 ComfyUI 桥接走专业路线。

## 用户画像（按优先级）

1. **独立游戏开发者（主要）**：会用 Godot/Unity，美术能力弱，需要大量风格统一的像素素材（地块、道具、角色、特效）。痛点：AI 出图风格漂移、网格不对齐、不能直接当 tileset 用。
2. **像素美术师（次要）**：用 AI 起稿提效，但要求像素级控制权。痛点：现有 AI 工具产出"伪像素"，修起来比重画还慢。
3. **非技术爱好者（兼顾）**：不懂 SD/ComfyUI，想做小游戏或表情包素材。需要"填提示词→出可用素材"的傻瓜路径。

## 核心用户旅程（v1.0 验收场景）

> 小李在做一款 16-bit 风格的农场游戏。他打开 PixelForge，新建项目并选择「16-bit / DB32 调色板 / 32px 基准 / 无描边」风格预设。在无限画布上他搭了一条节点链：风格预设节点 → 尺寸比例节点 → 物体描述节点（填入"木桶、栅栏、稻草人"三行）→ 云端生成节点 → 像素清洗节点 → 自动切分节点。点击运行，画布上铺开 3×4 张候选素材，全部已对齐到 32×32 真像素网格并映射到 DB32。他框选 8 张满意的拖进素材库，对其中一张稻草人双击进入编辑器，手动修掉帽子上的杂色，加两帧摆动动画。最后他把一批地块素材拖进地图拼接画板，铺出一块农田场景，导出 PNG 分层文件和 spritesheet，直接丢进游戏引擎。

这个旅程跑通且顺手，即为 MVP 达成。

## 体验原则（执行 AI 做 UI/交互决策时的依据）

1. **像素清晰度优先**：任何缩放、预览、导出环节都必须最近邻插值，1:1 像素边界清晰可见。出现模糊 = P0 bug。
2. **不打断创作流**：所有 AI 任务异步执行，画布永远可操作；任务进度在画布元素上原位显示，不弹模态框。
3. **批量优先**：每个功能设计时先问"一次处理 50 张时体验如何"。单张操作是批量操作的特例。
4. **可撤销**：一切修改画布/图像状态的操作进 undo 栈（Ctrl+Z 全局可用）。AI 生成结果作为新元素出现，永不覆盖原图。
5. **渐进暴露复杂度**：默认界面只露核心路径（生成→清洗→导出）。节点参数折叠高级项；插件、ComfyUI 桥、自定义脚本藏在"扩展"菜单。
6. **本地数据主权**：项目文件、素材库、API key 全部本地存储。云端调用仅发送必要的提示词与参考图，明确告知。
7. **键盘友好**：高频操作（平移空格、缩放滚轮、框选 V、抠图 W、切分 S）有快捷键，与 Aseprite/PS 习惯对齐。

## 风格预设——产品灵魂功能的设计依据

调研结论（详见 04-research）：AI 像素工具最大痛点是**风格一致性**。PixelForge 的答案是把"风格"做成一等公民数据对象（详见 contracts/STYLE-PRESETS.md）：

- 风格预设 = 分辨率档位 × 调色板 × 基准尺寸 × 描边规则 × 抖动规则 × 透视类型 + 提示词模板片段。
- 预设同时作用于**生成端**（拼接进提示词、选择 LoRA/风格参数）与**清洗端**（量化目标调色板、网格尺寸先验、描边后处理），双端夹击保证一致性。
- 内置 6 档历史风格预设（GB 4色 / NES / 16-bit / Hi-bit / 1-bit / HD-2D 素材），用户可派生自定义。

## 版本里程碑对应的产品形态

| 版本 | 形态 | 可独立交付的价值 |
|---|---|---|
| v0.1 (M0–M2) | 像素清洗台 | 拖入 AI 图 → 真像素化 + 抠图切分，无需任何 API key |
| v0.3 (M3–M4) | 生成工作台 | 节点工作流 + 云端生成，端到端素材生产 |
| v0.5 (M5–M6) | 完整工作台（MVP） | + 地图拼接、动效合成、手绘修补 |
| v1.0 (M7) | 可扩展平台 | + 插件系统、ComfyUI 桥、本地模型 |
| v2.x (M8) | 远期 | 体素生成与编辑（评估后立项） |
